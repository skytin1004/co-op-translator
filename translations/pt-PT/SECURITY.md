## Segurança

A Microsoft leva a sério a segurança dos nossos produtos e serviços de software, o que inclui todos os repositórios de código-fonte geridos através das nossas organizações no GitHub, que incluem [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) e [Xamarin](https://github.com/xamarin).

Se acredita ter encontrado uma vulnerabilidade de segurança em algum repositório detido pela Microsoft que cumpra a [definição da Microsoft de vulnerabilidade de segurança](https://aka.ms/security.md/definition), por favor reporte-a conforme descrito abaixo.

## Reportar Problemas de Segurança

**Por favor, não reporte vulnerabilidades de segurança através de issues públicas no GitHub.**

Em vez disso, reporte-as ao Microsoft Security Response Center (MSRC) em [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

Se preferir submeter sem iniciar sessão, envie um email para [secure@microsoft.com](mailto:secure@microsoft.com). Se possível, encripte a sua mensagem com a nossa chave PGP; por favor, faça o download a partir da [página da chave PGP do Microsoft Security Response Center](https://aka.ms/security.md/msrc/pgp).

Deverá receber uma resposta dentro de 24 horas. Se por alguma razão não receber, por favor faça um seguimento por email para garantir que recebemos a sua mensagem original. Informação adicional pode ser encontrada em [microsoft.com/msrc](https://www.microsoft.com/msrc).

Por favor, inclua as informações solicitadas abaixo (na medida do possível) para nos ajudar a compreender melhor a natureza e o âmbito do possível problema:

  * Tipo de problema (ex. buffer overflow, injeção SQL, cross-site scripting, etc.)
  * Caminhos completos do(s) ficheiro(s) fonte relacionados com a manifestação do problema
  * O local do código-fonte afetado (tag/branch/commit ou URL direto)
  * Qualquer configuração especial necessária para reproduzir o problema
  * Passo a passo para reproduzir o problema
  * Código de prova de conceito ou exploit (se possível)
  * Impacto do problema, incluindo como um atacante poderia explorar o problema

Esta informação ajudará a priorizar o seu reporte de forma mais rápida.

Se está a reportar para um bug bounty, reportes mais completos podem contribuir para uma recompensa maior. Por favor, visite a nossa página do [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) para mais detalhes sobre os nossos programas ativos.

## Línguas Preferidas

Preferimos que todas as comunicações sejam feitas em inglês.

## Política

A Microsoft segue o princípio de [Divulgação Coordenada de Vulnerabilidades](https://aka.ms/security.md/cvd).
