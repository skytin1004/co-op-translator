## セキュリティ

Microsoft は、当社のソフトウェア製品およびサービスのセキュリティを真剣に考えており、これには [Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet)、[Xamarin](https://github.com/xamarin) を含む当社の GitHub 組織で管理されるすべてのソースコードリポジトリが含まれます。

Microsoft が定義する [セキュリティ脆弱性](https://aka.ms/security.md/definition) に該当する Microsoft 所有のリポジトリでセキュリティ脆弱性を発見したと思われる場合は、以下に記載された方法でご報告ください。

## セキュリティ問題の報告

**セキュリティ脆弱性は、GitHub の公開イシューを通じて報告しないでください。**

代わりに、Microsoft Security Response Center (MSRC) の [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) で報告してください。

ログインせずに提出したい場合は、[secure@microsoft.com](mailto:secure@microsoft.com) へメールを送信してください。可能であれば、当社の PGP 鍵でメッセージを暗号化してください。PGP 鍵は [Microsoft Security Response Center PGP Key ページ](https://aka.ms/security.md/msrc/pgp) からダウンロードできます。

通常、24時間以内に返信を受け取れます。何らかの理由で返信がない場合は、メールでフォローアップし、元のメッセージが当社に届いていることを確認してください。追加情報は [microsoft.com/msrc](https://www.microsoft.com/msrc) にあります。

問題の性質と範囲をよりよく理解するために、以下のリクエストされた情報を可能な限り含めてください：

  * 問題の種類（例：バッファオーバーフロー、SQLインジェクション、クロスサイトスクリプティングなど）
  * 問題の発生に関連するソースファイルの完全なパス
  * 影響を受けるソースコードの場所（タグ/ブランチ/コミットまたは直接 URL）
  * 問題を再現するために必要な特別な構成
  * 問題を再現するための手順
  * プルーフ・オブ・コンセプトまたはエクスプロイトコード（可能な場合）
  * 問題の影響および攻撃者が問題をどのように悪用するか

これらの情報は、報告を迅速に分類するのに役立ちます。

バグ報奨金で報告する場合は、より完全な報告が報奨金の金額を増やすことに貢献します。現在のプログラムの詳細は、当社の [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ページをご覧ください。
