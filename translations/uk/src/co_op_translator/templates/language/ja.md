# Локалізація Markdown-файлів

Цей документ описує процес локалізації Markdown-файлів для нашого проєкту.

## Основні правила

- Зберігайте структуру Markdown без змін.
- Не змінюйте шляхи посилань.
- Локалізуйте лише текстовий вміст, що читається людиною.

## Приклади

Цей документ використовує [Co-op Translator](https://github.com/Azure/co-op-translator) для автоматизації процесу.

## Поради

[!TIP]
Слідкуйте за узгодженістю термінів у всьому документі.

[!WARNING]
Не редагуйте URL-адреси під час локалізації.
