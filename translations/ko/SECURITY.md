## 보안

Microsoft는 당사의 소프트웨어 제품 및 서비스의 보안을 매우 중요하게 생각하며, 여기에는 [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) 및 [Xamarin](https://github.com/xamarin)을 포함하는 GitHub 조직을 통해 관리되는 모든 소스 코드 저장소가 포함됩니다.

[Microsoft의 보안 취약점 정의](https://aka.ms/security.md/definition)에 부합하는 Microsoft 소유 저장소에서 보안 취약점을 발견했다고 생각되면 아래에 설명된 대로 신고해 주시기 바랍니다.

## 보안 문제 신고

**보안 취약점은 공개 GitHub 이슈를 통해 신고하지 마십시오.**

대신 Microsoft 보안 대응 센터(MSRC) 웹사이트 [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)에서 신고해 주십시오.

로그인 없이 제출하시려면 이메일로 [secure@microsoft.com](mailto:secure@microsoft.com)으로 보내 주십시오. 가능하면 당사 PGP 키로 메시지를 암호화해 주세요; [Microsoft Security Response Center PGP Key 페이지](https://aka.ms/security.md/msrc/pgp)에서 다운로드할 수 있습니다.

24시간 이내에 답변을 받으실 수 있습니다. 만약 그렇지 않은 경우 원본 메시지가 접수되었는지 확인하기 위해 이메일로 다시 연락해 주십시오. 자세한 정보는 [microsoft.com/msrc](https://www.microsoft.com/msrc)에서 확인할 수 있습니다.

가능한 한 아래의 요청된 정보를 포함해 주시면 문제의 성격과 범위를 더 잘 이해하는 데 도움이 됩니다:

  * 문제 유형(예: 버퍼 오버플로, SQL 인젝션, 크로스 사이트 스크립팅 등)
  * 문제 발생과 관련된 소스 파일 전체 경로
  * 영향을 받는 소스 코드 위치(태그/브랜치/커밋 또는 직접 URL)
  * 문제 재현을 위해 필요한 특별한 구성
  * 문제 재현을 위한 단계별 지침
  * 개념 증명 또는 익스플로잇 코드(가능한 경우)
  * 문제의 영향, 공격자가 문제를 어떻게 악용할 수 있는지 포함

이 정보는 신고를 보다 신속하게 분류하는 데 도움이 됩니다.

버그 현상금 제보를 하시는 경우, 보다 완전한 보고서는 더 높은 현상금 수상에 기여할 수 있습니다. 당사의 [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) 페이지에서 활성화된 프로그램에 대한 자세한 내용을 확인해 주십시오.
