## Security

Microsoft আমাদের সফটওয়্যার পণ্য এবং পরিষেবার নিরাপত্তাকে গুরুত্ব সহকারে নেয়, যার মধ্যে আমাদের GitHub সংস্থাগুলির মাধ্যমে পরিচালিত সমস্ত সোর্স কোড রিপোজিটরি অন্তর্ভুক্ত, যার মধ্যে রয়েছে [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) এবং [Xamarin](https://github.com/xamarin)।

আপনি যদি বিশ্বাস করেন যে আপনি কোনও Microsoft-মালিকানাধীন রিপোজিটরিতে একটি নিরাপত্তা দুর্বলতা পেয়েছেন যা [Microsoft-এর নিরাপত্তা দুর্বলতার সংজ্ঞা](https://aka.ms/security.md/definition) পূরণ করে, তাহলে অনুগ্রহ করে এটি নীচে বর্ণিত হিসাবে আমাদের প্রতিবেদন করুন।

## Reporting Security Issues

**দয়া করে পাবলিক GitHub ইস্যুর মাধ্যমে নিরাপত্তা দুর্বলতা রিপোর্ট করবেন না।**

তার পরিবর্তে, অনুগ্রহ করে Microsoft Security Response Center (MSRC) এ রিপোর্ট করুন [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)।

আপনি যদি লগইন ছাড়াই জমা দিতে ইচ্ছুক হন, তাহলে ইমেইল পাঠান [secure@microsoft.com](mailto:secure@microsoft.com)। সম্ভব হলে, আমাদের PGP কী দিয়ে আপনার বার্তাটি এনক্রিপ্ট করুন; দয়া করে এটি [Microsoft Security Response Center PGP Key পৃষ্ঠা](https://aka.ms/security.md/msrc/pgp) থেকে ডাউনলোড করুন।

আপনি ২৪ ঘণ্টার মধ্যে একটি প্রতিক্রিয়া পাওয়ার কথা। কোনো কারণে আপনি না পেলে, নিশ্চিত করার জন্য ইমেইলের মাধ্যমে ফলো আপ করুন যাতে আমরা আপনার মূল বার্তাটি পেয়েছি। অতিরিক্ত তথ্য পাওয়া যাবে [microsoft.com/msrc](https://www.microsoft.com/msrc) এ। 

দয়া করে নিচের অনুরোধকৃত তথ্য অন্তর্ভুক্ত করুন (যতটা সম্ভব প্রদান করুন) যাতে আমরা সম্ভাব্য সমস্যার প্রকৃতি এবং পরিধি ভালভাবে বুঝতে পারি:

  * সমস্যার ধরন (যেমন: বাফার ওভারফ্লো, SQL ইনজেকশন, ক্রস-সাইট স্ক্রিপ্টিং ইত্যাদি)
  * সম্পর্কিত সোর্স ফাইল(গুলি) এর সম্পূর্ণ পাথ
  * প্রভাবিত সোর্স কোডের অবস্থান (ট্যাগ/ব্রাঞ্চ/কমিট বা সরাসরি URL)
  * সমস্যা পুনরুত্পাদনের জন্য প্রয়োজনীয় বিশেষ কনফিগারেশন
  * সমস্যা পুনরুত্পাদনের ধাপে ধাপে নির্দেশাবলী
  * প্রমাণ-সিদ্ধান্ত বা এক্সপ্লয়িট কোড (যদি সম্ভব হয়)
  * সমস্যার প্রভাব, যার মধ্যে একজন আক্রমণকারী কীভাবে সমস্যাটি ব্যবহার করতে পারে

এই তথ্যটি আমাদের আপনার প্রতিবেদন দ্রুত উদ্ধার করতে সাহায্য করবে।

আপনি যদি বাগ বাউন্টির জন্য রিপোর্ট করছেন, তবে আরো সম্পূর্ণ প্রতিবেদন উচ্চতর বাউন্টি পুরস্কারে সহায়ক হতে পারে। আমাদের সক্রিয় প্রোগ্রামগুলোর বিস্তারিত জানতে অনুগ্রহ করে আমাদের [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) পেজ দেখুন।
