# Command reference

The **Co-op Translator** CLI বিভিন্ন বিকল্প অফার করে অনুবাদ প্রক্রিয়া কাস্টমাইজ করার জন্য:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | আপনার প্রোজেক্ট নির্দিষ্ট ভাষায় অনুবাদ করে। উদাহরণ: translate -l "es fr de" স্প্যানিশ, ফরাসি, এবং জার্মান ভাষায় অনুবাদ করে। translate -l "all" ব্যবহার করুন সমস্ত সমর্থিত ভাষায় অনুবাদ করার জন্য।
translate -l "language_codes" -u              | অনুবাদ আপডেট করে বিদ্যমান অনুবাদ মুছে ফেলে এবং পুনরায় তৈরি করে। সতর্কতা: এটি নির্দিষ্ট ভাষাগুলোর সমস্ত চলতি অনুবাদ মুছে ফেলবে।
translate -l "language_codes" -img            | কেবল চিত্র ফাইলসমূহ অনুবাদ করে।
translate -l "language_codes" -md             | কেবল Markdown ফাইলসমূহ অনুবাদ করে।
translate -l "language_codes" -nb             | কেবল Jupyter notebook ফাইলসমূহ (.ipynb) অনুবাদ করে।
translate -l "language_codes" --fix           | পূর্ববর্তী মূল্যায়ন ফলাফলের ভিত্তিতে কম বিশ্বাসযোগ্য স্কোর সহ ফাইলগুলো পুনরায় অনুবাদ করে।
translate -l "language_codes" -d              | বিশদ লগিংয়ের জন্য ডিবাগ মোড সক্ষম করে।
translate -l "language_codes" --save-logs, -s | DEBUG-স্তরের লগ <root_dir>/logs/-এর অধীনে ফাইলে সংরক্ষণ করে (কনসোল নিয়ন্ত্রণ থাকে -d দ্বারা)
translate -l "language_codes" -r "root_dir"   | প্রোজেক্টের মূল ডিরেক্টরি নির্ধারণ করে
translate -l "language_codes" -f              | চিত্র অনুবাদের জন্য দ্রুত মোড ব্যবহার করে (গুণমান এবং সামঞ্জস্যে সামান্য খরচে ৩ গুণ দ্রুত প্লটিং)।
translate -l "language_codes" -y              | সমস্ত প্রম্পট স্বয়ংক্রিয়ভাবে নিশ্চিত করে (CI/CD পাইপলাইনের জন্য সুবিধাজনক)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | অনুবাদকৃত markdown এবং নোটবুকগুলিতে মেশিন অনুবাদ অস্বীকৃতি অংশ যোগ করা সক্ষম বা নিষ্ক্রিয় করে (ডিফল্ট: সক্ষম)।
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | আপনার রিপোজিটরি URL দিয়ে README ভাষার অংশকে ব্যক্তিগতকরণ করে (স্পার্স চেকআউট)।
translate -l "language_codes" --help          | CLI এর মধ্যে উপলব্ধ কমান্ডসমূহের সহায়তা বিস্তারিত
evaluate -l "language_code"                  | নির্দিষ্ট ভাষার অনুবাদ মান মূল্যায়ন করে এবং বিশ্বাসযোগ্যতা স্কোর প্রদান করে
evaluate -l "language_code" -c 0.8           | কাস্টম বিশ্বাসযোগ্যতা থ্রেশহোল্ড সহ অনুবাদ মূল্যায়ন করে
evaluate -l "language_code" -f               | দ্রুত মূল্যায়ন মোড (শুধুমাত্র নিয়ম-ভিত্তিক, কোন LLM নয়)
evaluate -l "language_code" -D               | গভীর মূল্যায়ন মোড (শুধুমাত্র LLM-ভিত্তিক, আরও ব্যাপক কিন্তু ধীর)
evaluate -l "language_code" --save-logs, -s  | DEBUG-স্তরের লগ <root_dir>/logs/-এ ফাইলে সংরক্ষণ করে
migrate-links -l "language_codes"             | অনুবাদকৃত Markdown ফাইল পুনরায় প্রক্রিয়া করে নোটবুকগুলোর (.ipynb) লিঙ্ক আপডেট করে। অনুবাদকৃত নোটবুক পাওয়া গেলে তা পছন্দ করে; অন্যথায় মূল নোটবুক fallback করতে পারে।
migrate-links -l "language_codes" -r          | প্রোজেক্ট মূল ডিরেক্টরি নির্দিষ্ট করে (ডিফল্ট: বর্তমান ডিরেক্টরি)।
migrate-links -l "language_codes" --dry-run   | কোন ফাইল পরিবর্তন হবে তা দেখায়, কোন পরিবর্তন লেখে না।
migrate-links -l "language_codes" --no-fallback-to-original | অনুবাদকৃত নোটবুক না থাকলে মূল নোটবুকের লিঙ্ক পুনর্লিখন না করে (শুধুমাত্র অনুবাদকৃত লিঙ্ক আপডেট করে)।
migrate-links -l "language_codes" -d          | বিশদ লগিংয়ের জন্য ডিবাগ মোড সক্ষম করে।
migrate-links -l "language_codes" --save-logs, -s | DEBUG-স্তরের লগ <root_dir>/logs/-এ ফাইলে সংরক্ষণ করে
migrate-links -l "all" -y                      | সমস্ত ভাষা প্রক্রিয়া করে এবং সতর্কতা প্রম্পট স্বয়ংক্রিয়ভাবে নিশ্চিত করে।

## Usage examples

  1. ডিফল্ট আচরণ (বিদ্যমান অনুবাদ মুছে না ফেলে নতুন অনুবাদ যোগ করুন):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. শুধুমাত্র নতুন কোরিয়ান চিত্র অনুবাদ যোগ করুন (কোন বিদ্যমান অনুবাদ মুছে ফেলা হয় না):    translate -l "ko" -img

  3. সমস্ত কোরিয়ান অনুবাদ আপডেট করুন (সতর্কতা: এটি সমস্ত বিদ্যমান কোরিয়ান অনুবাদ মুছে ফেলে পুনরায় অনুবাদ করে):    translate -l "ko" -u

  4. শুধুমাত্র কোরিয়ান চিত্র আপডেট করুন (সতর্কতা: এটি সমস্ত বিদ্যমান কোরিয়ান চিত্র মুছে ফেলে পুনরায় অনুবাদ করে):    translate -l "ko" -img -u

  5. অন্য অনুবাদগুলোর ওপর প্রভাব না ফেলে কোরিয়ানের জন্য নতুন মার্কডাউন অনুবাদ যোগ করুন:    translate -l "ko" -md

  6. পূর্ববর্তী মূল্যায়ন ফলাফলের ভিত্তিতে কম বিশ্বাসযোগ্য অনুবাদ ঠিক করুন: translate -l "ko" --fix

  7. নির্দিষ্ট ফাইলগুলোর জন্য কেবল কম বিশ্বাসযোগ্য অনুবাদ (মার্কডাউন) ঠিক করুন: translate -l "ko" --fix -md

  8. নির্দিষ্ট ফাইলগুলোর জন্য কেবল কম বিশ্বাসযোগ্য অনুবাদ (চিত্র) ঠিক করুন: translate -l "ko" --fix -img

  9. চিত্র অনুবাদের জন্য দ্রুত মোড ব্যবহার করুন:    translate -l "ko" -img -f

  10. কাস্টম থ্রেশহোল্ড সহ কম বিশ্বাসযোগ্য অনুবাদ ঠিক করুন: translate -l "ko" --fix -c 0.8

  11. ডিবাগ মোড উদাহরণ: - translate -l "ko" -d: ডিবাগ লগিং সক্রিয় করুন।
  12. লগ ফাইলে সংরক্ষণ করুন: translate -l "ko" -s
  13. কনসোল DEBUG এবং ফাইল DEBUG: translate -l "ko" -d -s
  14. আউটপুটে মেশিন অনুবাদ অস্বীকৃতি যোগ না করেই অনুবাদ করুন: translate -l "ko" --no-disclaimer

  15. কোরিয়ান অনুবাদের জন্য নোটবুক লিঙ্ক স্থানান্তর করুন (উপলব্ধ থাকলে অনুবাদকৃত নোটবুকের লিঙ্ক আপডেট করুন):    migrate-links -l "ko"

  15. ড্রাই-রান সহ লিঙ্ক স্থানান্তর (কোন ফাইল লেখে না):    migrate-links -l "ko" --dry-run

  16. শুধুমাত্র অনুবাদকৃত নোটবুক থাকলে লিঙ্ক আপডেট করুন (মূল লিঙ্কের fallback করবেন না):    migrate-links -l "ko" --no-fallback-to-original

  17. নিশ্চিতকরণ প্রম্পট সহ সমস্ত ভাষা প্রক্রিয়া করুন:    migrate-links -l "all"

  18. সমস্ত ভাষা প্রক্রিয়া করুন এবং স্বয়ংক্রিয়ভাবে নিশ্চিত করুন:    migrate-links -l "all" -y
  19. migrate-links এর জন্য লগ ফাইলে সংরক্ষণ করুন:    migrate-links -l "ko ja" -s

  20. README ভাষার পরামর্শকে আপনার রিপোজিটরি URL সহ ব্যক্তিগতকরণ করুন:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Evaluation Examples

> [!WARNING]  
> **Beta Feature**: মূল্যায়ন কার্যকারিতা বর্তমানে বিটাতে আছে। এই ফিচারটি অনুবাদকৃত ডকুমেন্ট মূল্যায়নের জন্য মুক্তি পেয়েছে, এবং মূল্যায়ন পদ্ধতি ও বিস্তারিত বাস্তবায়ন এখনও উন্নয়নের অধীনে এবং পরিবর্তনের জন্য খোলা।

  1. কোরিয়ান অনুবাদ মূল্যায়ন করুন: evaluate -l "ko"

  2. কাস্টম বিশ্বাসযোগ্যতা থ্রেশহোল্ড সহ মূল্যায়ন করুন: evaluate -l "ko" -c 0.8

  3. দ্রুত মূল্যায়ন (শুধুমাত্র নিয়ম-ভিত্তিক): evaluate -l "ko" -f

  4. গভীর মূল্যায়ন (শুধুমাত্র LLM-ভিত্তিক): evaluate -l "ko" -D
