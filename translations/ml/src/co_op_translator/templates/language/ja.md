# വാചകരേറ്റ് കൂടുതൽ സൗകര്യമുള്ള ഭാഷാ മാറ്റം

[Speech Service ലാംഗ്വേജ് അണ്ടര്സ്റ്റാൻഡിങ്](https://learn.microsoft.com/azure/cognitive-services/speech-service/language-understanding) മോഡലുകൾ വാചകരേറ്റ് കൂടുതൽ സൗകര്യമാകുന്ന കസ്റ്റം ഇന്റന്റുകൾക്ക് സഹായകമാണ്. ഇവിടെ ഒരു [കസ്റ്റം സ്ലോട്ട് ടൈപ്പ്](https://learn.microsoft.com/azure/cognitive-services/luis/luis-reference-prebuilt-entities) നിർമ്മിച്ച് ഇതിനെ വാചകരേറ്റ് സാധാരണയാക്കുന്നതിനുള്ള മാർഗ്ഗങ്ങൾ വിശദീകരിച്ചിരിക്കുന്നു.

## വാചക എൻജിനിയറിംഗ് (Language Engineering) ഉദ്ദേശ്യം

വാചകരേറ്റ് കൂടുതൽ സൗകര്യമാക്കുക എന്നത് നമ്മുടെ മുഖ്യ ലക്ഷ്യമാണ്. അതായത്, ഉപയോക്താക്കളുടെ ഉപയോഗത്തിലെ വാചകങ്ങളിൽ മാറ്റങ്ങൾ ഉണ്ടാകുമ്പോഴെങ്കിലും, ഇവയെയും ശരിയായി തിരിച്ചറിയാൻ സഹായിക്കുക.

## ഉദാഹരണം

```plaintext
ഉപയോക്താവ്: "അന്ന് പാലാ യാത്ര"
സ്ലോട്ട്: സ്ഥലം -> പാലാ
```

എങ്കിൽ, ഉപയോക്താവ് "പാലാ യാത്ര നാളെ" എന്നും പറഞ്ഞാലോ അതേ സ്ലോട്ട് ശരിയായി തിരിച്ചറിയാനാകും.

## എങ്ങനെ സാധ്യമാക്കാം?

- കസ്റ്റം സ്ലോട്ടുകൾ സൃഷ്ടിക്കുക.
- വാചക വ്യത്യാസങ്ങൾ ഉൾപ്പെടുത്തുക.
- സാധാരണ സ്ലോട്ടുകൾക്ക് വ്യത്യസ്ത വാചകത്തിന്റെയും ഉദാഹരണങ്ങൾ ചേർക്കുക.

[Speech Service സ്ലോട്ടിംഗ്](https://learn.microsoft.com/azure/cognitive-services/speech-service/language-understanding-slots) കുറിച്ച് കൂടുതൽ വിവരങ്ങൾക്കായി സന്ദർശിക്കുക.
