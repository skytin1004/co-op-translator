## Security

Microsoft ഞങ്ങളുടെ സോഫ്റ്റ്വെയർ ഉൽപ്പന്നങ്ങളും സേവനങ്ങളും ഉൾപ്പെടെ, [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ಮತ್ತು [Xamarin](https://github.com/xamarin) ഉൾപ്പെടെയുള്ള GitHub സംഘടനകളിലൂടെ നിയന്ത്രിക്കുന്ന എല്ലാ ഉറവിട കോഡ് രെപ്പോസിറ്ററികളുടെയും സുസ്ഥിരതയെ ഗുരുതരമായി കാണുന്നു.

[Microsoftയുടെ സുസ്ഥിരതയുടെ നിർവചനത്തേക്കുറിച്ച്](https://aka.ms/security.md/definition) യോജിക്കുന്ന ഏതൊരു Microsoft ഉടമസ്ഥതയിലുള്ള രെപ്പോസിറ്ററിയിൽ സുരക്ഷാ ത732ashaം കണ്ടെത്തിയെന്ന് നിങ്ങൾ കരുതുന്നുണ്ടെങ്കിൽ, ദയവായി ചുവടെ വിവരിച്ചിരിക്കുന്നവിധം ഞങ്ങൾക്കു റിപ്പോർട്ട് ചെയ്യുക.

## Reporting Security Issues

**സുരക്ഷാ ത732ashaങ്ങൾ പൊതുവായ GitHub വിഷയങ്ങൾ വഴി റിപ്പോർട്ട് ചെയ്യരുത്.**

പകരം, ദയവായി Microsoft Security Response Center (MSRC) ൽ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) വഴി അവ അറിയിക്കുക.

ലോഗിൻ ചെയ്യാതെ സമർപ്പിക്കാൻ ആഗ്രഹിക്കുന്നവർക്ക്, [secure@microsoft.com](mailto:secure@microsoft.com) എന്ന ഇമെയിലിലേക്ക് അയയ്ക്കുക. കഴിഞാൽ, നമ്മുടെ PGP കീ ഉപയോഗിച്ച് നിങ്ങളുടെ സന്ദേശം എൻക്രിപ്റ്റ് ചെയ്യുക; അത് [Microsoft Security Response Center PGP Key പേജ്](https://aka.ms/security.md/msrc/pgp) മുതൽ ഡൗൺലോഡ് ചെയ്യുക.

24 മണിക്കൂറിനുള്ളിൽ നിങ്ങൾക്ക് മറുപടി ലഭിക്കണം. എന്തെങ്കിലുമൊരു കാരണത്താൽ ഉണ്ടാകുന്നില്ലെങ്കിൽ, ദയവായി ഞങ്ങളുടെ ആദ്യ സന്ദേശം ലഭിച്ചുവെന്ന് ഉറപ്പു വരുത്താൻ ഇമെയിലിലൂടെ പിന്തുടരുക. അധിക വിവരങ്ങൾ [microsoft.com/msrc](https://www.microsoft.com/msrc) ൽ ലഭ്യമാണ്.

താഴെക്കാണുന്ന ആവശ്യമായ വിവരങ്ങൾ (വരുന്നതുപോൽ നിങ്ങൾ നൽകാൻ കഴിയുന്നത്ര) ഉൾപ്പെടുത്തുക, ഇതുവഴി പ്രശ്നത്തിന്റെ സ്വഭാവവും പരിധിയും ഞങ്ങൾക്കു മെച്ചമായി മനസ്സിലാക്കാൻ സഹായിക്കും:

  * പ്രശ്‌ന തരം (ഉദാ. ബഫർ ഓവർഫ്ലോ, SQL ഇഞ്ചക്ഷൻ, ക്രോസ്-സൈറ്റ് സ്ക്രിപ്റ്റിംഗ്, മുതലായവ)
  * പ്രശ്‌നത്തെ ബാധിക്കുന്ന ഉറവിട ഫയൽ(കൾ)ന്റെ പൂർണ്ണ പാതകൾ
  * ബാധിച്ച ഉറവിട കോഡിന്റെ സ്ഥാനം (ടാഗ്/ബ്രാഞ്ച്/കോമിറ്റ് അല്ലെങ്കിൽ നേരിട്ടുള്ള URL)
  * പ്രശ്നം പുനരാവൃതിയാക്കാൻ ആവശ്യമായ പ്രത്യേക കോംഫിഗറേഷൻ
  * പ്രശ്നം പുനരാവൃത്തി ചെയ്യുന്ന കാൽപാടുകൾ
  * പ്രൂഫ്-ഓഫ്-കൺസെപ്റ്റ് അല്ലെങ്കിൽ എക്സ്പ്ലോയിറ്റ് കോഡ് (കഴിയുമ്പോൾ)
  * പ്രശ്‌നത്തിന്റെ പ്രഭാവം, ആക്രമകൻ എങ്ങനെ ഈ പ്രശ്നം ഉപയോഗപ്പെടുത്താൻ കഴിയും എന്നതിന്റെ വിശദാംശങ്ങൾ

ഈ വിവരങ്ങൾ നിങ്ങളുടെ റിപ്പോർട്ട് കൂടുതൽ വേഗത്തിൽ പരിശോധിക്കാൻ ഞങ്ങളെ സഹായിക്കും.

ബഗ് ബൗന്റിക്ക് റിപ്പോർട്ട് നൽകുമ്പോൾ, കൂടുതൽ സമഗ്രമായ റിപ്പോർട്ടുകൾക്ക് ഉയർന്ന ബൗണ്ടി അവാർഡ് ലഭിക്കുന്നതിനുള്ള സാദ്ധ്യത കൂടുതലാണ്. നമ്മുടെ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) പേജ് വഴിതുടർന്ന് നമ്മുടെ ക്രിയാത്മക പ്രോഗ്രാമുകൾക്കുറിച്ചുള്ള കൂടുതൽ വിവരങ്ങൾ കാണുക.
