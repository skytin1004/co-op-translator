## Security

Microsoft നമ്മുടെ സോഫ്റ്റ്വേര്‍ ഉല്‍പ്പന്നങ്ങളുടെയും സേവനങ്ങളുടെയും സുരക്ഷയെ ഗൗരവത്തോടെ എടുത്തു കാണുന്നു, ഇതില്‍ ഞങ്ങളുടെ GitHub സ്ഥാപനങ്ങളിലൂടെ മാനേജു ചെയ്യുന്ന എല്ലാ സോഴ്‌സ് കോഡ് സംഭരണക들도 ഉള്‍പ്പെടുന്നു, അവieന്‍ शामिल [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) and [Xamarin](https://github.com/xamarin).

നിങ്ങള്‍ക്ക് ഏതെങ്കിലും Microsoft-അധികൃത സംഭരണകത്തില്‍ [Microsoft-ലെ സുരക്ഷാ ദുർബലതയുടെ വ്യാഖ്യാനം](https://aka.ms/security.md/definition)满足 ചെയുന്ന സുരക്ഷാ ദുർബലത കണ്ടെത്തിയിരിക്കുമെന്ന് കരുതുന്നു എങ്കില്‍, താഴെപ്പറഞ്ഞിരിക്കുന്ന വിധത്തില്‍ ഞങ്ങളോട് ദയവായി റിപ്പോര്‍ട്ട് ചെയ്യുക.

## സുരക്ഷാ പ്രശ്നങ്ങള്‍ റിപ്പോര്‍ട്ട് ചെയ്യല്‍

**സുരക്ഷാ ദുർബലതകള്‍ പൊതു GitHub പ്രശ്നങ്ങളിലൂടെ റിപ്പോര്‍ട്ട് ചെയ്യരുത്.**

മരാമത്ത് തുടരാനായി, ദയവായി അവ Microsoft Security Response Center (MSRC)-ല്‍ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)ല്‍ റിപ്പോര്‍ട്ട് ചെയ്യുക.

ലോഗിന്‍ ചെയ്യാതെ സമര്‍പ്പിക്കാന്‍ നിങ്ങൾ ആഗ്രഹിക്കുന്നുവെങ്കില്‍, [secure@microsoft.com](mailto:secure@microsoft.com) എന്ന ഇമെയിലിലേക്ക് അയയ്‌ക്കുക. സാധ്യമെങ്കിൽ, ഞങ്ങളുടെ PGP കീ ഉപയോഗിച്ച് നിങ്ങളുടെ സന്ദേശം കാൺഷ്യായ്ക്കുക; ഇത് [Microsoft Security Response Center PGP Key പേജ്](https://aka.ms/security.md/msrc/pgp)ല്‍ നിന്നു ഡൗണ്‍ലോഡ് ചെയ്യുക.

നിങ്ങള്‍ക്ക് 24 മണിക്കൂറിനകം പ്രതികരണം ലഭിക്കേണ്ടതാണ്. എന്തെങ്കിലും കാരണം കൊണ്ട് ലഭിക്കാത്ത പക്ഷം, നിങ്ങളുടെ ആദ്യ സന്ദേശം ഞങ്ങള്‍ സ്വീകരിച്ചിട്ടുണ്ടെന്ന് ഉറപ്പാക്കാന്‍ ഇമെയിലിലൂടെ ഫോളോ അപ് ചെയ്യുക. കൂടുതല്‍ വിവരങ്ങള്‍ക്ക് [microsoft.com/msrc](https://www.microsoft.com/msrc) സന്ദര്‍ശിക്കുക.

ദയവായി താഴെ പറഞ്ഞിരിക്കുന്ന ആവശ്യമായ വിവരങ്ങള്‍ (നിങ്ങള്‍ നല്‍കാന്‍ കഴിയുന്നത്ര) ഉള്‍പ്പെടുത്തുക, ഇത് പ്രശ്നത്തിന്റെ സ്വഭാവവും പരിധിയും ഞങ്ങള്‍ക്ക് മനസ്സിലാക്കാന്‍ സഹായിക്കും:

  * പ്രശ്നത്തിന്റെ തരം (ഉദാ: ബഫര്‍ ഓവര്‍ഫ്ലോ, SQL ഇഞ്ചക്ഷന്‍, ക്രോസ്-സൈറ്റ് സ്ക്രിപ്റ്റിംഗ്, തുടങ്ങിയവ)
  * പ്രശ്നം പ്രകടിപ്പിക്കുന്ന സോഴ്‌സ് ഫയല്‍(ക)ളുടെ പൂർണ്ണ പാതകൾ
  * ബാധിക്കപ്പെട്ട സോഴ്‌സ് കോഡിന്റെ സ്ഥലം (ടാഗ്/ബ്രാഞ്ച്/കമ്മിറ്റ് അല്ലെങ്കിൽ നേരിട്ടുള്ള URL)
  * പ്രശ്നം പുനരാവൃതമാക്കാന്‍ ആവശ്യമായ പ്രത്യേക ക്രമീകരണം
  * പ്രശ്നം പുനരാവൃതമാക്കാനുള്ള ഘട്ടംഘട്ടമായ നിര്‍ദ്ദേശങ്ങള്‍
  * തെളിവ് ആശയം അല്ലെങ്കിൽ എക്സ്പ്ലോയിറ്റ് കോഡ് (സാധ്യമായുവെങ്കില്‍)
  * പ്രശ്നത്തിന്റെ പ്രഭാവം, ആക്രമകന്‍ എങ്ങനെ ഈ പ്രശ്നം ഉപയോഗിക്കാവുന്നതാണെന്ന് ഉള്‍പ്പെടെ

ഈ വിവരങ്ങള്‍ ഞങ്ങളുടെ റിപ്പോര്ട്ട് ത്രയാജ് ചെയ്യുന്നതില്‍ വേഗത നല്‍കും.

ബഗ് ബൗണ്ടിക്കായി റിപ്പോര്ട്ട് ചെയ്യുന്നുവെങ്കിൽ, കൂടുതല്‍ സമ്പൂര്‍ണമായ റിപ്പോര്ട്ടുകള്‍ ഉയർന്ന ബൗണ്ടി പുരസ്കാരം നേടാന്‍ സഹായിക്കും. ഞങ്ങളുടെ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) പേജ് സന്ദര്‍ശിച്ച് ഞങ്ങളുടെ സജീവ പ്രോഗ്രാമുകള്‍ സംബന്ധിച്ച കൂടുതല്‍ വിവരങ്ങള്‍ കണ്ടെത്തുക.

## ഇഷ്ടമുള്ള ഭാഷകള്‍

ഞങ്ങള്‍ എല്ലാ ആശയവിനിമയങ്ങളും ഇംഗ്ലീഷിലാണ് പ്രിയപ്പെടുന്നത്.

## നയം

Microsoft [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) എന്ന സിദ്ധാന്തം പാലിക്കുന്നു.
