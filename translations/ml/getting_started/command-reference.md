# Command reference

The **Co-op Translator** CLI ശബ്ദാനുബന്ധമായ വിവർത്തന പ്രക്രിയയെ ഇഷ്ടാനുസൃതമാക്കാൻ നിരവധി ഓപ്ഷനുകൾ നൽകുന്നു:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | നിങ്ങളുടെ പ്രോജക്ട് വ്യക്തമാക്കിയ ഭാഷകളിലേക്ക് പരിഭാഷപ്പെടുത്തുന്നു. ഉദാഹരണം: translate -l "es fr de" സ്പാനിഷ്, ഫ്രഞ്ച്, ജർമ്മൻ എന്നിവിലേക്ക് വിവർത്തനം ചെയ്യുന്നു. എല്ലാം പിന്തുണയ്ക്കുന്ന ഭാഷകളിലേക്ക് വിവർത്തനം ചെയ്യാൻ translate -l "all" ഉപയോഗിക്കുക.
translate -l "language_codes" -u              | നിലവിലുള്ള വിവർത്തനങ്ങൾ ഇല്ലാതാക്കി, പുനഃസൃഷ്ടിച്ച് വിവർത്തനങ്ങൾ പുതുക്കുന്നു. മുന്നറിയിപ്പ്: ഇത് വ്യക്തമാക്കിയ ഭാഷകളിലെ എല്ലാ നിലവിലെ വിവർത്തനങ്ങളും ഇല്ലാതാക്കും.
translate -l "language_codes" -img            | ചിത്രം ഫയലുകൾ മാത്രമേ വിവർത്തനം ചെയ്യൂ.
translate -l "language_codes" -md             | Markdown ഫയലുകൾ മാത്രമേ വിവർത്തനം ചെയ്യൂ.
translate -l "language_codes" -nb             | Jupyter നോട്ട്‌ബുക്ക് ഫയലുകൾ (.ipynb) മാത്രമേ വിവർത്തനം ചെയ്യൂ.
translate -l "language_codes" --fix           | മുൻ മുൻകൂട്ടി വിലയിരുത്തലിന്റെ അടിസ്ഥാനത്തിൽ താഴ്ന്ന വിശ്വാസ സ്കോറുകൾ ഉള്ള ഫയലുകൾ പുനഃവിവർത്തനം ചെയ്യുന്നു.
translate -l "language_codes" -d              | വിശദമായ ലോഗിംഗ് സാധ്യമാക്കുന്ന ഡീബഗ് മോഡ് സജ്ജമാക്കുന്നു.
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ അകത്ത് DEBUG-നില തരം ലോഗുകൾ ഫയലുകളായി സംരക്ഷിക്കുക (കോൺസോൾ -d-നിബന്ധനയിൽ നിയന്ത്രിതമാണ്)
translate -l "language_codes" -r "root_dir"   | പ്രോജക്റ്റിന്റെ റൂട്ട് ഡയറക്ടറി നിർദ്ദേശിക്കുന്നു
translate -l "language_codes" -f              | ചിത്രം വിവർത്തനത്തിന് ഫാസ്റ്റ് മോഡ് ഉപയോഗിക്കുക (ഗുണമേന്മയിലും സംരഭണത്തിലും ചെറിയ ബാധ്യതയിൽ 3x വേഗത്തിലായത്).
translate -l "language_codes" -y              | എല്ലാ പ്രോത്സാഹനങ്ങളും സ്വയമേവ സമ്മതിക്കുക (CI/CD പൈപ്പ്‌ലൈൻസിന് ഉപകരിക്കുന്നു)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | വിവർത്തനം ചെയ്ത Markdown-ലും നോട്ട്‌ബുക്കുകളിലും മെഷീൻ വിവർത്തന സമ്മതികരണ വകുപ്പ് ചേർക്കൽ ഓൺ അഥവാ ഓഫ് ചെയ്യുക (പൂർവനിരീക്ഷണം: സജ്ജമാക്കിയിട്ടുണ്ട്).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | നിങ്ങളുടെ റെപ്പോസിറ്ററി URL ഉപയോഗിച്ച് README ഭാഷാനിർദ്ദേശ വിഭാഗം വ്യക്തിഗതമാക്കുക (സ്പാർസ് ചെട്ടൗട്ട്).
translate -l "language_codes" --help          | CLI-ൽ ലഭ്യമായ കമാൻഡുകൾ കാണിക്കുന്ന സഹായ വിശദാംശങ്ങൾ
evaluate -l "language_code"                  | പ്രത്യേക ഭാഷയുടെ വിവർത്തന ഗുണമെനിട്ട് വിലയിരുത്തുന്നു, വിശ്വാസ സ്കോറുകൾ നൽകുന്നു
evaluate -l "language_code" -c 0.8           | ആക്റ്റിവേറ്റു ചെയ്ത വിശ്വാസ പരിധിയോടെ വിവർത്തനങ്ങൾ വിലയിരുത്തുന്നു
evaluate -l "language_code" -f               | വേഗതയുള്ള വിലയിരുത്തൽ മോഡ് (നിബന്ധനാ അടിസ്ഥാനമുളളത് മാത്രം, LLM ഇല്ലാത്തത്)
evaluate -l "language_code" -D               | ആഴത്തിലുള്ള വിലയിരുത്തൽ മോഡ് (LLM അടിസ്ഥാനമുളളത് മാത്രം, കൂടുതൽ വിശദമായതും മന്ദഗതിയിലുമുള്ളത്)
evaluate -l "language_code" --save-logs, -s  | <root_dir>/logs/ അകത്ത് DEBUG-നില തരം ലോഗുകൾ ഫയലുകളായി സംരക്ഷിക്കുക
migrate-links -l "language_codes"             | വിവർത്തനം ചെയ്ത Markdown ഫയലുകൾ പുനഃസംസ്‌കരിച്ച് നോട്ട്‌ബുക്ക് (.ipynb) ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യും. ലഭ്യമായപ്പോൾ വിവർത്തിച്ച നോട്ട്‌ബുക്കുകൾ മുൻഗണന; ഇല്ലെങ്കിൽ യഥاورീതിയിലുള്ള നോട്ട്‌ബുക്കുകൾക്ക് മടക്കി പോകാം.
migrate-links -l "language_codes" -r          | പ്രോജക്റ്റ് റൂട്ട് ഡയറക്ടറി വ്യക്തമാക്കുക (പൂർവനിർദ്ദേശം: ഇപ്പോഴത്തെ ഡയറക്ടറി).
migrate-links -l "language_codes" --dry-run   | മാറ്റങ്ങൾ രേഖപ്പെടുത്താതെ ഏത് ഫയലുകൾ മാറുമെന്നു കാണിക്കുക.
migrate-links -l "language_codes" --no-fallback-to-original | വിവർത്തിച്ചത് ഇല്ലാത്തപ്പോൾ യഥാർത്ഥ നോട്ട്‌ബുക്കുകൾക്ക് മടക്കി പോകാതെ പുതുക്കൽ നടത്തുകയില്ല (വിവർത്തിച്ചവ ഉണ്ടാകുമ്പോൾ മാത്രം അപ്ഡേറ്റ് ചെയ്യും).
migrate-links -l "language_codes" -d          | വിശദമായ ലോഗിംഗിന് ഡീബഗ് മോഡ് സജ്ജമാക്കുക.
migrate-links -l "language_codes" --save-logs, -s | <root_dir>/logs/ അകത്ത് DEBUG-നില ലോഗുകൾ ഫയലുകളായി സംരക്ഷിക്കുക
migrate-links -l "all" -y                      | എല്ലാ ഭാഷകളും പ്രോസസ് ചെയ്യുക, മുന്നറിയിപ്പ് പ്രൊംപ്റ്റ് സ്വയമേവ അംഗീകരിക്കുക.

## Usage examples

  1. ပူဇော်ဘာത്തനം (ഇതിനിടയിൽ നിലവിലുള്ള വിവർത്തനങ്ങൾ ഇല്ലാതാക്കാതെ പുതിയ വിവർത്തനങ്ങൾ ചേർക്കുക):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. നിലവിലുള്ള വിവർത്തനങ്ങൾ ഇല്ലാതാക്കാതെ പുതിയ കൊറിയൻ ചിത്രം വിവർത്തനങ്ങൾ മാത്രം ചേർക്കുക:    translate -l "ko" -img

  3. എല്ലാ കൊറിയൻ വിവർത്തനങ്ങളും പുതുക്കുക (മുന്നറിയിപ്പ്: പുനഃവിവർത്തനം നടത്താൻ മുമ്പ് നിലവിലുള്ള എല്ലാ കൊറിയൻ വിവർത്തനങ്ങളും ഇല്ലാതാക്കും):    translate -l "ko" -u

  4. കൊറിയൻ ചിത്രങ്ങൾ മാത്രം പുതുക്കുക (മുന്നറിയിപ്പ്: പുനഃവിവർത്തനം ചെയ്യുന്നതിന് മുമ്പ് എല്ലാ കൊറിയൻ ചിത്രങ്ങളും ഇല്ലാതാക്കും):    translate -l "ko" -img -u

  5. മറ്റു വിവർത്തനങ്ങൾ സ്വാധീനിക്കാതെ കൊറിയൻ മാർക്ക്ഡൗൺ വിവർത്തനങ്ങൾ ചേർക്കുക:    translate -l "ko" -md

  6. മുൻ‌കൂട്ടി വിലയിരുത്തലിന്റെ അടിസ്ഥാനത്തിൽ താഴ്ന്ന വിശ്വാസം ഉള്ള വിവർത്തനങ്ങൾ ശരിയാക്കുക: translate -l "ko" --fix

  7. നിർദ്ദിഷ്ട ഫയലുകൾക്കുള്ള താഴ്ന്ന വിശ്വാസം ഉള്ള വിവർത്തനങ്ങൾ മാത്രമേ ശരിയാക്കൂ (മാർക്ക്ഡൗൺ): translate -l "ko" --fix -md

  8. നിർദ്ദിഷ്ട ഫയലുകൾക്കുള്ള താഴ്ന്ന വിശ്വാസം ഉള്ള വിവർത്തനങ്ങൾ മാത്രമേ ശരിയാക്കൂ (ചിത്രങ്ങൾ): translate -l "ko" --fix -img

  9. ചിത്രം വിവർത്തനത്തിന് ഫാസ്റ്റ് മോഡ് ഉപയോഗിക്കുക:    translate -l "ko" -img -f

  10. ഇഷ്ടാനുസൃത പരിധിയോടെ താഴ്ന്ന വിശ്വാസം ഉള്ള വിവർത്തനങ്ങൾ ശരിയാക്കുക: translate -l "ko" --fix -c 0.8

  11. ഡീബഗ് മോഡ് ഉദാഹരണം: - translate -l "ko" -d: ഡീബഗ് ലോഗിംഗ് സജ്ജമാക്കുക.
  12. ലോഗുകൾ ഫയലുകളിൽ സംരക്ഷിക്കുക: translate -l "ko" -s
  13. കോൺസോൾ DEBUG-യും ഫയൽ DEBUG-ഉം: translate -l "ko" -d -s
  14. ഔട്ട്പുട്ടിലേക്ക് മെഷീൻ വിവർത്തന സമ്മതമില്ലാ രേഖ ചേർക്കാതെയാക്കുക: translate -l "ko" --no-disclaimer

  15. കൊറിയൻ വിവർത്തനങ്ങൾക്കായുള്ള നോട്ട്‌ബുക്ക് ലിങ്കുകൾ മാറ്റിസ്ഥാപിക്കുക (ലഭ്യമായപ്പോൾ വിവർത്തിച്ച നോട്ട്‌ബുക്കുകളിലേക്കുള്ള ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യുക):    migrate-links -l "ko"

  15. ഡ്രൈ-റൺ ഉപയോഗിച്ച് ലിങ്കുകൾ മാറ്റുക (ഫയൽ എഴുതേണ്ടതില്ല):    migrate-links -l "ko" --dry-run

  16. വിവർത്തിച്ച നോട്ട്‌ബുക്കുകൾ ഉണ്ടെങ്കിൽ മാത്രമേ ലിങ്കുകൾ പുതുക്കൂ (യഥാർത്ഥത്തിലേക്ക് മടക്കാതിരിക്കുക):    migrate-links -l "ko" --no-fallback-to-original

  17. എല്ലാ ഭാഷകളും പ്രോസസ് ചെയ്യുക, സ്ഥിരീകരണ പ്രൊംപ്റ്റ് കാണിക്കുക:    migrate-links -l "all"

  18. എല്ലാ ഭാഷകളും പ്രോസസ് ചെയ്ത് സ്വയമെന്നെ സ്ഥിരീകരിക്കുക:    migrate-links -l "all" -y
  19. migrate-links ലെ ലോഗുകൾ ഫയലായി സംരക്ഷിക്കുക:    migrate-links -l "ko ja" -s

  20. README ഭാഷകൾ സംബന്ധിച്ച നിർദ്ദേശം നിങ്ങളുടെ റെപ്പോ URL-നൊപ്പം വ്യക്തിഗതമാക്കുക:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Evaluation Examples

> [!WARNING]  
> **ബീറ്റ ഫീച്ചർ**: വിലയിരുത്തൽ ഫംഗ്ഷണാലിറ്റി ഇപ്പോൾ ബീറ്റ ഘട്ടത്തിലാണ്. ഈ ഫീച്ചർ വിവർത്തനം ചെയ്ത ഡോക്യുമെന്റുകൾ അവലോകനം ചെയ്യാനായി പുറത്തിറക്കിയതാണ്, വിലയിരുത്തൽ മാർഗങ്ങൾയും വിശദമായ നടപ്പാക്കലും ഇപ്പോഴും വികസന ഘട്ടത്തിലാണ്, മാറ്റത്തിന് വിധേയമാണ്.

  1. കൊറിയൻ വിവർത്തനങ്ങൾ വിലയിരുത്തുക: evaluate -l "ko"

  2. ഇഷ്ടാനുസൃത വിശ്വാസ പരിധിയോടെ വിലയിരുത്തുക: evaluate -l "ko" -c 0.8

  3. വേഗതയുള്ള വിലയിരുത്തൽ (നിബന്ധനാ അടിസ്ഥാനമുളളത് മാത്രം): evaluate -l "ko" -f

  4. ആഴത്തിലുള്ള വിലയിരുത്തൽ (LLM അടിസ്ഥാനമുളളത് മാത്രം): evaluate -l "ko" -D
