# Command reference

**Co-op Translator** CLI വിവർത്തന പ്രക്രിയ മറ്റ് ഇഷ്ടാനുസൃതമാക്കുന്നതിനായി പല ഓപ്ഷനുകളും നൽകുന്നു:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | നിർദ്ദിഷ്ട ഭാഷകളിലേക്ക് നിങ്ങളുടെ പ്രോജക്റ്റ് വിവർത്തനം ചെയ്യുന്നു. ഉദാഹരണം: translate -l "es fr de" സ്പാനിഷ്, ഫ്രഞ്ച്, ജർമൻ ഭാഷകളിലേക്ക് വിവർത്തനം ചെയ്യുന്നു. എല്ലാവിധ പിന്തുണയുള്ള ഭാഷകളിലേക്ക് വിവർത്തനം ചെയ്യാൻ translate -l "all" ഉപയോഗിക്കുക.
translate -l "language_codes" -u              | നിലവിലുള്ള വിവർത്തനങ്ങൾ നീക്കം ചെയ്ത് പുനഃസൃഷ്ടിച്ച് വിവർത്തനങ്ങൾ അപ്ഡേറ്റ് ചെയ്യുന്നു. മുന്നറിയിപ്പ്: ഇത് നൽകിയ ഭാഷകളിലെ എല്ലാ നിലവിലെ വിവർത്തനങ്ങളും മായ്ക്കും.
translate -l "language_codes" -img            | ചിത്ര ഫയലുകൾ മാത്രമാണ് വിവർത്തനം ചെയ്യുന്നത്.
translate -l "language_codes" -md             | മാർക്ക്ഡൗൺ ഫയലുകൾ മാത്രമാണ് വിവർത്തനം ചെയ്യുന്നത്.
translate -l "language_codes" -nb             | Jupyter നോട്ട്ബുക്ക് (.ipynb) ഫയലുകൾ മാത്രമാണ് വിവർത്തനം ചെയ്യുന്നത്.
translate -l "language_codes" --fix           | മുൻപരിശോധന ഫലങ്ങളെ അടിസ്ഥാനമാക്കി കുറഞ്ഞ ആത്മവിശ്വാസ സ്കോറുള്ള ഫയലുകൾ പുനഃവിവർത്തനം ചെയ്യുന്നു.
translate -l "language_codes" -d              | വിശദമായ ലോഗിംഗിനായി ഡീബഗ് മോഡ് സജ്ജമാക്കുന്നു.
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ എന്നതിങ്കൽ DEBUG-നിലത്തിലുള്ള ലോഗുകൾ ഫയലുകളിൽ സംരക്ഷിക്കുന്നു (കൺസോൾ -d നാൽ നിയന്ത്രിതമാണ്)
translate -l "language_codes" -r "root_dir"   | പ്രോജക്റ്റിന്റെ റൂട്ട് ഡയറക്ടറി വ്യക്തമാക്കുന്നു
translate -l "language_codes" -f              | ഇൻമേജ് വിവർത്തനത്തിന് ഫാസ്റ്റ് മോഡിന്റെ ഉപയോഗം (ഗുണമേന്മക്കും പൊരുത്തത്തിനും ചെറിയ ബാധവുമുണ്ട്, 3 വരം വരെ വേഗത്തിൽ പ്ലോട്ടിങ്).
translate -l "language_codes" -y              | എല്ലാ പ്രൊംപ്റ്റുകളും സ്വയമേവ സ്ഥിരീകരിക്കുന്നു (CI/CD പൈപ്പ്‌ലൈൻസിനായുള്ള ഉപരി).
translate -l "language_codes" --add-disclaimer/--no-disclaimer | വിവർത്തന മാർക്ക്ഡൗൺ, നോട്ട്ബുക്കുകളിൽ യന്ത്ര വിവർത്തന ഡിസ്ക്ലെയിമർ സെക്ഷൻ ചേർക്കൽ സജ്ജമാക്കുന്നതോ സ്‌നാനം ചെയ്യുന്നതോ (ഇനീഷ്യൽ: സജ്ജീകരിച്ചിരിക്കുന്നു).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | നിങ്ങളുടെ റിപൊസിറ്ററി URL ഉപയോഗിച്ച് README ഭാഷാ സെക്ഷന്‍ കസ്റ്റമൈസ് ചെയ്യുന്നു (സ്പാർസ് ചെക്ക്ഔട്ട്).
translate -l "language_codes" --help          | CLI-യിൽ ലഭ്യമായ കമാൻഡുകൾ കാണിക്കുന്ന സഹായ വിശദാംശങ്ങൾ
evaluate -l "language_code"                  | പ്രത്യേക ഭാഷയ്ക്കായുള്ള വിവർത്തന ഗുണനിലവാരം വിലയിരുത്തുകയും ആത്മവിശ്വാസ സ്കോറുകൾ നൽകുകയും ചെയ്യുന്നു
evaluate -l "language_code" -c 0.8           | ഇഷ്‌ടാനുസൃത ആത്മവിശ്വാസ പരിധിയോടെ വിവർത്തനങ്ങൾ വിലയിരുത്തുന്നു
evaluate -l "language_code" -f               | വേഗത്തിലുള്ള വിലയിരുത്തൽ മോഡ് (നിയമಮಾನ്യമായത് മാത്രം, LLM ഇല്ലാതെ)
evaluate -l "language_code" -D               | ആഴത്തിലുള്ള വിലയിരുത്തൽ മോഡ് (LLM അടിസ്ഥാനവും കൂടുതൽ തുലനാത്മകവും, എന്നാൽ മന്ദഗതിയുള്ളതും)
evaluate -l "language_code" --save-logs, -s  | <root_dir>/logs/ എന്നതിങ്കൽ DEBUG-നിലയിലെ ലോഗുകൾ ഫയലുകളിൽ സംരക്ഷിക്കുന്നു
migrate-links -l "language_codes"             | വിവർത്തനമാക്കിയ മാർക്ക്ഡൗൺ ഫയലുകൾ പുനഃസംസ്‌ക്കരിച്ച് നോട്ട്ബുക്ക് (.ipynb) ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യുന്നു. ലഭ്യമെങ്കിൽ വിവർത്തനക്കഴിഞ്ഞ നോട്ട്ബുക്കുകളെ മുൻഗണന നൽകുന്നു; അല്ലെങ്കിൽ യഥാർത്ഥ നോട്ട്ബുക്കിലേക്ക് വീണ്ടെടുക്കാനും കഴിയും.
migrate-links -l "language_codes" -r          | പ്രോജക്റ്റിന്റെ റൂട്ട് ഡയറക്ടറി വ്യക്തമാക്കുക (ഡീഫോൾട്ട്: നിലവിലെ ഡയറക്ടറി).
migrate-links -l "language_codes" --dry-run   | മാറ്റങ്ങൾ എഴുതാതെയാണ് എവിടെ өзг બદલാപ്പ് കാണിക്കുക.
migrate-links -l "language_codes" --no-fallback-to-original | വിവർത്തനം ചെയ്ത നോട്ട്ബുക്കുകൾ ലഭ്യമല്ലാതെ വന്നാൽ യഥാർത്ഥ നോട്ട്ബുക്കുകളിലേക്ക് ലിങ്കുകൾ പുനഃരചയിക്കുക വേണ്ട (മാത്രം വിവർത്തനമുള്ളപ്പോൾ അപ്ഡേറ്റ് ചെയ്യുക).
migrate-links -l "language_codes" -d          | വിശദമായ ലോഗിംഗിനായി ഡീബഗ് മോഡ് സജ്ജമാക്കുക.
migrate-links -l "language_codes" --save-logs, -s | <root_dir>/logs/ എന്നതിങ്കൽ DEBUG-നിലയിലെ ലോഗുകൾ ഫയലുകളിൽ സംരക്ഷിക്കുന്നു
migrate-links -l "all" -y                      | എല്ലാ ഭാഷകളും പ്രോസസ് ചെയ്തു മുന്നറിയിപ്പ് പ്രൊംപ്റ്റ് സ്വയമേവ സ്ഥിരീകരിക്കുന്നു.

## ഉപയോഗത്തിന്റെ ഉദാഹരണങ്ങൾ

  1. ഡീഫോൾട്ട് പെരുമാറ്റം (നിലവിലുള്ള വിവർത്തനങ്ങൾ നീക്കം ചെയ്യാതെ പുതിയ വിവർത്തനങ്ങൾ ചേർക്കുക):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. പുതിയ കൊറിയൻ ചിത്രം വിവർത്തനങ്ങൾ മാത്രം ചേർക്കുക (നിലവിലുള്ള വിവർത്തനങ്ങൾ ഇല്ല):    translate -l "ko" -img

  3. എല്ലാ കൊറിയൻ വിവർത്തനങ്ങളും അപ്ഡേറ്റ് ചെയ്യുക (മുൻകൂറ്: എല്ലാ നിലവിലുള്ള കൊറിയൻ വിവർത്തനങ്ങളും നശിപ്പിച്ച് പുനഃവിവർത്തനം ചെയ്യുന്നു):    translate -l "ko" -u

  4. sadece കൊറിയൻ ചിത്രങ്ങൾ അപ്ഡേറ്റ് ചെയ്യുക (മുൻകൂറ്: നിലവിളിക്കുന്ന കൊറിയൻ ചിത്രങ്ങൾ മായ്ക്കുന്നു):    translate -l "ko" -img -u

  5. മറ്റുള്ള വിവർത്തനങ്ങളെ ബാധിക്കാതെ കൊറിയൻ മാർക്ക്ഡൗൺ വിവർത്തനങ്ങൾ ചേർക്കുക:    translate -l "ko" -md

  6. മുൻപരിശോധന ഫലങ്ങളനുസരിച്ച് കുറഞ്ഞ ആത്മവിശ്വാസമുള്ള വിവർത്തനങ്ങൾ ശരിയാക്കുക: translate -l "ko" --fix

  7. പ്രത്യേക ഫയലുകൾക്കായാണ് കുറഞ്ഞ ആത്മവിശ്വാസ വിവർത്തനങ്ങൾ ശരിയാക്കുന്നത് (markdown): translate -l "ko" --fix -md

  8. പ്രത്യേക ഫയലുകൾക്കായാണ് കുറഞ്ഞ ആത്മവിശ്വാസ വിവർത്തനങ്ങൾ ശരിയാക്കുന്നത് (ചിത്രങ്ങൾ): translate -l "ko" --fix -img

  9. ഇൻമേജ് വിവർത്തനത്തിന് ഫാസ്റ്റ് മോഡ് ഉപയോഗിക്കുക:    translate -l "ko" -img -f

  10. ഇഷ്ടാനുസൃത പരിധിയോടെ കുറഞ്ഞ ആത്മവിശ്വാസം ശരിയാക്കുക: translate -l "ko" --fix -c 0.8

  11. ഡീബഗ് മോഡിന്റെ ഉദാഹരണം: - translate -l "ko" -d: ഡീബഗ് ലോഗിംഗ് അവസരം നൽകുന്നു.
  12. ലോഗുകൾ ഫയലുകളിൽ സംരക്ഷിക്കുക: translate -l "ko" -s
  13. കൺസോൾ DEBUGയും ഫയൽ DEBUGയും: translate -l "ko" -d -s
  14. റെസൽട്ടുകളിൽ മെഷീൻ വിവർത്തനം ഡിസ്ക്ലെയിമർ ചേർക്കാതെ വിവർത്തനം ചെയ്യുക: translate -l "ko" --no-disclaimer

  15. കൊറിയൻ വിവർത്തനങ്ങളുടെ നോട്ട്ബുക്ക് ലിങ്കുകൾ മാറ്റിവെക്കുക (ലഭ്യമെങ്കിൽ വിവർത്തനം ചെയ്ത നോട്ട്ബുക്കുകൾക്ക് ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യുക):    migrate-links -l "ko"

  15. ഡ്രൈ-റൺ ഉപയോഗിച്ചുള്ള മൈഗ്രേഷൻ (ഫയൽ എഴുത്തില്ല):    migrate-links -l "ko" --dry-run

  16. വിവർത്തനം ചെയ്ത നോട്ട്ബുക്കുകൾ ലഭ്യമുള്ളപ്പോഴേക്കും മാത്രമേ ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യൂ (മൂലേക്കു തിരിച്ചുള്ള വീണ്ടെടുക്കൽ വേണ്ട):    migrate-links -l "ko" --no-fallback-to-original

  17. എല്ലാ ഭാഷകളും സ്ഥിരമാക്കിയ യുവ അഴിക്കങ്ങൽ പ്രൊംപ്റ്റുമായി പ്രോസസ് ചെയ്യുക:    migrate-links -l "all"

  18. എല്ലാ ഭാഷകളും പ്രോസസ് ചെയ്ത് സ്വയമേവ സ്ഥിരീകരിക്കുക:    migrate-links -l "all" -y
  19. migrate-links നുള്ള ലോഗുകൾ ഫയലുകളിൽ സേവ് ചെയ്യുക:    migrate-links -l "ko ja" -s

  20. README ഭാഷാ ഉപദേശത്തിനു നിങ്ങളുടെ റിപൊസ URL ഉപയോഗിച്ച് വ്യക്തിഗതമാക്കുക:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### വിലയിരുത്തല്‍ ഉദാഹരണങ്ങള്‍  

> [!WARNING]  
> **Beta Feature**: വിലയിരുത്തൽ സവിശേഷത ഇപ്പോൾ ബീറ്റയിൽ ആണ്. വിവർത്തനം ചെയ്ത ഡോക്യുമെന്റുകൾ വിലയിരുത്താൻ ഈ സവിശേഷത പുറത്തിറങ്ങിയതാണ്, വിലയിരുത്തല്‍ രീതി, വിശദമായ നടപ്പാക്കല്‍ തുടങ്ങിയവ ഇതുവരെ വികസനത്തിലുണ്ട് കൂടാതെ മാറ്റത്തിന് വിധേയവുമാണ്.

  1. കൊറിയൻ വിവർത്തനങ്ങൾ വിലയിരുത്തുക: evaluate -l "ko"

  2. ഇഷ്‌ടാനുസൃത ആത്മവിശ്വാസ പരിധിയോടെ വിലയിരുത്തുക: evaluate -l "ko" -c 0.8

  3. വേഗത്തിലുള്ള വിലയിരുത്തൽ (നിയമമാന്യമായത് മാത്രം): evaluate -l "ko" -f
