# Microsoft Co-op Translator Troubleshooting Guide


## ចំណាំទូទៅ
Microsoft Co-Op Translator គឺជាឧបករណ៍ដ៏មានអំណាចសម្រាប់ការបកប្រែឯកសារ Markdown បានយ៉ាងរលូន។ មគ្គុទេសក៍នេះនឹងជួយអ្នកដោះស្រាយបញ្ហាទូទៅដែលត្រូវបានជួបប្រទៈពេលប្រើកម្មវិធីនេះ។

## បញ្ហាទូទៅ និងដំណោះស្រាយ

### 1. បញ្ហាទាក់ទងនឹង Tag Markdown
**បញ្ហា៖** ឯកសារ Markdown ដែលបានបកប្រែមាន tag `markdown` នៅផ្នែកលើ បណ្តាលឲ្យមានបញ្ហាក្នុងការបង្ហាញ។

**ដំណោះស្រាយ៖** ដើម្បីដោះស្រាយបញ្ហានេះ សូមលុប tag `markdown` នៅជំពូកឯកសារនោះ។ វានឹងអនុញ្ញាតឱ្យឯកសារ Markdown បង្ហាញបានត្រឹមត្រូវ។

**ជំហាន៖**
1. បើកឯកសារ Markdown (`.md`) ដែលបានបកប្រែ។
2. រកទីតាំង tag `markdown` នៅផ្នែកលើឯកសារ។
3. លុប tag `markdown` នោះ។
4. រក្សាទុកការផ្លាស់ប្តូរទៅឯកសារ។
5. បើកឯកសារឡើងវិញដើម្បីធ្វើអោយប្រាកដថាវាបង្ហាញបានត្រឹមត្រូវ។

### 2. បញ្ហា URL រូបភាពបញ្ចូល
**បញ្ហា៖** URL របស់រូបភាពដែលបញ្ចូលមិនត្រូវទៅតាមភាសា ដែលបណ្តាលឲ្យមានរូបភាពមិនត្រឹមត្រូវ ឬខ្វះកន្លែងបង្ហាញរូបភាព។

**ដំណោះស្រាយ៖** សូមពិនិត្យ URL របស់រូបភាពបញ្ចូល និងធ្វើអោយប្រាកដថាត្រូវនឹងភាសានោះ។ រូបភាពទាំងអស់មាននៅក្នុងថត `translated_images` ហើយរាល់រូបភាពមាន tag ភាសានៅក្នុងឈ្មោះឯកសាររូបភាព។

**ជំហាន៖**
1. បើកឯកសារ Markdown ដែលបានបកប្រែ។
2. ស្វែងរករូបភាពបញ្ចូល និង URL របស់ពួកវា។
3. ធ្វើអោយប្រាកដថា ភាសានៅក្នុងឈ្មោះឯកសាររបស់រូបភាពត្រូវនឹងភាសានៃឯកសារ។
4. បច្ចុប្បន្នភាព URL ប្រសិនបើចាំបាច់។
5. រក្សាទុកការផ្លាស់ប្តូរ ហើយបើកឯកសារឡើងវិញ ដើម្បីបញ្ជាក់ថារូបភាពបង្ហាញបានត្រឹមត្រូវ។

### 3. ភាពត្រឹមត្រូវនៃការបកប្រែ
**បញ្ហា៖** មាតិកាដែលបានបកប្រែមានភាពមិនត្រឹមត្រូវ ឬត្រូវការកែសម្រួលបន្ថែម។

**ដំណោះស្រាយ៖** សូមពិនិត្យឯកសារដែលបានបកប្រែ និងធ្វើការកែច្នៃដែលចាំបាច់ ដើម្បីបង្កើនភាពត្រឹមត្រូវ និងអានបានល្អ។

**ជំហាន៖**
1. បើកឯកសារដែលបានបកប្រែ។
2. ពិនិត្យមាតិកាដោយការយកចិត្តទុកដាក់។
3. បកប្រែ និងកែសម្រួល ដើម្បីជួយឲ្យមានភាពត្រឹមត្រូវ។
4. រក្សាទុកការផ្លាស់ប្តូរ។

## 4. បញ្ហាអនុញ្ញាត Redacted ឬ 404

ប្រសិនបើរូបភាព ឬអត្ថបទមិនត្រូវបានបកប្រែទៅភាសាត្រឹមត្រូវ ហើយពេលប្រតិបត្តិការជាមួយភាពស៊ើបអង្កេត -d មានកំហុស 401 សូមស្គាល់ឲ្យដឹងថា នេះគឺជាការបរាជ័យផ្នែកផ្ទៀងផ្ទាត់បញ្ជាក់—អាចជាកូនសោមិនត្រឹមត្រូវ ប្រែប្រួលឬមិនភ្ជាប់ទៅតំបន់endpoint ។

ចូលរត់ co-op translator ជាមួយ [ផ្លូវផ្គាប់ -d debug](https://github.com/Azure/co-op-translator/blob/main/getting_started/command-reference.md) ដើម្បីទទួលបានការយល់ដឹងជារួមប្រភពបញ្ហា។

- **សារ​កំហុស**: `Access denied due to invalid subscription key or wrong API endpoint.`
- **ហេតុសម្បត្តិអាចមាន**៖
  - កូនសោសេវាកម្មត្រូវបានលុប ឬមិនត្រឹមត្រូវក្នុងការស្នើសុំ។
  - កូនសោសេវាកម្ម AI ឬកូនសោសេវាកម្មសុវត្ថិភាព អាចជាកូនសោរបស់ធនធាន Azure ផ្សេងទៀត (ដូចជា Translator ឬ OpenAI) ជំនួសវត្ថុធាតុ **Azure AI Vision**។

 **ប្រភេទធនធាន**
  - ទៅកាន់ [Azure Portal](https://portal.azure.com) ឬ [Azure AI Foundry](https://ai.azure.com) ហើយធ្វើអោយប្រាកដថា វត្តមានប្រភេទ `Azure AI services` → `Vision` ។
  - ប្រាកដថា API keys ត្រឹមត្រូវ និងកូនសោត្រូវបានប្រើ។

## 5. កំហុសកំណត់រចនាសម្ព័ន្ធ (ការដោះស្រាយកំហុសថ្មី)

ចាប់ពីប្រព័ន្ធបកប្រែជ្រើសរើសថ្មី Co-op Translator ឥលូវផ្ដល់សារ​កំហុសច្បាស់លាស់ ពេលដែលសេវាកម្មចាំបាច់មិនបានកំណត់រចនាសម្ព័ន្ធ។

### 5.1. សេវាកម្ម Azure AI មិនបានកំណត់រចនាសម្ព័ន្ធសម្រាប់ការបកប្រែរូបភាព

**បញ្ហា៖** អ្នកបានស្នើរសុំបកប្រែរូបភាព (`-img` flag) ប៉ុន្តែសេវាកម្ម Azure AI មិនបានកំណត់រចនាសម្ព័ន្ធត្រឹមត្រូវ។

**សារ​កំហុស:**
```
Error: Image translation requested but Azure AI Service is not configured.
Please add AZURE_AI_SERVICE_API_KEY and AZURE_AI_SERVICE_ENDPOINT to your .env file.
Check Azure AI Service availability and configuration.
```

**ដំណោះស្រាយ:**
1. **ជម្រើស 1**: កំណត់រចនាសម្ព័ន្ធសេវាកម្ម Azure AI
   - បន្ថែម `AZURE_AI_SERVICE_API_KEY` ទៅក្នុងឯកសារ `.env` របស់អ្នក
   - បន្ថែម `AZURE_AI_SERVICE_ENDPOINT` ទៅក្នុងឯកសារ `.env` របស់អ្នក
   - ពិនិត្យការចូលប្រើសេវាកម្ម

2. **ជម្រើស 2**: លុបសំណើសុំបកប្រែរូបភាព
   ```bash
   # ជំនួសដោយ: translate -l "ko" -img
   # ប្រើ: translate -l "ko" -md
   ```

### 5.2. កំណត់រចនាសម្ព័ន្ធចាំបាច់បាត់បង់

**បញ្ហា៖** ការកំណត់រចនាសម្ព័ន្ធ LLM មួយចំនួនខ្វះ។

**សារ​កំហុស:**
```
Error: No language model configuration found.
Please configure either Azure OpenAI or OpenAI in your .env file.
```

**ដំណោះស្រាយ:**
1. ពិនិត្យឯកសារ `.env` របស់អ្នក ដើម្បីធ្វើអោយប្រាកដថាមានយ៉ាងហោចណាស់ការកំណត់រចនាសម្ព័ន្ធ LLM មួយក្នុងចំណោម ៖
   - **Azure OpenAI**: `AZURE_OPENAI_API_KEY` និង `AZURE_OPENAI_ENDPOINT`
   - **OpenAI**: `OPENAI_API_KEY`
   
   អ្នកត្រូវការព័ត៌មានពី Azure OpenAI ឬ OpenAI មួយក្នុងចំនោមទាំងពីរ មិនមែនទាំងពីរដោយដូចគ្នា។

### 5.3. ការជ្រុះច្រឡំក្នុងការបកប្រែជ្រើសរើស

**បញ្ហា៖** មិនមានឯកសារណាមួយបានបកប្រែ ទោះបីបញ្ជាកម្មង់ជោគជ័យក៏ដោយ។

**ហេតុអាចមាន:**
- សញ្ញាទំនាក់ទំនងប្រភេទឯកសារ (`-md`, `-img`, `-nb`) មិនត្រឹមត្រូវ
- គ្មានឯកសារមានជំនួសក្នុងគម្រោង
- រចនាសម្ព័ន្ធថតមិនត្រឹមត្រូវ

**ដំណោះស្រាយ:**
1. **ប្រើរបៀប debug** ដើម្បីមើលអ្វីកំពុងកើតឡើង៖
   ```bash
   translate -l "ko" -md -d
   ```

2. **ពិនិត្យប្រភេទឯកសារ** ក្នុងគម្រោងរបស់អ្នក៖
   ```bash
   # សម្រាប់ឯកសារ markdown
   find . -name "*.md" -not -path "./translations/*"
   
   # សម្រាប់សៀវភៅកំណត់ត្រា
   find . -name "*.ipynb" -not -path "./translations/*"
   
   # សម្រាប់រូបភាព
   find . -name "*.png" -o -name "*.jpg" -o -name "*.jpeg" -not -path "./translations/*"
   ```

3. **ធានាការបញ្ចូលសញ្ញា** ត្រឹមត្រូវ៖
   ```bash
   # ប្រែសម្រួលអ្វីៗទាំងអស់ (លំនាំដើម)
   translate -l "ko"
   
   # ប្រែសម្រួលប្រភេទជាក់លាក់
   translate -l "ko" -md -img
   ```

## 6. ការផ្លាស់ប្តូរពីប្រព័ន្ធចាស់

### 6.1. របៀប Markdown-Only ត្រូវបានបញ្ឈប់ប្រើ

**បញ្ហា៖** ពាក្យបញ្ជាដែលពឹងផ្អែកលើ fallback Markdown-Only ឥឡូវមិនដំណើរការ ដូចដែលបានរំពឹងទុក។

**អ្វីដែលធ្លាប់ធ្វើ:**
```bash
# នេះធ្លាប់ត្រូវបានប្រើសម្រាប់ផ្លាស់ប្តូរជាម៉ូដ markdown តែប៉ុណ្ណោះដោយស្វ័យប្រវត្តិ
translate -l "ko"  # (នៅពេលដែល Azure AI Vision មិនត្រូវបានកំណត់)
```

**អ្វីដែលអាចធ្វើបានថ្មី:**
```bash
# នេះឥឡូវនេះបង្កើតកំហុសប្រសិនបើការបកប្រែរូបភាពត្រូវបានស្នើសុំ ប៉ុន្តែមិនបានកំណត់រចនាសម្ព័ន្ធ។
translate -l "ko" -img
```

**ដំណោះស្រាយ:**
- **បញ្ជាក់ជាក់លាក់** អំពីអ្វីដែលអ្នកចង់បកប្រែ៖
  ```bash
  translate -l "ko" -md        # តែ markdown តែប៉ុណ្ណោះ
  translate -l "ko" -md -img   # Markdown និងរូបភាព
  translate -l "ko"            # អ្វីៗគ្រប់យ៉ាង (ប្រសិនបើសេវាកម្មទាំងអស់ត្រូវបានកំណត់រចនា)
  ```

### 6.2. ពាក់ព័ន្ធនឹងតំណភ្ជាប់មិនរំពឹងទុក

**បញ្ហា៖** តំណភ្ជាប់ក្នុងឯកសារដែលបានបកប្រែ បង្ហាញទៅកាន់កន្លែងដែលមិនរំពឹងទុក។

**ហេតុ៖** ការបម្លែងតំណភ្ជាប់ δυναμικόប្តូរថាតើប្រភេទឯកសារអ្វីដែលបានជ្រើស។

**ដំណោះស្រាយ:**
1. **យល់ពីរបៀបតំណភ្ជាប់ថ្មី៖**
   - បញ្ចូល `-nb`: តំណចំណាត់ថ្នាក់ Notebook បង្ហាញទៅកាន់ជម្រើសទម្រង់បកប្រែ
   - មិនបញ្ចូល `-nb`: តំណ Notebook បង្ហាញទៅកាន់ឯកសារដើម
   - បញ្ចូល `-img`: តំណរូបភាពបង្ហាញទៅកាន់ជម្រើសបកប្រែ
   - មិនបញ្ចូល `-img`: តំណរូបភាពបង្ហាញទៅកាន់ឯកសារដើម

2. **ជ្រើសរើសការលាយឡំត្រឹមត្រូវ** សម្រាប់ករណីប្រើប្រាស់របស់អ្នក៖
   ```bash
   # តំណខាងក្នុងទាំងអស់បញ្ជាក់ទៅកាន់ជំរើសដែលបានបកបរា
   translate -l "ko" -md -img -nb
   
   # មានតែកំណត់ត្រាទៅជាភាសាបកបរា តំណផ្សេងទៀតបញ្ជាក់ទៅកាន់លំនាំដើម</s>
   translate -l "ko" -md
   ```

## 7. GitHub Action ដំណើរការនៅឡើយ ប៉ុន្តែមិនបានបង្កើត Pull Request (PR)

**រាងបញ្ហា:** កំណត់ហេតុវីមផតសម្រាប់ `peter-evans/create-pull-request` បង្ហាញ៖

> Branch 'update-translations' មិននៅមុខ base 'main' ហើយមិនត្រូវបានបង្កើតឡើយ

**ហេតុសាច់ធុរកិច្ច៖**
- **គ្មានការផ្លាស់ប្តូរ:** ជំហានបកប្រែមិនបានបង្កើត diff ទេ (repo បានធ្វើបច្ចុប្បន្នភាពរួចហើយ)។
- **ការមិនគិតពីតម្លៃផ្តល់ចេញ:** `.gitignore` មិនអនុញ្ញាតឲ្យ commit ឯកសារដែលអ្នករំពឹងទុក (ដូចជា `*.ipynb`, `translations/`, `translated_images/`)។
- **add-paths មិនផ្គូរផ្គង:** ផ្លូវដែលផ្ដល់ទៅសកម្មភាពមិនត្រូវនឹងទីតាំងលទ្ធផលពិតប្រាកដ។
- **ត្រួតពិនិត្យល្បឿន/លក្ខខណ្ឌកម្មវិធី:** ជំហានបកប្រែបានបញ្ចប់មុនពេល ឬបានសរសេរទៅថតដែលមិនរំពឹងទុក។

**វិធីដោះស្រាយ / ពិនិត្យៈ**
1. **ធ្វើអោយប្រាកដថាលទ្ធផលមាន៖** បន្ទាប់ពីបកប្រែ សូមពិនិត្យថា workspace មានឯកសារថ្មី/បានផ្លាស់ប្តូរនៅ `translations/` និង/ឬ `translated_images/`។
   - ប្រសិនបើកំពុងបកប្រែ notebook ប្រាកដថា .ipynb បានសរសេរពិតប្រាកដនៅក្រោម `translations/<lang>/...`។
2. **ពិនិត្យ .gitignore:** កុំបានរំលងលទ្ធផលដែលបង្កើត។ ត្រូវប្រាកដថាមិនរំលង៖
   - `translations/`
   - `translated_images/`
   - `*.ipynb` (ប្រសិនបើកំពុងបកប្រែ notebook)
3. **ធានាថា add-paths ត្រូវនឹងលទ្ធផល:** ប្រើតម្លៃបន្ទាត់ច្រើន ហើយរួមបញ្ចូលទាំងពីរថត ប្រសិនបើមាន៖
   ```yaml
   with:
     add-paths: |
       translations/
       translated_images/
   ```
4. **បង្ខំសម្រាប់ PR ដើម្បី debug:** អនុញ្ញាតការបញ្ចូល commit ទទេបណ្តោះអាសន្ន ដើម្បីធ្វើអោយប្រាកដថាពាក្យបញ្ជាក់ត្រឹមត្រូវ៖
   ```yaml
   with:
     commit-empty: true
   ```
5. **រត់ជាមួយ режиме debug:** បន្ថែម `-d` ទៅពាក្យបញ្ជារ ដើម្បីបង្ហាញថា មានឯកសារអ្វីខ្លះដែលបានរកឃើញ និងបានសរសេរ។
6. **សិទ្ធិ (GITHUB_TOKEN):** ប្រាកដថា workflow មានសិទ្ធិសរសេរដើម្បីបង្កើត commit និង PR៖
   ```yaml
   permissions:
     contents: write
     pull-requests: write
   ```

## បញ្ជីពិនិត្យបន្ទាន់ក្នុងការដោះស្រាយបញ្ហា

ពេលបញ្ហាបកប្រែ៖

1. **ប្រើរបៀប debug**៖ បន្ថែម `-d` ដើម្បីមើលកំណត់ហេតុលម្អិត
2. **ពិនិត្យសញ្ញាទាំងអស់**៖ ប្រាកដថា `-md`, `-img`, `-nb` ត្រូវនឹងគំរោងអ្នក
3. **ពិនិត្យកំណត់រចនាសម្ព័ន្ធ**៖ ពិនិត្យឯកសារ `.env` មានកូនសោត្រឹមត្រូវ
4. **សាកល្បងជាថ្នាក់ៗ**៖ ចាប់ផ្តើមពី `-md` ទេបន្ទាប់បន្ថែមប្រភេទផ្សេងៗ
5. **ពិនិត្យរចនាសម្ព័ន្ធឯកសារ**៖ ធ្វើអោយប្រាកដថា ឯកសារដើមមាន និងអាចចូលដំណើរការ

សម្រាប់ព័ត៌មានលម្អិតអំពីពាក្យបញ្ជា និងសញ្ញា សូមមើល [Command Reference](./command-reference.md)។
