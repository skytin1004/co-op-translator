## សន្តិសុខ

Microsoft យកចិត្តទុកដាក់យ៉ាងខ្លាំងចំពោះសន្តិសុខនៃផលិតផល និងសេវាកម្ម software របស់យើង ដែលរួមបញ្ចូលទៅដល់គ្រប់ឃ្លោងកូដប្រភពដែលគ្រប់គ្រងតាមរយៈអង្គការជាច្រើនរបស់យើងលើ GitHub ដែលរួមមាន [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) និង [Xamarin](https://github.com/xamarin)។

បើអ្នកជឿថាអ្នកបានរកឃើញចំណុចងាយរងគ្រោះសន្តិសុខនៅក្នុងឃ្លាំងអចិន្រ្តៃយ៍ដែលជា mil owned by Microsoft ដែលផ្គូផ្គងនឹង [គំនិតនៃចំណុចងាយរងគ្រោះសន្តិសុខរបស់ Microsoft](https://aka.ms/security.md/definition), សូមរាយការណ៍វាមកកាន់យើងតាមការពិពណ៍នាខាងក្រោម ។

## របាយការណ៍បញ្ហាសន្តិសុខ

**សូមកុំរាយការណ៍ចំណុចងាយរងគ្រោះសន្តិសុខតាមរយៈបញ្ហាសាធារណៈលើ GitHub។**

ជំនួសវិញ សូមរាយការណ៍វាទៅកាន់មជ្ឈមណ្ឌលប្រតិបត្ដិការឆ្លើយតបសន្តិសុខ Microsoft (MSRC) នៅ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)។

បើអ្នកចូលចិត្តដាក់ស្នើដោយមិនចូលប្រើប្រាស់គណនី អូស៊ីល​អ៊ីមែលទៅ [secure@microsoft.com](mailto:secure@microsoft.com)។ បើអាចធ្វើបាន សូម mãសុវត្ថិភាពសាររបស់អ្នកជាមួយកូនសោ PGP របស់យើង​; សូមទាញយកវាពីទំព័រ [Microsoft Security Response Center PGP Key](https://aka.ms/security.md/msrc/pgp)។

អ្នកគួរត្រូវទទួលបានចម្លើយក្នុងរយៈពេល 24ម៉ោង។ ប្រសិនបើមានមូលហេតុណាមួយដែលមិនទាន់ទទួលបាន សូមតាមដានតាមអ៊ីមែលដើម្បីធានាថាយើងបានទទួលសារដើមរបស់អ្នក។ ព័ត៌មានបន្ថែមអាចរកបាននៅ [microsoft.com/msrc](https://www.microsoft.com/msrc)។

សូមរាប់បញ្ចូលព័ត៌មានដែលបានស្នើសុំខាងក្រោម (តាមដែលអាចផ្គត់ផ្គង់បាន) ដើម្បីជួយយើងយល់កាន់តែក្រាស់ពីធាតុ និងវិវត្តន៍នៃបញ្ហាអាចមាននេះ៖

  * ប្រភេទបញ្ហា (ឧ. buffer overflow, SQL injection, cross-site scripting, លំដាប់)
  * ផ្លូវពេញនៃឯកសារប្រភពដែលទាក់ទងនឹងបញ្ហា
  * ទីតាំងនៃកូដប្រភពដែលខូច (tag/branch/commit ឬ URL ត្រង់)
  * ការកំណត់ជាក់លាក់ណាមួយដែលត្រូវការដើម្បីកើតឡើងវិញបញ្ហា
  * ជំហាន-ដោយ-ជំហានដើម្បីកើតឡើងវិញបញ្ហា
  * កូដគំរូបង្ហាញ ឬកូដប្រើប្រាស់ (បើយល់អាច មាន)
  * ផលប៉ះពាល់នៃបញ្ហា រួមទាំងរបៀបដែលអ្នករកស៊ីអាចប្រើប្រាស់បញ្ហានេះ

ព័ត៌មាននេះនឹងជួយឲ្យយើងដំណើរការរាយការណ៍របស់អ្នកបានរហ័សបំផុត។

បើអ្នកកំពុងរាយការណ៍សម្រាប់រង្វាន់ bug bounty, របាយការណ៍ពេញលេញជាងនេះអាចជួយបង្កើនរង្វាន់។ សូមចូលទស្សនាទំព័រ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) សម្រាប់ព័ត៌មានលម្អិតបន្ថែមពីកម្មវិធីមានសកម្មរបស់យើង។

## ភាសាចូលចិត្ត

យើងចូលចិត្តឲ្យការទំនាក់ទំនងទាំងអស់ធ្វើឡើងជាភាសាអង់គ្លេស។

## នយោបាយ

Microsoft អនុវត្តតាមគោលការណ៍នៃ [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd)។
