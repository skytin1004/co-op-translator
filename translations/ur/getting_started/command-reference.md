# کمانڈ حوالہ

**Co-op Translator** CLI ترجمے کے عمل کو حسب ضرورت بنانے کے لیے کئی اختیارات فراہم کرتا ہے:

کمانڈ                                       | وضاحت
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | آپ کے پروجیکٹ کو مخصوص زبانوں میں ترجمہ کرتا ہے۔ مثال: translate -l "es fr de" ہسپانوی، فرانسیسی اور جرمن میں ترجمہ کرتا ہے۔ تمام حمایت یافتہ زبانوں میں ترجمہ کے لیے translate -l "all" استعمال کریں۔
translate -l "language_codes" -u              | ترجمے کو اپ ڈیٹ کرتا ہے، موجودہ ترجمے حذف کر کے دوبارہ بناتا ہے۔ انتباہ: یہ مخصوص زبانوں کے تمام موجودہ ترجمے حذف کر دے گا۔
translate -l "language_codes" -img            | صرف تصویر کی فائلوں کا ترجمہ کرتا ہے۔
translate -l "language_codes" -md             | صرف مارک ڈاؤن فائلوں کا ترجمہ کرتا ہے۔
translate -l "language_codes" -nb             | صرف Jupyter نوٹ بک فائلوں (.ipynb) کا ترجمہ کرتا ہے۔
translate -l "language_codes" --fix           | پچھلے جائزہ کے نتائج کی بنیاد پر کم اعتماد والے اسکور والے فائلوں کو دوبارہ ترجمہ کرتا ہے۔
translate -l "language_codes" -d              | مفصل لاگنگ کے لیے ڈیبگ موڈ فعال کرتا ہے۔
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ کے تحت فائلوں میں DEBUG سطح کے لاگز محفوظ کریں (کنسول پر -d کے ذریعے کنٹرول ہوتا ہے)
translate -l "language_codes" -r "root_dir"   | پروجیکٹ کی روٹ ڈائریکٹری کی وضاحت کرتا ہے
translate -l "language_codes" -f              | تصویر کے ترجمے کے لیے تیز رفتار موڈ استعمال کرتا ہے (معیار اور ترتیب کے معمولی فرق کے ساتھ تین گنا تیز تر تصویر کشی)۔
translate -l "language_codes" -y              | تمام پرامپٹس کو خودکار طور پر تصدیق کرتا ہے (CI/CD پائپ لائنز کے لیے مفید)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | ترجمہ شدہ مارک ڈاؤن اور نوٹ بکس میں مشین ترجمہ کی چھوٹ شامل کرنے یا نہ کرنے کو فعال یا غیر فعال کرتا ہے (پہلے سے فعال ڈیفالٹ)۔
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | آپ کی ریپوزیٹری URL کے ساتھ README زبانوں کے سیکشن کی مشورہ جاتی ذاتی تخصیص (اسپارس چیک آؤٹ)۔
translate -l "language_codes" --help          | CLI میں دستیاب کمانڈز کی تفصیلات دکھاتا ہے
evaluate -l "language_code"                  | خاص زبان کے ترجمے کے معیار کا جائزہ لیتا ہے اور اعتماد کے اسکور فراہم کرتا ہے
evaluate -l "language_code" -c 0.8           | حسب ضرورت اعتماد کی حد کے ساتھ ترجموں کا جائزہ لیتا ہے
evaluate -l "language_code" -f               | تیز جائزہ موڈ (صرف قاعدہ پر مبنی، کوئی LLM نہیں)
evaluate -l "language_code" -D               | گہرا جائزہ موڈ (صرف LLM پر مبنی، زیادہ تفصیلی لیکن سست)
evaluate -l "language_code" --save-logs, -s  | <root_dir>/logs/ کے تحت فائلوں میں DEBUG سطح کے لاگز محفوظ کریں
migrate-links -l "language_codes"             | ترجمہ شدہ مارک ڈاؤن فائلوں کو دوبارہ پروسیس کرتا ہے تاکہ نوٹ بکس (.ipynb) کے لنکس کو اپ ڈیٹ کیا جا سکے۔ ترجمہ شدہ نوٹ بکس کو ترجیح دیتا ہے اگر دستیاب ہوں؛ ورنہ اصل نوٹ بکس پر واپس جا سکتا ہے۔
migrate-links -l "language_codes" -r          | پروجیکٹ کی روٹ ڈائریکٹری کی وضاحت کریں (ڈیفالٹ: موجودہ ڈائریکٹری)۔
migrate-links -l "language_codes" --dry-run   | دکھاتا ہے کہ کون سی فائلیں بدلیں گی بغیر کسی تبدیلی کو لکھے۔
migrate-links -l "language_codes" --no-fallback-to-original | جب ترجمہ شدہ ہم منصب غائب ہوں تو اصل نوٹ بکس کے لنکس کو دوبارہ لکھنے سے گریز کریں (صرف اپ ڈیٹ کریں جب ترجمہ شدہ موجود ہو)۔
migrate-links -l "language_codes" -d          | مفصل لاگنگ کے لیے ڈیبگ موڈ فعال کریں۔
migrate-links -l "language_codes" --save-logs, -s | <root_dir>/logs/ کے تحت DEBUG سطح کے لاگز محفوظ کریں
migrate-links -l "all" -y                      | تمام زبانوں کو پروسیس کریں اور وارننگ پرامپٹ کو خودکار طور پر تصدیق کریں۔

## استعمال کی مثالیں

  1. ڈیفالٹ رویہ (موجودہ ترجمے حذف کیے بغیر نئے ترجمے شامل کریں):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. صرف نئے کورین تصویر ترجمے شامل کریں (موجودہ ترجمے حذف نہیں ہوتے):    translate -l "ko" -img

  3. تمام کورین ترجمے اپ ڈیٹ کریں (انتباہ: یہ تمام موجودہ کورین ترجمے حذف کر کے دوبارہ ترجمہ کرے گا):    translate -l "ko" -u

  4. صرف کورین تصاویر اپ ڈیٹ کریں (انتباہ: یہ تمام موجودہ کورین تصویریں حذف کر کے دوبارہ ترجمہ کرے گا):    translate -l "ko" -img -u

  5. دیگر ترجموں پر اثر ڈالے بغیر کورین کے لیے نئے مارک ڈاؤن ترجمے شامل کریں:    translate -l "ko" -md

  6. پچھلے جائزے کے نتائج کی بنیاد پر کم اعتماد والے ترجمے درست کریں: translate -l "ko" --fix

  7. مخصوص فائلوں کے لیے فقط مارک ڈاؤن میں کم اعتماد والے ترجمے درست کریں: translate -l "ko" --fix -md

  8. مخصوص فائلوں کے لیے فقط تصاویر میں کم اعتماد والے ترجمے درست کریں: translate -l "ko" --fix -img

  9. تصویر کے ترجمے کے لیے تیز رفتار موڈ استعمال کریں:    translate -l "ko" -img -f

  10. حسب ضرورت حد کے ساتھ کم اعتماد والے ترجمے درست کریں: translate -l "ko" --fix -c 0.8

  11. ڈیبگ موڈ کی مثال: - translate -l "ko" -d: ڈیبگ لاگنگ فعال کریں۔
  12. لاگز فائلوں میں محفوظ کریں: translate -l "ko" -s
  13. کنسول DEBUG اور فائل DEBUG دونوں: translate -l "ko" -d -s
  14. مشین ترجمہ کی چھوٹ شامل کیے بغیر ترجمہ کریں: translate -l "ko" --no-disclaimer

  15. کورین ترجموں کے لیے نوٹ بک لنکس کو منتقل کریں (جب ترجمہ شدہ نوٹ بکس دستیاب ہوں تو لنکس اپ ڈیٹ کریں):    migrate-links -l "ko"

  15. ڈرائی رن کے ساتھ لنکس منتقل کریں (کوئی فائل نہیں لکھی جائے گی):    migrate-links -l "ko" --dry-run

  16. صرف تب لنکس اپ ڈیٹ کریں جب ترجمہ شدہ نوٹ بکس موجود ہوں (اصلی پر واپس نہ جائیں):    migrate-links -l "ko" --no-fallback-to-original

  17. تمام زبانوں کو پرامپٹ کے ساتھ پروسیس کریں:    migrate-links -l "all"

  18. تمام زبانوں کو خودکار تصدیق کے ساتھ پروسیس کریں:    migrate-links -l "all" -y
  19. migrate-links کے لیے لاگز فائلوں میں محفوظ کریں:    migrate-links -l "ko ja" -s

  20. اپنی ریپوزیٹری URL کے ساتھ README زبانوں کے مشورے کو ذاتی بنائیں:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### جائزہ لینے کی مثالیں

> [!WARNING]  
> **بیٹا فیچر**: جائزہ لینے کی فعالیت اس وقت بیٹا میں ہے۔ یہ فیچر ترجمہ شدہ دستاویزات کے جائزے کے لیے جاری کیا گیا ہے، اور جائزے کے طریقے اور تفصیلی نفاذ ابھی ترقی کے مراحل میں ہیں اور تبدیلی کے تابع ہیں۔

  1. کورین ترجموں کا جائزہ لیں: evaluate -l "ko"

  2. حسب ضرورت اعتماد کی حد کے ساتھ جائزہ لیں: evaluate -l "ko" -c 0.8

  3. تیز جائزہ (صرف قاعدہ پر مبنی): evaluate -l "ko" -f

  4. گہرا جائزہ (صرف LLM پر مبنی): evaluate -l "ko" -D
