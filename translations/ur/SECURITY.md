## سیکیورٹی

مائیکروسافٹ ہمارے سافٹ ویئر مصنوعات اور خدمات کی سیکیورٹی کو سنجیدگی سے لیتا ہے، جس میں ہمارے GitHub تنظیموں کے ذریعے منظم تمام سورس کوڈ ذخائر شامل ہیں، جن میں [Microsoft](https://github.com/Microsoft)، [Azure](https://github.com/Azure)، [DotNet](https://github.com/dotnet)، [AspNet](https://github.com/aspnet) اور [Xamarin](https://github.com/xamarin) شامل ہیں۔

اگر آپ کو یقین ہے کہ آپ نے کسی بھی مائیکروسافٹ کے ملکیتی ذخیرے میں ایسی سیکیورٹی خامی دریافت کی ہے جو [مائیکروسافٹ کی سیکیورٹی خامی کی تعریف](https://aka.ms/security.md/definition) پر پوری اترتی ہو، تو براہ کرم اسے نیچے دیے گئے طریقے سے ہمیں رپورٹ کریں۔

## سیکیورٹی مسائل کی رپورٹنگ

**براہ کرم سیکیورٹی خامیوں کو عوامی GitHub مسائل کے ذریعے رپورٹ نہ کریں۔**

اس کے بجائے، براہ کرم انہیں مائیکروسافٹ سیکیورٹی ریسپانس سینٹر (MSRC) پر رپورٹ کریں [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)۔

اگر آپ لاگ ان کیے بغیر جمع کرانا پسند کرتے ہیں، تو ای میل بھیجیں [secure@microsoft.com](mailto:secure@microsoft.com) پر۔ اگر ممکن ہو تو، اپنے پیغام کو ہمارے PGP کلید کے ساتھ انکرپٹ کریں؛ براہ کرم اسے [مائیکروسافٹ سیکیورٹی ریسپانس سینٹر PGP کلید کے صفحہ](https://aka.ms/security.md/msrc/pgp) سے ڈاؤن لوڈ کریں۔

آپ کو 24 گھنٹے کے اندر جواب ملنا چاہیے۔ اگر کسی وجہ سے نہ ملا تو براہ کرم ای میل کے ذریعے فالو اپ کریں تاکہ اس بات کو یقینی بنایا جا سکے کہ ہمیں آپ کا اصل پیغام موصول ہو چکا ہے۔ مزید معلومات [microsoft.com/msrc](https://www.microsoft.com/msrc) پر دستیاب ہیں۔

براہ کرم ممکنہ مسئلے کی نوعیت اور دائرہ کار کو بہتر سمجھنے کے لیے درج ذیل درخواست کردہ معلومات شامل کریں (جتنا ہو سکے فراہم کریں):

  * مسئلے کی قسم (مثلاً بفر اوور فلو، SQL انجیکشن، کراس سائٹ اسکرپٹنگ، وغیرہ)
  * مسئلے کے ظہور سے متعلق سورس فائلوں کے مکمل راستے
  * متاثرہ سورس کوڈ کا مقام (ٹیگ/برانچ/کمیٹ یا براہ راست URL)
  * مسئلہ دوبارہ پیدا کرنے کے لیے کوئی خاص کنفیگریشن ضروری ہو
  * مسئلہ دوبارہ پیدا کرنے کے لیے مرحلہ وار ہدایات
  * پروف آف کانسیپٹ یا ایکسپلائٹ کوڈ (اگر ممکن ہو)
  * مسئلے کا اثر، بشمول یہ کہ حملہ آور کس طرح اس مسئلے کا فائدہ اٹھا سکتا ہے

یہ معلومات ہمیں آپ کی رپورٹ کو جلد از جلد جانچنے میں مدد دے گی۔

اگر آپ بگ بائونٹی کے لیے رپورٹ کر رہے ہیں، تو مکمل رپورٹس اعلیٰ بائونٹی انعام میں مدد کر سکتی ہیں۔ مزید تفصیلات کے لیے براہ کرم ہمارا [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) صفحہ ملاحظہ کریں۔
