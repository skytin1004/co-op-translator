## سیکیورٹی

مائیکروسافٹ اپنے سافٹ ویئر مصنوعات اور خدمات کی سیکیورٹی کو سنجیدگی سے لیتا ہے، جس میں ہمارے GitHub تنظیموں کے ذریعے منظم تمام سورس کوڈ ریپوزیٹریز شامل ہیں، جن میں [Microsoft](https://github.com/Microsoft)، [Azure](https://github.com/Azure)، [DotNet](https://github.com/dotnet)، [AspNet](https://github.com/aspnet) اور [Xamarin](https://github.com/xamarin) شامل ہیں۔

اگر آپ کو یقین ہے کہ آپ نے کسی مائیکروسافٹ کی ملکیت والی ریپوزیٹری میں کوئی سیکیورٹی کمزوری دیکھی ہے جو [مائیکروسافٹ کی سیکیورٹی کمزوری کی تعریف](https://aka.ms/security.md/definition) پر پوری اترتی ہو، تو براہ کرم نیچے دیے گئے طریقے کے مطابق ہمیں اطلاع دیں۔

## سیکیورٹی مسائل کی رپورٹنگ

**براہ کرم سیکیورٹی کمزوریوں کی رپورٹنگ پبلک GitHub مسائل کے ذریعے نہ کریں۔**

اس کی بجائے، براہ کرم انہیں مائیکروسافٹ سیکیورٹی ریسپانس سینٹر (MSRC) پر رپورٹ کریں [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) پر۔

اگر آپ بغیر لاگ ان کیے جمع کروانا پسند کرتے ہیں تو، ای میل بھیجیں [secure@microsoft.com](mailto:secure@microsoft.com) پر۔ اگر ممکن ہو تو، ہمارے PGP کلید کے ساتھ اپنا پیغام انکرپٹ کریں؛ براہ کرم اسے [Microsoft Security Response Center PGP Key صفحہ](https://aka.ms/security.md/msrc/pgp) سے ڈاؤن لوڈ کریں۔

آپ کو 24 گھنٹوں کے اندر جواب موصول ہونا چاہئے۔ اگر کسی وجہ سے ایسا نہ ہو، تو براہ کرم ای میل کے ذریعہ فالو اپ کریں تاکہ یہ یقینی بنایا جا سکے کہ ہمیں آپ کا اصل پیغام موصول ہو چکا ہے۔ مزید معلومات [microsoft.com/msrc](https://www.microsoft.com/msrc) پر دستیاب ہیں۔

براہ کرم نیچے درج معلومات شامل کریں (جہنی ممکن ہو فراہم کریں) تاکہ ہم ممکنہ مسئلے کی نوعیت اور دائرہ کار کو بہتر سمجھ سکیں:

  * مسئلے کی قسم (جیسے بفر اوور فلو، SQL انجیکشن، کراس سائٹ اسکرپٹنگ، وغیرہ)
  * مسئلے کی ظاہری شکل سے متعلق سورس فائلوں کے مکمل راستے
  * متاثرہ سورس کوڈ کا مقام (ٹیگ/برانچ/کمیٹ یا براہ راست URL)
  * مسئلہ دوبارہ پیدا کرنے کے لیے کوئی خاص ترتیب
  * مسئلہ دوبارہ پیدا کرنے کے لیے مرحلہ وار ہدایات
  * ثبوت کے طور پر تصور یا ایکسپلائٹ کوڈ (اگر ممکن ہو)
  * مسئلے کا اثر، بشمول یہ کہ حملہ آور کس طرح مسئلے کا فائدہ اٹھا سکتا ہے

یہ معلومات ہمیں آپ کی رپورٹ کو تیزی سے جانچنے میں مدد دے گی۔

اگر آپ بگ باؤنٹی کے لیے رپورٹ کر رہے ہیں تو، مکمل رپورٹس زیادہ باؤنٹی انعام میں مددگار ثابت ہو سکتی ہیں۔ براہ کرم ہمارے [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) صفحہ پر ہماری فعال پروگراموں کی مزید تفصیلات دیکھیں۔
