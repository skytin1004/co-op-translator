## جائزہ

**Co-op Translator** آپ کے تعلیمی GitHub مواد کو باآسانی متعدد زبانوں میں لوکلائز کرنے میں مدد دیتا ہے۔
جب آپ اپنے مارک ڈاون فائلیں، تصاویر یا نوٹ بکس کو اپ ڈیٹ کرتے ہیں، تو ترجمہ خودکار طریقے سے ہم آہنگ رہتے ہیں، یہ یقینی بناتے ہوئے کہ آپ کا مواد عالمی سطح پر سیکھنے والوں کے لیے درست اور تازہ ترین رہے۔

## ترجمے کی حالت کیسے منظم کی جاتی ہے

Co-op Translator ترجمہ شدہ مواد کو **ورژن شدہ سافٹ ویئر آرٹیفیکٹس** کے طور پر منظم کرتا ہے،  
نہ کہ جامد فائلوں کے طور پر۔

یہ آلہ ترجمہ شدہ مارک ڈاؤن، تصاویر، اور نوٹ بکس کی حالت  
**زبان کے مطابق میٹا ڈیٹا** کے ذریعے ٹریک کرتا ہے۔

یہ ڈیزائن Co-op Translator کو اجازت دیتا ہے کہ:

- پرانے ترجموں کا قابل اعتماد پتہ لگایا جائے
- مارک ڈاؤن، تصاویر، اور نوٹ بکس کو مستقل مزاجی سے ٹریٹ کیا جائے
- بڑے، تیزی سے بڑھتے ہوئے کثیرالزبانی ذخیرہ جات میں محفوظ طریقے سے اسکیل کیا جائے

ترجموں کو منظم شدہ آرٹیفیکٹس کے طور پر ماڈل کرکے،  
ترجمہ ورک فلو جدید سافٹ ویئر انحصار اور آرٹیفیکٹ مینجمنٹ کے طریقوں کے ساتھ فطری طور پر ہم آہنگ ہوتے ہیں۔

→ [ترجمہ کی حالت کو کیسے منظم کیا جاتا ہے](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)


## جلدی شروع کریں

```bash
# ایک ورچوئل ماحول بنائیں اور اسے فعال کریں (تجویز کیا گیا)
python -m venv .venv
# ونڈوز
.venv\Scripts\activate
# میک او ایس/لینکس
source .venv/bin/activate
# پیکیج انسٹال کریں
pip install co-op-translator
# ترجمہ کریں
translate -l "ko ja fr" -md
```

Docker:

```bash
# GHCR سے عوامی امیج کھینچیں
docker pull ghcr.io/azure/co-op-translator:latest
# موجودہ فولڈر کو ماؤنٹ کریں اور .env فراہم کریں (Bash/Zsh)
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## کم از کم سیٹ اپ

1. یقین دہانی کریں کہ آپ کے پاس معاون پائیتھن ورژن ہے (فی الوقت 3.10-3.12)۔ poetry (pyproject.toml) میں یہ خود کار طریقے سے ہینڈل ہوتا ہے۔
2. ایک `.env` فائل بنائیں ٹیمپلیٹ استعمال کرتے ہوئے: [.env.template](../../.env.template)
3. ایک LLM فراہم کنندہ کی ترتیب دیں (Azure OpenAI یا OpenAI)
4. (اختیاری) تصویر کے ترجمے کے لیے (`-img`)، Azure AI Vision کو ترتیب دیں
5. (اختیاری) آپ متعدد کریڈینشل سیٹ ترتیب دے سکتے ہیں متغیرات کو suffix جیسے `_1`, `_2` وغیرہ دے کر دہرائیں۔ ایک سیٹ کے تمام متغیرات کو ایک ہی suffix شیئر کرنا ضروری ہے۔
6. (سفارش کی گئی) پچھلے ترجموں کو صاف کریں تاکہ تضادات سے بچا جا سکے (مثلاً `translations/`)
7. (سفارش کی گئی) اپنے README میں ترجمے کا سیکشن شامل کریں [README languages template](./getting_started/README_languages_template.md) استعمال کرتے ہوئے
8. دیکھیں: [Azure AI سیٹ اپ کریں](./getting_started/set-up-azure-ai.md)

## استعمال

تمام معاون اقسام کا ترجمہ کریں:

```bash
translate -l "ko ja"
```

صرف مارک ڈاؤن:

```bash
translate -l "de" -md
```

مارک ڈاؤن + تصاویر:

```bash
translate -l "pt" -md -img
```

صرف نوٹ بکس:

```bash
translate -l "zh" -nb
```

مزید جھنڈے: [کمانڈ کا حوالہ](./getting_started/command-reference.md)

بڑے پیمانے پر قدرتی زبان، تصویر، اور تقریر کے ماڈلز - جیسے اس نمونے میں استعمال ہونے والے - ممکنہ طور پر ایسے طریقوں سے برتاؤ کر سکتے ہیں جو غیر منصفانہ، ناقابل اعتماد، یا ناگوار ہوں، جس سے نقصان ہو سکتا ہے۔ خطرات اور حدود کے بارے میں معلومات کے لیے براہ کرم [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) دیکھیں۔

ان خطرات کو کم کرنے کے لیے تجویز کردہ طریقہ یہ ہے کہ آپ اپنے فن تعمیر میں ایک حفاظتی نظام شامل کریں جو نقصان دہ رویے کا پتہ لگا سکے اور اسے روک سکے۔ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) ایک آزاد حفاظتی تہہ فراہم کرتا ہے، جو درخواستوں اور خدمات میں نقصان دہ صارف پیدا کردہ اور AI پیدا کردہ مواد کا پتہ لگا سکتا ہے۔ Azure AI Content Safety میں متن اور تصویر کے API شامل ہیں جو آپ کو نقصان دہ مواد کی شناخت کرنے کی اجازت دیتے ہیں۔ ہمارے پاس ایک انٹرایکٹو Content Safety Studio بھی ہے جو آپ کو مختلف طریقوں میں نقصان دہ مواد کی شناخت کے لیے نمونہ کوڈ دیکھنے، دریافت کرنے، اور آزمانے کی سہولت دیتا ہے۔ درج ذیل [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) آپ کو سروس کو درخواستیں بھیجنے میں رہنمائی فراہم کرتی ہے۔

ایک اور پہلو جس پر غور کرنا چاہیے وہ مجموعی درخواست کی کارکردگی ہے۔ کثیر الوضعی اور کثیر ماڈلز والی درخواستوں کے ساتھ، ہم کارکردگی کو اس کے طور پر سمجھتے ہیں کہ نظام آپ اور آپ کے صارفین کی توقعات کے مطابق کام کرے، بشمول نقصان دہ نتائج نہ پیدا کرنا۔ اپنی مجموعی درخواست کی کارکردگی کا اندازہ لگانا ضروری ہے، جو آپ [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) کا استعمال کر کے کر سکتے ہیں۔

آپ اپنی AI درخواست کا جائزہ اپنی ترقی کے ماحول میں [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) استعمال کرتے ہوئے کر سکتے ہیں۔ چاہے آپ کے پاس کوئی ٹیسٹ ڈیٹا سیٹ ہو یا کوئی ہدف، آپ کی تولیدی AI درخواست کی تخلیقات کو بلٹ ان ایویلیویٹرز یا اپنی پسند کے کسٹم ایویلیویٹرز کے ساتھ مقداری طور پر ناپا جاتا ہے۔ اپنے نظام کی تشخیص کے لیے prompt flow sdk کے ساتھ شروع کرنے کے لیے، آپ [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) پر عمل کر سکتے ہیں۔ ایک بار جب آپ تشخیصی عمل مکمل کر لیں، تو آپ [Azure AI Studio میں نتائج کو دیکھ سکتے ہیں](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results)۔
