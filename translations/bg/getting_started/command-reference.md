# Command reference

The **Co-op Translator** CLI предлага няколко опции за персонализиране на процеса на превод:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | Превежда вашия проект на посочените езици. Пример: translate -l "es fr de" превежда на испански, френски и немски. Използвайте translate -l "all", за да преведете на всички поддържани езици.
translate -l "language_codes" -u              | Актуализира преводите, като изтрива съществуващите и ги създава отново. Внимание: Това ще изтрие всички текущи преводи за посочените езици.
translate -l "language_codes" -img            | Превежда само файлове с изображения.
translate -l "language_codes" -md             | Превежда само Markdown файлове.
translate -l "language_codes" -nb             | Превежда само Jupyter notebook файлове (.ipynb).
translate -l "language_codes" --fix           | Повтаря превода на файлове с ниски нива на увереност, базирано на предишни резултати от оценка.
translate -l "language_codes" -d              | Активира режим на отстраняване на грешки за подробни логове.
translate -l "language_codes" --save-logs, -s | Запазва DEBUG ниво на логове във файлове под <root_dir>/logs/ (конзолата остава контролирана от -d)
translate -l "language_codes" -r "root_dir"   | Указва коренната директория на проекта
translate -l "language_codes" -f              | Използва бърз режим за превод на изображения (до 3 пъти по-бързо изчертаване с минимална загуба на качество и подравняване).
translate -l "language_codes" -y              | Автоматично потвърждава всички заявки (полезно за CI/CD пайплайни)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | Активира или деактивира добавянето на раздел за уточнение за машинен превод в преведените markdown и notebooks (по подразбиране: активирано).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | Персонализира секцията с езици в README (sparse checkout) с URL на вашето репо.
translate -l "language_codes" --help          | справка в CLI, показваща наличните команди
evaluate -l "language_code"                  | Оценява качеството на превод за конкретен език и предоставя оценки на увереността
evaluate -l "language_code" -c 0.8           | Оценява преводите с персонализиран праг на увереност
evaluate -l "language_code" -f               | Бърз режим на оценка (само по правила, без LLM)
evaluate -l "language_code" -D               | Дълбок режим на оценка (само базиран на LLM, по-задълбочен, но по-бавен)
evaluate -l "language_code" --save-logs, -s  | Запазва DEBUG ниво на логове във файлове под <root_dir>/logs/
migrate-links -l "language_codes"             | Преработва преведените Markdown файлове, за да актуализира връзките към notebooks (.ipynb). Предпочита преведените notebooks ако са налични; в противен случай може да се върне към оригиналните notebooks.
migrate-links -l "language_codes" -r          | Указва коренната директория на проекта (по подразбиране: текущата директория).
migrate-links -l "language_codes" --dry-run   | Показва кои файлове биха се променили, без да записва промени.
migrate-links -l "language_codes" --no-fallback-to-original | Не променя връзките към оригинални notebooks, когато преводните липсват (актуализира само когато има превод).
migrate-links -l "language_codes" -d          | Активира режим на отстраняване на грешки за подробни логове.
migrate-links -l "language_codes" --save-logs, -s | Запазва DEBUG ниво на логове във файлове под <root_dir>/logs/
migrate-links -l "all" -y                      | Обработва всички езици и автоматично потвърждава предупреждението.

## Usage examples

  1. По подразбиране (добавя нови преводи без да изтрива съществуващите):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. Добавя само нови преводи на корейски изображения (без изтриване на съществуващи преводи):    translate -l "ko" -img

  3. Актуализира всички корейски преводи (Внимание: Това изтрива всички съществуващи корейски преводи преди превода отново):    translate -l "ko" -u

  4. Актуализира само корейските изображения (Внимание: Това изтрива всички съществуващи корейски изображения преди превода отново):    translate -l "ko" -img -u

  5. Добавя нови markdown преводи за корейски без да влияе на други преводи:    translate -l "ko" -md

  6. Поправя преводи с ниско доверие базирани на предишни резултати от оценка: translate -l "ko" --fix

  7. Поправя ниско доверени преводи само за конкретни файлове (markdown): translate -l "ko" --fix -md

  8. Поправя ниско доверени преводи само за конкретни файлове (изображения): translate -l "ko" --fix -img

  9. Използва бърз режим за превод на изображения:    translate -l "ko" -img -f

  10. Поправя ниско доверени преводи с персонализиран праг: translate -l "ko" --fix -c 0.8

  11. Пример за режим на отстраняване на грешки: - translate -l "ko" -d: Активира логване за отстраняване на грешки.
  12. Запазва логове във файлове: translate -l "ko" -s
  13. DEBUG в конзолата и DEBUG във файлове: translate -l "ko" -d -s
  14. Превод без добавяне на уточнения за машинен превод в изходите: translate -l "ko" --no-disclaimer

  15. Мигрира връзки към notebooks за корейски преводи (актуализира връзките към преведени notebooks, когато са налични):    migrate-links -l "ko"

  15. Мигриране с dry-run (без запис на файлове):    migrate-links -l "ko" --dry-run

  16. Актуализира връзки само когато преведени notebooks съществуват (не се връща към оригиналните):    migrate-links -l "ko" --no-fallback-to-original

  17. Обработва всички езици с подкана за потвърждение:    migrate-links -l "all"

  18. Обработва всички езици и автоматично потвърждава:    migrate-links -l "all" -y
  19. Запазва логове във файлове за migrate-links:    migrate-links -l "ko ja" -s

  20. Персонализира съвета за езици в README с URL на вашето репо:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Evaluation Examples

> [!WARNING]  
> **Бета функция**: Функцията за оценка в момента е в бета версия. Тази функция беше пусната, за да се оценяват преведените документи, а методите за оценка и подробната реализация все още са в процес на разработка и подлежат на промяна.

  1. Оценява корейски преводи: evaluate -l "ko"

  2. Оценява с персонализиран праг на увереност: evaluate -l "ko" -c 0.8

  3. Бърза оценка (само на базата на правила): evaluate -l "ko" -f

  4. Дълбока оценка (само LLM базирана): evaluate -l "ko" -D
