# مرجع دستورات

رابط خط فرمان **Co-op Translator** گزینه‌های متعددی برای سفارشی‌سازی فرآیند ترجمه ارائه می‌دهد:

دستور                                      | توضیحات  
--------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------  
translate -l "language_codes"                 | پروژه شما را به زبان‌های مشخص‌شده ترجمه می‌کند. مثال: translate -l "es fr de" به اسپانیایی، فرانسوی و آلمانی ترجمه می‌کند. از translate -l "all" برای ترجمه به همه زبان‌های پشتیبانی‌شده استفاده کنید.  
translate -l "language_codes" -u              | ترجمه‌ها را به‌روز می‌کند با حذف ترجمه‌های موجود و بازایجاد آن‌ها. هشدار: این دستور همه ترجمه‌های فعلی برای زبان‌های مشخص‌شده را حذف می‌کند.  
translate -l "language_codes" -img            | فقط فایل‌های تصویری را ترجمه می‌کند.  
translate -l "language_codes" -md             | فقط فایل‌های Markdown را ترجمه می‌کند.  
translate -l "language_codes" -nb             | فقط فایل‌های دفترچه یادداشت Jupyter (.ipynb) را ترجمه می‌کند.  
translate -l "language_codes" --fix           | فایل‌هایی که امتیاز اطمینان پایینی دارند بر اساس نتایج ارزیابی قبلی دوباره ترجمه می‌کند.  
translate -l "language_codes" -d              | حالت اشکال‌زدایی را برای ثبت لاگ دقیق فعال می‌کند.  
translate -l "language_codes" --save-logs, -s | لاگ‌های DEBUG را در فایل‌هایی تحت مسیر <root_dir>/logs/ ذخیره می‌کند (کنسول همچنان توسط -d کنترل می‌شود)  
translate -l "language_codes" -r "root_dir"   | پوشه ریشه پروژه را مشخص می‌کند  
translate -l "language_codes" -f              | از حالت سریع برای ترجمه تصاویر استفاده می‌کند (تا ۳ برابر سریع‌تر با اندکی افت کیفیت و تراز)  
translate -l "language_codes" -y              | همه درخواست‌ها را به صورت خودکار تایید می‌کند (مفید برای خط لوله‌های CI/CD)  
translate -l "language_codes" --add-disclaimer/--no-disclaimer | افزودن یا غیرفعال کردن بخش افشای مسئولیت ترجمه ماشینی به فایل‌های Markdown و دفترچه‌ها (پیش‌فرض: فعال).  
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | بخش راهنمای زبان‌های README را با استفاده از URL مخزن شما شخصی‌سازی می‌کند.  
translate -l "language_codes" --help          | نمایش جزئیات کمک درون CLI برای دستورات موجود  
evaluate -l "language_code"                  | کیفیت ترجمه یک زبان خاص را ارزیابی کرده و امتیاز اطمینان ارائه می‌دهد  
evaluate -l "language_code" -c 0.8           | ارزیابی ترجمه‌ها با آستانه اطمینان سفارشی  
evaluate -l "language_code" -f               | حالت ارزیابی سریع (فقط مبتنی بر قوانین، بدون LLM)  
evaluate -l "language_code" -D               | حالت ارزیابی عمیق (فقط مبتنی بر LLM، دقیق‌تر اما کندتر)  
evaluate -l "language_code" --save-logs, -s  | لاگ‌های DEBUG را در فایل‌هایی تحت مسیر <root_dir>/logs/ ذخیره می‌کند  
migrate-links -l "language_codes"             | پردازش مجدد فایل‌های Markdown ترجمه‌شده برای به‌روزرسانی لینک‌ها به دفترچه‌های یادداشت (.ipynb). ترجیح روی دفترچه‌های ترجمه‌شده است وگرنه به دفترچه‌های اصلی بازمی‌گردد.  
migrate-links -l "language_codes" -r          | پوشه ریشه پروژه را مشخص می‌کند (پیش‌فرض: پوشه جاری).  
migrate-links -l "language_codes" --dry-run   | نمایش فایل‌هایی که تغییر خواهند کرد بدون نوشتن تغییرات.  
migrate-links -l "language_codes" --no-fallback-to-original | هنگام نبود نسخه‌های ترجمه‌شده، لینک‌ها به دفترچه‌های اصلی بازنویسی نشوند (فقط هنگام وجود نسخه ترجمه‌شده به‌روزرسانی می‌شود).  
migrate-links -l "language_codes" -d          | حالت اشکال‌زدایی برای ثبت لاگ دقیق فعال می‌کند.  
migrate-links -l "language_codes" --save-logs, -s | لاگ‌های DEBUG را در فایل‌هایی تحت مسیر <root_dir>/logs/ ذخیره می‌کند  
migrate-links -l "all" -y                      | پردازش همه زبان‌ها و تایید خودکار درخواست هشدار.

## نمونه‌های استفاده

  1. رفتار پیش‌فرض (افزودن ترجمه‌های جدید بدون حذف ترجمه‌های موجود):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. افزودن فقط ترجمه‌های تصویری کره‌ای جدید (بدون حذف ترجمه‌های موجود):    translate -l "ko" -img

  3. به‌روز کردن همه ترجمه‌های کره‌ای (هشدار: همه ترجمه‌های کره‌ای موجود حذف و سپس دوباره ترجمه می‌شوند):    translate -l "ko" -u

  4. به‌روز کردن فقط تصاویر کره‌ای (هشدار: همه تصاویر کره‌ای موجود حذف و دوباره ترجمه می‌شوند):    translate -l "ko" -img -u

  5. افزودن ترجمه‌های Markdown جدید برای کره‌ای بدون تأثیر روی ترجمه‌های دیگر:    translate -l "ko" -md

  6. اصلاح ترجمه‌های با اطمینان پایین بر اساس نتایج ارزیابی قبلی: translate -l "ko" --fix

  7. اصلاح ترجمه‌های با اطمینان پایین فقط برای فایل‌های مشخص (Markdown): translate -l "ko" --fix -md

  8. اصلاح ترجمه‌های با اطمینان پایین فقط برای فایل‌های مشخص (تصاویر): translate -l "ko" --fix -img

  9. استفاده از حالت سریع برای ترجمه تصاویر:    translate -l "ko" -img -f

  10. اصلاح ترجمه‌های با اطمینان پایین با آستانه سفارشی: translate -l "ko" --fix -c 0.8

  11. نمونه حالت اشکال‌زدایی: - translate -l "ko" -d: فعال‌سازی ثبت لاگ اشکال‌زدایی.  
  12. ذخیره لاگ‌ها در فایل‌ها: translate -l "ko" -s  
  13. ثبت لاگ اشکال‌زدایی در کنسول و فایل: translate -l "ko" -d -s  
  14. ترجمه بدون افزودن بخش افشای مسئولیت ترجمه ماشینی: translate -l "ko" --no-disclaimer

  15. مهاجرت لینک‌های دفترچه یادداشت برای ترجمه کره‌ای (به‌روزرسانی لینک‌ها به دفترچه‌های ترجمه شده در صورت وجود):    migrate-links -l "ko"

  15. مهاجرت لینک‌ها با اجرای آزمایشی (بدون نوشتن فایل):    migrate-links -l "ko" --dry-run

  16. فقط زمانی لینک‌ها به‌روزرسانی شوند که دفترچه‌های ترجمه‌شده موجود باشند (بازگشت به اصلی صورت نگیرد):    migrate-links -l "ko" --no-fallback-to-original

  17. پردازش همه زبان‌ها با درخواست تایید:    migrate-links -l "all"

  18. پردازش همه زبان‌ها با تایید خودکار:    migrate-links -l "all" -y  
  19. ذخیره لاگ‌ها در فایل‌ها برای migrate-links:    migrate-links -l "ko ja" -s

  20. شخصی‌سازی راهنمای زبان‌های README با URL مخزن شما:  
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### نمونه‌های ارزیابی

> [!WARNING]  
> **ویژگی بتا**: عملکرد ارزیابی در حال حاضر در مرحله بتا است. این ویژگی برای ارزیابی اسناد ترجمه‌شده ارائه شده و روش‌ها و جزئیات پیاده‌سازی آن همچنان در حال توسعه و ممکن است تغییر کنند.

  1. ارزیابی ترجمه‌های کره‌ای: evaluate -l "ko"

  2. ارزیابی با آستانه اطمینان سفارشی: evaluate -l "ko" -c 0.8

  3. ارزیابی سریع (فقط مبتنی بر قوانین): evaluate -l "ko" -f

  4. ارزیابی عمیق (فقط مبتنی بر LLM): evaluate -l "ko" -D
