## Security

ไมโครซอฟท์ให้ความสำคัญกับความปลอดภัยของผลิตภัณฑ์ซอฟต์แวร์และบริการของเราอย่างจริงจัง ซึ่งรวมถึงแหล่งเก็บซอร์สโค้ดทั้งหมดที่จัดการผ่านองค์กร GitHub ของเรา ซึ่งได้แก่ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) และ [Xamarin](https://github.com/xamarin)

หากคุณเชื่อว่าคุณพบช่องโหว่ด้านความปลอดภัยในแหล่งเก็บข้อมูลที่เป็นของไมโครซอฟท์ซึ่งตรงตาม [คำนิยามของไมโครซอฟท์เกี่ยวกับช่องโหว่ด้านความปลอดภัย](https://aka.ms/security.md/definition) โปรดรายงานให้เราทราบตามที่อธิบายไว้ด้านล่าง

## Reporting Security Issues

**โปรดอย่ารายงานช่องโหว่ด้านความปลอดภัยผ่าน GitHub issues สาธารณะ**

แต่โปรดรายงานไปยัง Microsoft Security Response Center (MSRC) ที่ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)

ถ้าคุณต้องการส่งรายงานโดยไม่ต้องเข้าสู่ระบบ ให้ส่งอีเมลไปที่ [secure@microsoft.com](mailto:secure@microsoft.com) หากเป็นไปได้ กรุณาเข้ารหัสข้อความของคุณด้วยกุญแจ PGP ของเรา; ดาวน์โหลดได้จาก [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp)

คุณควรได้รับการตอบกลับภายใน 24 ชั่วโมง หากด้วยเหตุผลใดก็ตามที่คุณไม่ได้รับกรุณาติดตามผลทางอีเมลเพื่อยืนยันว่าเราได้รับข้อความฉบับแรกของคุณ ข้อมูลเพิ่มเติมสามารถดูได้ที่ [microsoft.com/msrc](https://www.microsoft.com/msrc)

กรุณารวมข้อมูลที่ร้องขอด้านล่างนี้ (มากเท่าที่คุณสามารถให้ได้) เพื่อช่วยให้เราเข้าใจธรรมชาติและขอบเขตของปัญหาที่เป็นไปได้ได้ดีขึ้น:

  * ประเภทของปัญหา (เช่น buffer overflow, SQL injection, cross-site scripting ฯลฯ)
  * เส้นทางเต็มของไฟล์ซอร์สโค้ดที่เกี่ยวข้องกับการแสดงออกของปัญหา
  * ตำแหน่งของซอร์สโค้ดที่ได้รับผลกระทบ (tag/branch/commit หรือ URL โดยตรง)
  * การตั้งค่าพิเศษใด ๆ ที่จำเป็นเพื่อจำลองปัญหา
  * คำแนะนำทีละขั้นตอนเพื่อจำลองปัญหา
  * โค้ด proof-of-concept หรือ exploit (ถ้าเป็นไปได้)
  * ผลกระทบของปัญหา รวมถึงวิธีที่ผู้โจมตีอาจใช้ประโยชน์จากปัญหา

ข้อมูลนี้จะช่วยให้เราแยกประเภทและจัดการรายงานของคุณได้รวดเร็วขึ้น

หากคุณรายงานสำหรับโครงการ bug bounty รายงานที่สมบูรณ์มากขึ้นสามารถช่วยให้ได้รับรางวัล bounty สูงขึ้นได้ กรุณาเยี่ยมชม [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) เพื่อดูรายละเอียดเพิ่มเติมเกี่ยวกับโครงการที่กำลังดำเนินอยู่ของเรา
