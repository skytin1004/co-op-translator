## 安全性

微軟非常重視我們軟體產品和服務的安全性，其中包括所有由我們的 GitHub 組織管理的原始碼庫，這些組織包括 [Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet) 及 [Xamarin](https://github.com/xamarin)。

如果您認為您已在任何微軟擁有的原始碼庫中發現了符合 [微軟安全漏洞定義](https://aka.ms/security.md/definition) 的安全漏洞，請依下述方式向我們回報。

## 回報安全問題

**請勿透過公開的 GitHub issue 回報安全漏洞。**

請改為透過微軟安全回應中心 (MSRC) 於 [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) 回報。

若您不願登入提交，請寄信至 [secure@microsoft.com](mailto:secure@microsoft.com)。若可能，請使用我們的 PGP 金鑰加密您的郵件；您可從 [微軟安全回應中心 PGP 金鑰頁面](https://aka.ms/security.md/msrc/pgp) 下載。

您應會在 24 小時內收到回覆。如未收到回覆，請透過電子郵件跟進，確認我們已收到您的原始訊息。更多資訊請參考 [microsoft.com/msrc](https://www.microsoft.com/msrc)。

請盡量提供下列資料，以協助我們更清楚了解可能問題的性質和範圍：

  * 問題類型（例如緩衝區溢位、SQL 注入、跨站腳本等）
  * 與問題現象相關的原始檔完整路徑
  * 受影響原始碼的位置（標籤/分支/提交或直接 URL）
  * 任何重現問題所需的特殊設定
  * 重現問題的逐步說明
  * 概念驗證或利用程式碼（如果可能）
  * 問題的影響，包括攻擊者可能如何利用該問題

此資訊將協助我們更快進行問題分類。

若您是進行漏洞賞金回報，更完整的回報將有助於獲得較高的獎金。請參閱我們的 [微軟漏洞賞金計畫](https://aka.ms/security.md/msrc/bounty) 頁面以了解更多活動計畫細節。
