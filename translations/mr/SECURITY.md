## Security

Microsoft आपल्या सॉफ्टवेअर उत्पादने आणि सेवा सुरक्षित ठेवण्यासाठी गंभीरपणे घेतो, ज्यामध्ये आमच्या GitHub संस्थांच्या माध्यमातून व्यवस्थापित सर्व स्रोत कोड संग्रहालये समाविष्ट आहेत, ज्यात [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) आणि [Xamarin](https://github.com/xamarin) यांचा समावेश आहे.

जर तुम्हाला अशी वाटत असेल की तुम्ही कोणत्याही Microsoft-स्वामित्वाच्या संग्रहालयात सुरक्षा धोका सापडला आहे जो [Microsoft च्या सुरक्षा धोका परिभाषेनुसार](https://aka.ms/security.md/definition) येतो, तर कृपया खाली दिलेल्या मार्गदर्शनानुसार आम्हाला कळवा.

## सुरक्षा समस्या कशी नोंदवायची

**कृपया सार्वजनिक GitHub प्रकरणांद्वारे सुरक्षा धोके नोंदवू नका.**

त्याऐवजी, कृपया Microsoft Security Response Center (MSRC) येथे [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) ला नोंदवा.

जर तुम्हाला लॉगिन न करता नोंदवायचे असेल, तर कृपया [secure@microsoft.com](mailto:secure@microsoft.com) यावर ईमेल पाठवा. शक्य असल्यास, आमच्या PGP कीसह तुमचा संदेश एन्क्रिप्ट करा; कृपया [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) वरून तो डाउनलोड करा.

तुम्हाला 24 तासांत प्रतिसाद मिळेल अशी अपेक्षा आहे. जर कोणत्याही कारणाने प्रतिसाद न मिळाला, तर कृपया आम्हाला तुमचा मूळ संदेश प्राप्त झाला आहे याची खात्री करण्यासाठी ईमेलद्वारे पाठपुरावा करा. अधिक माहिती [microsoft.com/msrc](https://www.microsoft.com/msrc) येथे उपलब्ध आहे.

कृपया खालील (जास्तीत जास्त शक्य असेल तेवढी) माहिती समाविष्ट करा ज्यामुळे आम्हाला संभाव्य समस्येचे स्वरूप आणि व्याप्ती अधिक चांगल्या प्रकारे समजेल:

  * समस्येचा प्रकार (उदा. बफर ओव्हरफ्लो, SQL इंजेक्शन, क्रॉस-साइट स्क्रिप्टिंग, इत्यादी)
  * संबंधित स्रोत फाइल(च्या) पूर्ण मार्ग
  * प्रभावित स्रोत कोडचा स्थान (टॅग/ब्रँच/कमिट किंवा थेट URL)
  * समस्या पुन्हा निर्माण करण्यासाठी आवश्यक कोणतेही विशेष कॉन्फिगरेशन
  * समस्या पुन्हा निर्माण करण्यासाठी पाव-या-पाव-या सूचना
  * पुरावा-कोन्सेप्ट किंवा एक्सप्लॉइट कोड (जर शक्य असेल तर)
  * समस्येचा परिणाम, ज्यामध्ये हल्लेखोर कसा एक्सप्लॉइट करू शकतो

ही माहिती आम्हाला तुमचा अहवाल जलद तपासण्यासाठी मदत करेल.

जर तुम्ही बग बाउंटीसाठी अहवाल करत असाल, तर अधिक संपूर्ण अहवाल मोठ्या बक्षीस देण्यास योगदान करू शकतात. कृपया आमच्या [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) पृष्ठावर आमच्या सक्रिय कार्यक्रमांविषयी अधिक तपशील पहा.
