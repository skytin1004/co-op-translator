# User Guide: Translation and Upload to Localizeflow

This article explains the procedure for translating and uploading local markdown work to Localizeflow.

## Preparation

1. You need a GitHub account.
2. When working on documents in your own repository, write your translations normally.
3. For those without repository permissions working on Pull Requests (PRs), use the GitHub App integrated with Localizeflow to edit translations.
4. The basic workflow is as required for the minimum translation unit (such as sentence, paragraph, or message).

## Translating and Uploading in Your Own Repository

When translating within your own repository:

- Your Pull Request involves your translation work.
- Use your environment to translate in files and commit changes.
- After committing, the Localizeflow GitHub App will automatically detect the update.
- The latest translation is then reflected.

## Translating and Uploading in a Different Repository via PR and the GitHub App

For those who do not have write permissions but are working on PRs:

- Use the GitHub App UI provided by Localizeflow.
- Browse the translation view from the Pull Request.
- You can directly edit, save, and add comments on translations.
- This allows easy contribution without cloning or local operations.

## Localizeflow GitHub App Features for PR Editing

The GitHub App allows:

- Inline editing of translations.
- Suggestion and comment features.
- Validation and detection of untranslated or inconsistent parts.
- Status tracking of translation progress.

## Notes

- Please keep the Markdown structure and variable placeholders intact while translating.
- For large translation tasks, coordinate with repository administrators beforehand.
- If you encounter any issues, feel free to open an issue or contact the repository maintainers.
