## భద్రత

Microsoft మా సాఫ్ట్‌వేర్ ఉత్పత్తులు మరియు సేవల భద్రతను సీరియస్‌గా తీసుకుంటుంది, దీనిలో మన GitHub సంస్థల ద్వారా నిర్వహించబడిన అన్ని సోర్స్ కోడ్ రిపాజిటరీలు ఉన్నాయి, ఇవి [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) మరియు [Xamarin](https://github.com/xamarin)ను కలిగి ఉంటాయి.

మీరు [Microsoft భద్రతా లోపం నిర్వచనం](https://aka.ms/security.md/definition) కలిగిన Microsoft-సొంత రిపాజిటరీలో భద్రతా లోపం గుర్తించినట్లు నమ్మితే, దయచేసి దిగువ వివరించబడిన విధంగా మాకు నివేదించండి.

## భద్రతా సమస్యలను నివేదించడం

**దయచేసి భద్రతా లోపాలను పబ్లిక్ GitHub ఇష్యూల ద్వారా నివేదించవద్దు.**

అదికారికంగా, దయచేసి వాటిని Microsoft Security Response Center (MSRC) వద్ద [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)లో నివేదించండి.

నువ్వు లాగిన్ చేయకుండా సమర్పించాలనుకుంటే, [secure@microsoft.com](mailto:secure@microsoft.com) కు ఇమెయిల్ పంపండి. సాద్యమైతే, దయచేసి మా PGP కీతో మీ సందేశాన్ని ఎన్క్రిప్ట్ చేయండి; దానిని [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) నుండి డౌన్‌లోడ్ చేసుకోండి.

మీకు 24 గంటల్లో ఒక ప్రతిస్పందన వస్తుందని ఆశించవచ్చు. ఏదైనా కారణంగా ప్రతిస్పందన రాకపోతే, మేము మీ అసలు సందేశం అందుకున్నామో లేదో నిర్ధారించుకునేందుకు ఇమెయిల్ ద్వారా ఫాలో అప్లో చేయండి. అదనపు సమాచారం [microsoft.com/msrc](https://www.microsoft.com/msrc) లో లభిస్తుంది.

దయచేసి క్రింద సూచించిన సమాచారాన్ని (మీరు అందగలిగినంత వరకు) సహా పంపండి, మనకి ఈ సమస్య స్వభావం మరియు పరిధిని బాగా అర్థం చేసుకోవడం కోసం:

  * సమస్య రకం (ఉదాహరణకు బఫర్ ఓవర్ఫ్లో, SQL ఇంజెక్షన్, క్రాస్-సైట్ స్క్రిప్టింగ్, మొదలైనవి)
  * సమస్య ఉత్పన్నమైన సోర్స్ ఫైల్(లు) యొక్క పూర్తి మార్గాలు
  * ప్రభావిత సోర్స్ కోడ్ ఉన్న స్థలం (ట్యాగ్/బ్రాంచ్/కమిట్ లేదా డైరెక్ట్ URL)
  * సమస్య పునరావృతం చేయడానికి అవసరమైన ప్రత్యేక కాన్ఫిగరేషన్
  * సమస్య పునరావృతం చేయడానికి దశల వారీ సూచనలు
  * ప్రూఫ్-ఆఫ్-కాన్సెప్ట్ లేదా ఎక్స్‌ప్లాయిట్ కోడ్ (సాధ్యమైతే)
  * ఈ సమస్య ప్రభావం, దాని ద్వారా ఒక దాడి యజమాని ఎలా దాన్ని ఉపయోగించవచ్చు

ఈ సమాచారం మీ నివేదికను త్వరగా పరిశీలించడంలో సహాయపడుతుంది.

మీరు బగ్ బౌంటీ కోసం నివేదిస్తున్నట్లయితే, మరింత పూర్తి నివేదికలు ఎక్కువ బౌంటీ అవార్డు పొందడానికి సహాయపడతాయి. మన [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) పేజీని సందర్శించండి మా క్రియాశీల కార్యక్రమాల గురించి మరింత వివరాలకు.

## ప్రాధాన్యతాత్మక భాషలు

మేము అన్ని సంభాషణలు ఇంగ్లీష్ లోనే ఉండాలని ఇష్టపడతాము.

## విధానం

Microsoft [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) సూత్రాన్ని అనుసరిస్తుంది.
