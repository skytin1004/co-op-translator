# కమాండ్ రిఫరెన్స్

**Co-op Translator** CLI అనువాద ప్రక్రియను అనుకూలీకరించడానికి అనేక ఎంపికలను అందిస్తుంది:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | మీ ప్రాజెక్టును పేర్కొన్న భాషలలోకి అనువదిస్తుంది. ఉదాహరణ: translate -l "es fr de" స్పానిష్, ఫ్రెంచ్ మరియు జర్మన్‌కు అనువదిస్తుంది. అన్ని మద్దతు ఉన్న భాషలలోకి అనువదించడానికి translate -l "all" ఉపయోగించండి.
translate -l "language_codes" -u              | గత అనువాదాలను తొలగించి వాటిని తిరిగి సృష్టించడం ద్వారా అనువాదాలను అప్డేట్ చేస్తుంది. హెచ్చరిక: ఇది పేర్కొన్న భాషల యొక్క అన్ని ప్రస్తుత అనువాదాలను తొలగిస్తుంది.
translate -l "language_codes" -img            | కేవలం ఇమేజ్ ఫైళ్లను మాత్రమే అనువదిస్తుంది.
translate -l "language_codes" -md             | కేవలం మార్క్‌డౌన్ ఫైళ్లను మాత్రమే అనువదిస్తుంది.
translate -l "language_codes" -nb             | కేవలం జుపైటర్ నోట్‌బుక్ ఫైళ్లను (.ipynb) మాత్రమే అనువదిస్తుంది.
translate -l "language_codes" --fix           | గత మూల్యాంకన ఫలితాల ఆధారంగా తక్కువ విశ్వాసం స్కోర్లు కలిగిన ఫైళ్లను తిరిగి అనువదిస్తుంది.
translate -l "language_codes" -d              | విస్తృతమైన లాగింగ్ కోసం డీబగ్ మోడ్‌ను ప్రారంభిస్తుంది.
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ కింద DEBUG-స్థాయి లాగ్స్‌ను ఫైళ్లలో సేవ్ చేస్తుంది (కన్సోల్ -d ద్వారా నియంత్రించబడుతుంది)
translate -l "language_codes" -ר "root_dir"   | ప్రాజెక్ట్ యొక్క రూట్ డైరెక్టరీని పేర్కొంటుంది
translate -l "language_codes" -f              | ఇమేజ్ అనువాదానికి ఫాస్ట్ మోడ్ ఉపయోగిస్తుంది (మంచి త్రుటి కొంత తక్కువ స్థాయిలో, తగ్గిన నాణ్యత మరియు అలైమెంట్ వద్ద 3 రెట్లు వేగంగా ప్లాట్ చేస్తుంది).
translate -l "language_codes" -y              | అన్ని ప్రాంప్ట్‌లను స్వయంచాలకంగా నిర్ధారిస్తుంది (CI/CD పైప్లైన్లకు ఉపయోగకరంగా)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | అనువదించిన మార్క్‌డౌన్ మరియు నోట్‌బుక్స్‌కు యంత్ర అనువాదం వికృతిని చేర్చడం ప్రారంభించడానికి లేదా నిలిపివేయడానికి (డిఫాల్ట్: ప్రారంభించబడింది).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | README భాషల విభాగ సలహాను మీ రిపోజిటరీ URL తో వ్యక్తిగతీకరించండి (స్పార్స్ చెకౌట్).
translate -l "language_codes" --help          | CLIలో అందుబాటులో ఉన్న కమాండ్ల సహాయం వివరాలు చూపిస్తుంది
evaluate -l "language_code"                  | ఒక నిర్దిష్ట భాష కోసం అనువాద నాణ్యతను అంచనా వేస్తుంది మరియు విశ్వాసం స్కోర్లు అందిస్తుంది
evaluate -l "language_code" -c 0.8           | కస్టమ్ విశ్వాస పరిమితితో అనువాదాలను అంచనా వేస్తుంది
evaluate -l "language_code" -f               | వేగవంతమైన అంచనా మోడు (నియమాధారితమే, LLM లేదు)
evaluate -l "language_code" -D               | లోతైన అంచనా మోడు (LLM ఆధారంగా మాత్రమే, మరింత పూర్తిగా కానీ నెమ్మదిగా)
evaluate -l "language_code" --save-logs, -s  | <root_dir>/logs/ కింద DEBUG-స్థాయి లాగ్స్‌ను ఫైళ్లలో సేవ్ చేస్తుంది
migrate-links -l "language_codes"             | అనువదించిన మార్క్‌డౌన్ ఫైళ్లను తిరిగి ప్రాసెస్ చేసి నోట్‌బుక్స్ (.ipynb) కు లింక్స్‌ను నవీకరించండి. అనువదించబడ్డ నోట్‌బుక్స్ అందుబాటులో ఉన్నప్పుడు అవి ప్రాధాన్యమిస్తుంది; లేకపోతే అసలు నోట్‌బుక్స్ fallback చేయవచ్చు.
migrate-links -l "language_codes" -r          | ప్రాజెక్ట్ రూట్ డైరెక్టరీని పేర్కొనండి (డిఫాల్ట్: ప్రస్తుత డైరెక్టరీ).
migrate-links -l "language_codes" --dry-run   | ఎటువంటి ఫైల్ రాయకుండా ఎటువంటి ఫైళ్లు మార్చబడుతాయో చూపిస్తుంది.
migrate-links -l "language_codes" --no-fallback-to-original | అనువదించబడిన కాపీలు లేకపోతే అసలు నోట్‌బుక్స్‌కు లింక్స్‌ను మళ్లీ రాయవద్దు (కేవలం అనువదించినవి ఉన్నప్పుడు మాత్రమే నవీకరించండి).
migrate-links -l "language_codes" -d          | విస్తృతమైన లాగింగ్ కోసం డీబగ్ మోడ్ ప్రారంభిస్తుంది.
migrate-links -l "language_codes" --save-logs, -s | <root_dir>/logs/ కింద DEBUG-స్థాయి లాగ్స్‌ను ఫైళ్లలో సేవ్ చేస్తుంది
migrate-links -l "all" -y                      | అన్ని భాషలలో ప్రాసెస్ చేసి హెచ్చరిక ప్రాంప్ట్‌ను స్వయంచాలకంగా నిర్ధారిస్తుంది.

## ఉపయోగించుట ఉదాహరణలు

  1. డిఫాల్ట్ ప్రవర్తన (గత అనువాదాలను తొలగించకుండా కొత్త అనువాదాలను జోడించండి):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. కేవలం కొత్త కొరియన్ ఇమేజ్ అనువాదాలను జోడించండి (గత అనువాదాలు తొలగించబడవు):    translate -l "ko" -img

  3. అన్ని కొరియన్ అనువాదాలను అప్డేట్ చేయండి (హెచ్చరిక: ఇది తిరిగి అనువదించే ముందు అన్ని గత కొరియన్ అనువాదాలను తొలగిస్తుంది):    translate -l "ko" -u

  4. కేవలం కొరియన్ చిత్రాలను మాత్రమే అప్డేట్ చేయండి (హెచ్చరిక: ఇది తిరిగి అనువదించే ముందు అన్ని గత కొరియన్ చిత్రాలను తొలగిస్తుంది):    translate -l "ko" -img -u

  5. ఇతర అనువాదాలను ప్రభావితం చేయకుండా కొరియన్ మార్క్‌డౌన్ అనువాదాలను మాత్రమే జోడించండి:    translate -l "ko" -md

  6. గత మూల్యాంకన ఫలితాల ఆధారంగా తక్కువ విశ్వాసం అనువాదాలను సరిచేయండి: translate -l "ko" --fix

  7. కేవలం నిర్దిష్ట ఫైళ్ల కోసం తక్కువ విశ్వాసం అనువాదాలను సరిచేయండి (మార్క్‌డౌన్): translate -l "ko" --fix -md

  8. కేవలం నిర్దిష్ట ఫైళ్ల కోసం తక్కువ విశ్వాసం అనువాదాలను సరిచేయండి (చిత్రాలు): translate -l "ko" --fix -img

  9. ఇమేజ్ అనువాదానికి ఫాస్ట్ మోడ్ ఉపయోగించండి:    translate -l "ko" -img -f

  10. కస్టమ్ పరిమితితో తక్కువ విశ్వాసం అనువాదాలను సరిచేయండి: translate -l "ko" --fix -c 0.8

  11. డీబగ్ మోడ్ ఉదాహరణలు: - translate -l "ko" -d: డీబగ్ లాగింగ్ ప్రారంభించండి.
  12. ఫైళ్లలో లాగ్‌లను సేవ్ చేయండి: translate -l "ko" -s
  13. కన్సోల్ DEBUG మరియు ఫైల్ DEBUG: translate -l "ko" -d -s
  14. అవుట్‌పుట్‌లకు యంత్ర అనువాద వికృతి చేర్చకుండా అనువదించండి: translate -l "ko" --no-disclaimer

  15. కొరియన్ అనువాదాలకు నోట్‌బుక్ లింక్స్ మార్చండి (అనువదించిన నోట్‌బుక్స్ ఉన్నప్పుడు లింక్స్ నవీకరించండి):    migrate-links -l "ko"

  15. డ్రై-రన్‌తో లింక్స్ మార్చండి (ఫైల్ రాయడం లేదు):    migrate-links -l "ko" --dry-run

  16. అనువదించిన నోట్‌బుక్స్ ఉన్నప్పుడు మాత్రమే లింక్స్ నవీకరించండి (అసలు వాటికి fallback చేయవద్దు):    migrate-links -l "ko" --no-fallback-to-original

  17. నిర్ధారణ ప్రాంప్ట్‌తో అన్ని భాషలను ప్రాసెస్ చేయండి:    migrate-links -l "all"

  18. అన్ని భాషలను ప్రాసెస్ చేసి ఆటో-నిర్ధారణ చేయండి:    migrate-links -l "all" -y
  19. migrate-links కోసం ఫైళ్లకు లాగ్స్ సేవ్ చేయండి:    migrate-links -l "ko ja" -s

  20. README భాషల సలహాను మీ రిపో URL తో వ్యక్తిగతీకరించండి:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### మూల్యాంకన ఉదాహరణలు

> [!WARNING]  
> **బీటా ఫీచర్**: మూల్యాంకన ఫంక్షనాలిటీ ప్రస్తుతం బీటాలో ఉంది. ఈ ఫీచర్ అనువదించిన డాక్యుమెంట్లను అంచనా వేయడానికి విడుదల చేయబడింది, మరియు మూల్యాంకన విధానాలు మరియు విస్తృత అమలుపై ఇంకా అభివృద్ధి జరుగుతున్నది, మార్పులకు లోనవవచ్చు.

  1. కొరియన్ అనువాదాలను మూల్యాంకనం చేయండి: evaluate -l "ko"

  2. కస్టమ్ విశ్వాస పరిమితితో మూల్యాంకనం చేయండి: evaluate -l "ko" -c 0.8

  3. వేగవంతమైన మూల్యాంకనం (నియమాధారితమే): evaluate -l "ko" -f

  4. లోతైన మూల్యాంకనం (LLM ఆధారంగా): evaluate -l "ko" -D
