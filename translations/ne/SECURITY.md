## सुरक्षा

Microsoft हाम्रो सफ्टवेयर उत्पादनहरू र सेवाहरूको सुरक्षा गम्भीरतापूर्वक लिन्छ, जसमा हाम्रा GitHub संगठनहरू मार्फत व्यवस्थापन गरिने सबै स्रोत कोड रिपोजिटोरीहरू समावेश छन्, जसमा [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) र [Xamarin](https://github.com/xamarin) समावेश छन्।

यदि तपाईंले कुनै Microsoft-स्वामित्वको रिपोजिटोरीमा सुरक्षा भुल भेट्टाउनु भएको छ जुन [Microsoft को सुरक्षा भुलको परिभाषा](https://aka.ms/security.md/definition) पूरा गर्छ भने, कृपया तल वर्णन गरिएको अनुसार हामीलाई रिपोर्ट गर्नुहोस्।

## सुरक्षा मुद्दा रिपोर्टिङ

**कृपया सार्वजनिक GitHub इश्यूहरू मार्फत सुरक्षा भुलहरू रिपोर्ट नगर्नुहोस्।**

यसका सट्टा, कृपया Microsoft Security Response Center (MSRC) मा [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) मा रिपोर्ट गर्नुहोस्।

यदि लगइन नगरी बुझाउन चाहनुहुन्छ भने, [secure@microsoft.com](mailto:secure@microsoft.com) मा इमेल पठाउनुहोस्। सकेसम्म, हाम्रो PGP कुञ्जीको प्रयोग गरेर तपाइँको सन्देशलाई इन्क्रिप्ट गर्नुहोस्; कृपया यसलाई [Microsoft Security Response Center PGP Key पृष्ठबाट](https://aka.ms/security.md/msrc/pgp) डाउनलोड गर्नुहोस्।

तपाईं २४ घ hora भित्र जवाफ प्राप्त गर्नु पर्छ। कुनै कारणले नपाएमा, कृपया इमेल मार्फत पुनः सम्पर्क गर्नुहोस् र हामीले तपाईंको मूल सन्देश प्राप्त गर्‍यौं भनी सुनिश्चित गर्नुहोस्। थप जानकारीहरू [microsoft.com/msrc](https://www.microsoft.com/msrc) मा फेला पार्न सकिन्छ।

कृपया तल सूचीबद्ध अनुरोध गरिएको जानकारी (जति सक्नुहुन्छ) समावेश गर्नुहोस् जसले हामीलाई सम्भावित मुद्दाको प्रकृति र दायरा बुझ्न मद्दत गर्नेछ:

  * मुद्दाको प्रकार (जस्तै बफर ओभरफ्लो, SQL इन्जेक्शन, क्रस-साइट स्क्रिप्टिङ आदि)
  * मुद्दाको प्रकटीकरणसँग सम्बन्धित स्रोत फाइल(हरू) को पूरै मार्गहरू
  * प्रभावित स्रोत कोडको स्थान (ट्याग/ब्रान्च/कमिट वा सिधा URL)
  * मुद्दा पुन: उत्पादन गर्न आवश्यक कुनै विशेष कन्फिगरेसन
  * मुद्दा पुन: उत्पादन गर्न चरण-द्वारा-चरण निर्देशहरू
  * प्रमाण-धारण वा शोषण कोड (यदि सम्भव छ भने)
  * मुद्दाको प्रभाव, जसमा कसरी एक आक्रमणकारीले मुद्दाको दुरुपयोग गर्न सक्छ

यो जानकारीले हामीलाई तपाईंको रिपोर्टलाई छिटो रूपमा मूल्याङ्कन गर्न मद्दत गर्नेछ।

यदि तपाईं बग बाउन्डीको लागि रिपोर्ट गर्दै हुनुहुन्छ भने, थप पूर्ण रिपोर्टहरू उच्च पुरस्कारमा योगदान गर्न सक्छन्। कृपया हाम्रो [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) पृष्ठमा हाम्रो सक्रिय कार्यक्रमहरूको बारेमा थप जानकारीको लागि भ्रमण गर्नुहोस्।
