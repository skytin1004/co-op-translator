## सुरक्षा

Microsoft हाम्रा सफ्टवेयर उत्पादन र सेवाहरूको सुरक्षा गम्भीर रूपमा लिन्छ, जसमा हाम्रा GitHub संगठनहरू मार्फत व्यवस्थापन गरिएका सबै सोर्स कोड रिपोजिटोरीहरू समावेश छन्, जसमा [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) र [Xamarin](https://github.com/xamarin) समावेश छन्।

यदि तपाईंले कुनै Microsoft-स्वामित्वको रिपोजिटोरीमा [Microsoft को सुरक्षा कमजोरीको परिभाषा](https://aka.ms/security.md/definition) अनुरूप सुरक्षा कमजोरी फेला पारेको विश्वास गर्नुहुन्छ भने, कृपया तल वर्णन गरिएको अनुसार हामीलाई रिपोर्ट गर्नुहोस्।

## सुरक्षा मुद्दा रिपोर्ट गर्ने तरिका

**कृपया सार्वजनिक GitHub समस्याहरू मार्फत सुरक्षा कमजोरीहरू रिपोर्ट नगर्नुहोस्।**

नत्र, कृपया तिनीहरूलाई Microsoft Security Response Center (MSRC) मा [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) मा रिपोर्ट गर्नुहोस्।

यदि तपाईं लगइन नगरी पेश गर्न चाहनुहुन्छ भने, इमेल पठाउनुहोस् [secure@microsoft.com](mailto:secure@microsoft.com)। यदि सम्भव छ भने, कृपया हाम्रो PGP कुञ्जी प्रयोग गरेर आफ्नो सन्देश इन्क्रिप्ट गर्नुहोस्; कृपया यसलाई [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) बाट डाउनलोड गर्नुहोस्।

तपाईंले २४ घण्टाभित्र जवाफ प्राप्त गर्नु पर्नेछ। यदि कुनै कारणले तपाईंले प्राप्त गर्नुभएन भने, कृपया हाम्रो मूल सन्देश प्राप्त भएको सुनिश्चित गर्न इमेल मार्फत अनुगमन गर्नुहोस्। थप जानकारी [microsoft.com/msrc](https://www.microsoft.com/msrc) मा फेला पार्न सकिन्छ।

कृपया तल सूचीबद्ध आवश्यक जानकारी समावेश गर्नुहोस् (जति सकिन्छ प्रदान गर्नुहोस्) जसले हामीलाई सम्भावित मुद्दाको प्रकृति र दायरालाई राम्रोसँग बुझ्न मद्दत गर्नेछ:

  * मुद्दाको प्रकार (उदाहरणका लागि, बफर ओभरफ्लो, SQL इन्जेक्शन, क्रस-साइट स्क्रिप्टिंग, आदि)
  * मुद्दा फेला परेको सोर्स फाइल(हरू)को पूर्ण पथहरू
  * प्रभावित सोर्स कोडको स्थान (ट्याग/ब्रान्च/क्रिप्ट वा सिधा URL)
  * मुद्दा पुन: उत्पन्न गर्न आवश्यक कुनै विशेष कन्फिगरेसन
  * मुद्दा पुन: उत्पन्न गर्न चरण-द्वारा-चरण निर्देशनहरू
  * प्रमाण-कोन्सेप्ट वा एक्स्प्लोइट कोड (यदि सम्भव छ भने)
  * मुद्दाको प्रभाव, जसमा कसरी आक्रमणकारीले यसलाई उपयोग गर्न सक्छ

यस जानकारीले हामीलाई तपाईंको रिपोर्टलाई छिटो ट्राइएज गर्न मद्दत गर्दछ।

यदि तपाईं बग बाउटीका लागि रिपोर्ट गर्दै हुनुहुन्छ भने, बढी पूर्ण रिपोर्टहरूले उच्च बाउटी पुरस्कार पाउन योगदान गर्न सक्छ। कृपया हाम्रा सक्रिय कार्यक्रममा थप विवरणका लागि [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) पृष्ठ भ्रमण गर्नुहोस्।
