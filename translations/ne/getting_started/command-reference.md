# Command reference

The **Co-op Translator** CLI ले अनुवाद प्रक्रियालाई अनुकूलन गर्न धेरै विकल्पहरू प्रदान गर्दछ:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | तपाइँको परियोजनालाई निर्दिष्ट भाषाहरूमा अनुवाद गर्दछ। उदाहरण: translate -l "es fr de" ले स्पेनी, फ्रेन्च, र जर्मनमा अनुवाद गर्छ। सबै समर्थित भाषाहरूमा अनुवाद गर्न translate -l "all" प्रयोग गर्नुहोस्।
translate -l "language_codes" -u              | अनुवादहरू अद्यावधिक गर्दछ, अवस्थित अनुवादहरू मेटाएर पुनः सिर्जना गर्छ। चेतावनी: यसले निर्दिष्ट भाषाहरूका लागि सबै वर्तमान अनुवादहरू मेटाउँछ।
translate -l "language_codes" -img            | केवल इमेज फाइलहरू अनुवाद गर्छ।
translate -l "language_codes" -md             | केवल मार्कडाउन फाइलहरू अनुवाद गर्छ।
translate -l "language_codes" -nb             | केवल जुुपिटर नोटबुक फाइलहरू (.ipynb) अनुवाद गर्छ।
translate -l "language_codes" --fix           | अघिल्लो मूल्यांकन नतिजाहरूको आधारमा न्यून आत्मविश्वास स्कोर भएका फाइलहरू पुनः अनुवाद गर्छ।
translate -l "language_codes" -d              | विस्तृत लगिङ्गको लागि डिबग मोड सक्षम गर्छ।
translate -l "language_codes" --save-logs, -s | DEBUG-स्तर लगहरू <root_dir>/logs/ अन्तर्गत फाइलहरूमा बचत गर्छ (कन्सोल -d द्वारा नियन्त्रित रहन्छ)
translate -l "language_codes" -r "root_dir"   | परियोजनाको रुट डाइरेक्टरी निर्दिष्ट गर्छ
translate -l "language_codes" -f              | इमेज अनुवादको लागि छिटो मोड प्रयोग गर्छ (थोरै गुणस्तर र समायोजनमा छिटो 3 गुणा प्लटिङ)
translate -l "language_codes" -y              | सबै प्रम्प्टहरू स्वचालित रूपमा पुष्टि गर्छ (CI/CD पाइपलाइनका लागि उपयोगी)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | अनुवाद गरिएको मार्कडाउन र नोटबुकमा मेसिन अनुवाद अस्वीकरण खण्ड थप्ने वा नथप्ने सक्षम/अक्षम गर्ने (पूर्वनिर्धारित: सक्रिय)।
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | तपाईँको रिपोजिटरी URL सहित README भाषाहरूको परामर्श अनुकूलन गर्ने (सङ्कुचित चेकआउट)।
translate -l "language_codes" --help          | CLI भित्र उपलब्ध आदेशहरूको मद्दत विवरण
evaluate -l "language_code"                  | एक विशिष्ट भाषाको अनुवाद गुणस्तर मूल्यांकन गर्छ र आत्मविश्वास स्कोरहरू प्रदान गर्छ
evaluate -l "language_code" -c 0.8           | अनुकूलित आत्मविश्वास सीमा साथ अनुवाद मूल्यांकन गर्छ
evaluate -l "language_code" -f               | छिटो मूल्यांकन मोड (नियम आधारित मात्र, LLM छैन)
evaluate -l "language_code" -D               | गहिरो मूल्यांकन मोड (LLM आधारित मात्र, थप गहिराई तर स्लो)
evaluate -l "language_code" --save-logs, -s  | DEBUG-स्तर लगहरू <root_dir>/logs/ मा फाइलहरूमा बचत गर्छ
migrate-links -l "language_codes"             | अनुवाद गरिएका मार्कडाउन फाइलहरू पुनः प्रक्रिया गरी नोटबुक (.ipynb) हरूमा लिंकहरू अपडेट गर्छ। उपलब्ध हुँदा अनुवादित नोटबुकहरू रोज्छ; होइन भने मूल नोटबुकहरूमा जान सक्छ।
migrate-links -l "language_codes" -r          | परियोजनाको रुट डाइरेक्टरी निर्दिष्ट गर्छ (पूर्वनिर्धारित: हालको डाइरेक्टरी)।
migrate-links -l "language_codes" --dry-run   | फाइलहरूमा लेख्ने बिना कुन फाइलहरू परिवर्तन हुने देखाउँछ।
migrate-links -l "language_codes" --no-fallback-to-original | अनुवादित नोटबुकहरू नभए मूल नोटबुकहरूमा लिंकहरू फेरि लेख्दैन (केवल अनुवादित हुँदा अपडेट गर्छ।
migrate-links -l "language_codes" -d          | विस्तृत लगिङ्गको लागि डिबग मोड सक्षम गर्छ।
migrate-links -l "language_codes" --save-logs, -s | DEBUG-स्तर लगहरू <root_dir>/logs/ मा फाइलहरूमा बचत गर्छ
migrate-links -l "all" -y                      | सबै भाषाहरू प्रक्रिया गर्छ र चेतावनी प्रम्प्ट स्वचालित पुष्टि गर्छ।

## Usage examples

  1. पूर्वनिर्धारित व्यवहार (अवस्थितहरू मेटाउँदा बिना नयाँ अनुवादहरू थप्ने):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. केवल नयाँ कोरियन इमेज अनुवादहरू थप्ने (अवस्थित अनुवादहरू मेटिँदैनन्):    translate -l "ko" -img

  3. सबै कोरियन अनुवादहरू अद्यावधिक गर्ने (चेतावनी: यसले पुनः अनुवाद गर्नु अघि सबै अवस्थित कोरियन अनुवादहरू मेटाउँछ):    translate -l "ko" -u

  4. मात्र कोरियन इमेजहरू अद्यावधिक गर्ने (चेतावनी: पुनः अनुवादअघि सबै अवस्थित कोरियन इमेजहरू मेटिन्छ):    translate -l "ko" -img -u

  5. अन्य अनुवादहरूलाई असर नगरी कोरियन मार्कडाउन नयाँ अनुवाद थप्ने:    translate -l "ko" -md

  6. अघिल्लो मूल्यांकन परिणामहरूमा आधारित न्यून आत्मविश्वास अनुवादहरू ठीक गर्ने: translate -l "ko" --fix

  7. विशिष्ट फाइलहरूका लागि मात्र (मार्कडाउन) न्यून आत्मविश्वास अनुवादहरू ठीक गर्ने: translate -l "ko" --fix -md

  8. विशिष्ट फाइलहरूका लागि मात्र (इमेजहरू) न्यून आत्मविश्वास अनुवादहरू ठीक गर्ने: translate -l "ko" --fix -img

  9. इमेज अनुवादको लागि छिटो मोड प्रयोग गर्ने:    translate -l "ko" -img -f

  10. अनुकूलित सीमा सहित न्यून आत्मविश्वास अनुवादहरू ठीक गर्ने: translate -l "ko" --fix -c 0.8

  11. डिबग मोड उदाहरण: - translate -l "ko" -d: डिबग लगिङ्ग सक्षम गर्ने।
  12. लगहरू फाइलमा बचत गर्ने: translate -l "ko" -s
  13. कन्सोल DEBUG र फाइल DEBUG: translate -l "ko" -d -s
  14. आउटपुटहरूमा मेसिन अनुवाद अस्वीकरण नथप्दै अनुवाद गर्ने: translate -l "ko" --no-disclaimer

  15. कोरियन अनुवादहरूको लागि नोटबुक लिंकहरू माईग्रेट गर्ने (अनुवादित नोटबुक उपलब्ध हुँदा लिंकहरू अपडेट गर्ने):    migrate-links -l "ko"

  15. ड्राइ-रन सहित लिंकहरू माईग्रेट गर्ने (फाइल लेख्ने बिना):    migrate-links -l "ko" --dry-run

  16. केवल अनुवादित नोटबुकहरू हुँदा लिंकहरू अपडेट गर्ने (मूलमा फर्कदैन):    migrate-links -l "ko" --no-fallback-to-original

  17. पुष्टि प्रम्प्ट सहित सबै भाषाहरू प्रक्रिया गर्ने:    migrate-links -l "all"

  18. सबै भाषाहरू प्रक्रिया गरी स्वचालित पुष्टि गर्ने:    migrate-links -l "all" -y
  19. migrate-links का लागि लगहरू फाइलमा बचत गर्ने:    migrate-links -l "ko ja" -s

  20. README भाषाहरूको परामर्श तपाइँको रिपो URL सहित निजीकृत गर्ने:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Evaluation Examples

> [!WARNING]  
> **Beta Feature**: मूल्यांकन कार्यक्षमता हाल बीटा मा छ। यो सुविधा अनुवादित दस्तावेजहरूको मूल्यांकन गर्न जारी गरिएको हो, र मूल्यांकन विधिहरू र विस्तृत कार्यान्वयन अझै विकासको अधीनमा छन् र परिवर्तन हुन सक्दछ।

  1. कोरियन अनुवाद मूल्यांकन गर्नुहोस्: evaluate -l "ko"

  2. अनुकूलित आत्मविश्वास सीमा सहित मूल्यांकन: evaluate -l "ko" -c 0.8

  3. छिटो मूल्यांकन (नियम आधारित मात्र): evaluate -l "ko" -f

  4. गहिरो मूल्यांकन (LLM आधारित मात्र): evaluate -l "ko" -D
