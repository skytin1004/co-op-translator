## Reporting Security Issues

**Abeg no dey report security vulnerabilities through public GitHub issues.**

Instead, abeg report dem to the Microsoft Security Response Center (MSRC) for [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

If you prefer to submit without logging in, send email to [secure@microsoft.com](mailto:secure@microsoft.com). If e fit, encrypt your message with our PGP key; abeg download am from the [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp).

You suppose receive response within 24 hours. If for any reason you no receive, abeg follow up via email to make sure say we don get your original message. More information dey for [microsoft.com/msrc](https://www.microsoft.com/msrc).

Abeg include the requested information wey dey below (as much as you fit provide) to help us understand better di nature and scope of the possible issue:

  * Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
  * Full paths of source file(s) wey relate to di manifestation of the issue
  * Di location of the affected source code (tag/branch/commit or direct URL)
  * Any special configuration wey you need to reproduce the issue
  * Step-by-step instructions to reproduce the issue
  * Proof-of-concept or exploit code (if e possible)
  * Impact of the issue, including how attacker fit take exploit the issue

This information go help us triage your report quick-quick.

If you dey report for bug bounty, more complete reports fit contribute to higher bounty award. Abeg visit our [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) page for more details about our active programs.
