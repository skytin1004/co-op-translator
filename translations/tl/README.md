## Overview

**Co-op Translator** ay tumutulong sa iyo na i-localize ang iyong pang-edukasyong nilalaman sa GitHub sa maraming wika nang madali.
Kapag na-update mo ang iyong mga Markdown file, mga imahe, o mga notebook, nananatiling awtomatikong naka-synchronize ang mga pagsasalin, tinitiyak na ang iyong nilalaman ay nananatiling tama at napapanahon para sa mga mag-aaral sa buong mundo.

Halimbawa kung paano naka-organisa ang naisaling nilalaman:

![Example](../../translated_images/tl/translation-ex.0c8aa6a7ee0aad2b.webp)

## Paano pinamamahalaan ang estado ng pagsasalin

Pinamamahalaan ng Co-op Translator ang naisaling nilalaman bilang **mga bersyonadong software artifacts**,  
hindi bilang mga static na file.

Tinutunton ng tool ang estado ng naisaling Markdown, mga imahe, at mga notebook
gamit ang **metadata na naka-scope sa wika**.

Pinapayagan ng disenyo na ito ang Co-op Translator na:

- Maaasahang makita ang mga lumang pagsasalin
- Tratuhin ng pare-pareho ang Markdown, mga imahe, at mga notebook
- Lumaki nang ligtas sa malalaki, mabilis na gumagalaw, multi-wika na mga repositoryo

Sa pagmomodelo sa mga pagsasalin bilang pinangangasiwaang mga artifact,
nagsusunod ang mga workflow ng pagsasalin nang natural sa makabagong
pamamahala ng mga dependency at artifact ng software.

→ [Paano pinamamahalaan ang estado ng pagsasalin](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)


## Quick start

```bash
# Lumikha at i-activate ang isang virtual na kapaligiran (inirerekomenda)
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
# I-install ang pakete
pip install co-op-translator
# Isalin
translate -l "ko ja fr" -md
```

Docker:

```bash
# Kunin ang pampublikong imahe mula sa GHCR
docker pull ghcr.io/azure/co-op-translator:latest
# Patakbuhin kasama ang kasalukuyang folder na naka-mount at ibinigay na .env (Bash/Zsh)
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## Minimal setup

1. Siguraduhin na mayroon kang suportadong bersyon ng Python (kasalukuyang 3.10-3.12). Sa poetry (pyproject.toml) awtomatikong ito ay hinahawakan.
2. Gumawa ng `.env` na file gamit ang template: [.env.template](../../.env.template)
3. Isaayos ang isang LLM provider (Azure OpenAI o OpenAI)
4. (Opsyonal) Para sa pagsasalin ng imahe (`-img`), isaayos ang Azure AI Vision
5. (Opsyonal) Maaari kang mag-configure ng maraming set ng kredensyal sa pamamagitan ng pag-duplicate ng mga variable na may mga suffix tulad ng `_1`, `_2`, atbp. Ang lahat ng variable sa isang set ay kailangang may parehong suffix.
6. (Inirerekomenda) Linisin ang anumang mga naunang pagsasalin upang maiwasan ang mga salungatan (hal., `translations/`)
7. (Inirerekomenda) Magdagdag ng seksyon ng pagsasalin sa iyong README gamit ang [README languages template](./getting_started/README_languages_template.md)
8. Tingnan: [Mag-setup ng Azure AI](./getting_started/set-up-azure-ai.md)

## Usage

Isalin ang lahat ng suportadong uri:

```bash
translate -l "ko ja"
```

Markdown lang:

```bash
translate -l "de" -md
```

Markdown + mga larawan:

```bash
translate -l "pt" -md -img
```

Mga notebook lang:

```bash
translate -l "zh" -nb
```

Iba pang mga flag: [Command reference](./getting_started/command-reference.md)

## Features

- Awtomatikong pagsasalin para sa Markdown, mga notebook, at mga larawan
- Pinananatiling naka-sync ang mga pagsasalin sa mga pagbabago sa pinagmulan
- Gumagana nang lokal (CLI) o sa CI (GitHub Actions)
- Gumagamit ng Azure OpenAI o OpenAI; opsyonal na Azure AI Vision para sa mga larawan
- Pinapanatili ang format at istraktura ng Markdown

## Responsible AI

Nakatuon ang Microsoft sa pagtulong sa aming mga customer na gamitin nang responsable ang aming mga AI na produkto, pagbabahagi ng aming mga natutunan, at pagtatayo ng mga partnership na nakabase sa tiwala sa pamamagitan ng mga tool tulad ng Transparency Notes at Impact Assessments. Marami sa mga mapagkukunan na ito ay matatagpuan sa [https://aka.ms/RAI](https://aka.ms/RAI).
Ang pamamaraan ng Microsoft sa responsible AI ay nakabatay sa aming mga prinsipyo ng AI na katarungan, pagiging maaasahan at kaligtasan, privacy at seguridad, inclusiveness, transparency, at accountability.

Ang mga malalaking modelo ng natural language, imahe, at pagsasalita - tulad ng mga ginamit sa sample na ito - ay posibleng kumilos sa mga paraan na hindi patas, hindi maaasahan, o nakasasakit, na maaaring magdulot ng pinsala. Mangyaring sumangguni sa [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) upang maging maalam tungkol sa mga panganib at limitasyon.

Ang inirerekomendang pamamaraan upang mabawasan ang mga panganib na ito ay ang pagsasama ng isang sistema ng kaligtasan sa iyong arkitektura na kayang tuklasin at pigilan ang mapanganib na pag-uugali. Nagbibigay ang [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) ng isang independiyenteng layer ng proteksyon, na kayang tuklasin ang mapanganib na content na gawa ng user at AI sa mga aplikasyon at serbisyo. Kasama sa Azure AI Content Safety ang text at image APIs na nagpapahintulot sa iyo na tuklasin ang mapanganib na materyal. Mayroon din kaming interactive Content Safety Studio na nagpapahintulot sa iyo na makita, tuklasin, at subukan ang sample code para sa pagtuklas ng mapanganib na content sa iba't ibang modality. Ang sumusunod na [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) ay gumagabay sa iyo sa paggawa ng mga kahilingan sa serbisyo.

Isa pang aspeto na dapat isaalang-alang ay ang pangkalahatang pagganap ng aplikasyon. Sa mga multi-modal at multi-model na aplikasyon, ang pagganap ay nangangahulugang gumaganap ang sistema ayon sa inaasahan mo at ng iyong mga user, kabilang ang hindi pagbuo ng mapanganib na output. Mahalaga na suriin ang pagganap ng iyong pangkalahatang aplikasyon gamit ang [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in).

Maaari mong suriin ang iyong AI na aplikasyon sa iyong development environment gamit ang [prompt flow SDK](https://microsoft.github.io/promptflow/index.html). Kapag may test dataset o target, ang mga henerasyon ng iyong generative AI application ay sinusukat nang kwantitatibo gamit ang built-in evaluators o custom evaluators na iyong pinili. Upang magsimula gamit ang prompt flow sdk para suriin ang iyong sistema, maaari mong sundan ang [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk). Kapag naisagawa mo na ang isang evaluation run, maaari mong [ipakita ang mga resulta sa Azure AI Studio](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).
