## Seguridad

Seryoso ang Microsoft sa seguridad ng aming mga produktong software at serbisyo, kabilang ang lahat ng source code repositories na pinamamahalaan sa pamamagitan ng aming mga organisasyong GitHub, na kinabibilangan ng [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) at [Xamarin](https://github.com/xamarin).

Kung naniniwala kang nakakita ka ng isang seguridad na kahinaan sa anumang repositoryong pag-aari ng Microsoft na tumutugma sa [depinisyon ng Microsoft ng seguridad na kahinaan](https://aka.ms/security.md/definition), pakisumite ito sa amin tulad ng nakasaad sa ibaba.

## Pag-uulat ng mga Isyu sa Seguridad

**Mangyaring huwag iulat ang mga seguridad na kahinaan sa pamamagitan ng pampublikong mga isyu sa GitHub.**

Sa halip, pakiulat ito sa Microsoft Security Response Center (MSRC) sa [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

Kung mas gusto mong magsumite nang hindi nagla-login, magpadala ng email sa [secure@microsoft.com](mailto:secure@microsoft.com). Kung maaari, i-encrypt ang iyong mensahe gamit ang aming PGP key; pakida-download ito mula sa [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp).

Dapat kang makatanggap ng tugon sa loob ng 24 na oras. Kung sakaling hindi mo ito matanggap, mangyaring mag-follow up sa pamamagitan ng email upang matiyak na natanggap namin ang iyong orihinal na mensahe. Karagdagang impormasyon ay matatagpuan sa [microsoft.com/msrc](https://www.microsoft.com/msrc).

Pakiusap na isama ang hinihiling na impormasyon na nakalista sa ibaba (hanggang sa makakaya mong ibigay) upang matulungan kaming mas maunawaan ang kalikasan at lawak ng posibleng isyu:

  * Uri ng isyu (hal. buffer overflow, SQL injection, cross-site scripting, atbp.)
  * Buong mga landas ng source file(s) na may kaugnayan sa pagpapakita ng isyu
  * Ang lokasyon ng apektadong source code (tag/branch/commit o direktang URL)
  * Anumang espesyal na konfigurasyon na kinakailangan upang maulit ang isyu
  * Hakbang-hakbang na mga tagubilin para maulit ang isyu
  * Proof-of-concept o exploit code (kung maaari)
  * Epekto ng isyu, kabilang kung paano maaaring samantalahin ito ng isang umaatake

Makakatulong ang impormasyong ito upang mas mabilis naming ma-triage ang iyong ulat.

Kung nag-uulat ka para sa bug bounty, mas kumpletong mga ulat ay maaaring mag-ambag sa mas mataas na gantimpala. Mangyaring bisitahin ang aming [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) na pahina para sa karagdagang detalye tungkol sa aming mga aktibong programa.
