## Mpangilio Mkuu

**Co-op Translator** hukusaidia kuweka maudhui yako ya elimu ya GitHub katika lugha mbalimbali kwa urahisi.
Unapoboresha faili zako za Markdown, picha, au daftari, tafsiri zinaendelea kuwa zimeendana kiotomatiki, kuhakikisha maudhui yako ni sahihi na ya kisasa kwa wanafunzi duniani kote.

Mfano jinsi maudhui yaliyotafsiriwa yanavyopangwa:

![Example](../../translated_images/sw/translation-ex.0c8aa6a7ee0aad2b.webp)

## Jinsi hali ya tafsiri inavyosimamiwa

Co-op Translator husimamia maudhui yaliyotafsiriwa kama **vifaa vya programu vilivyohifadhiwa toleo**,  
si kama faili hizi ni za kudumu.

Kifaa hiki hufuatilia hali ya Markdown, picha, na daftari zilizotafsiriwa  
kutumia **metadata inayohusiana na lugha**.

Muundo huu unamruhusu Co-op Translator:

- Kugundua kwa uhakika tafsiri za zamani
- Kutumia Markdown, picha, na daftari kwa usawa
- Kupanua kwa usalama kote kwenye miradi mikubwa, yenye mabadiliko ya kasi na lugha nyingi

Kwa kubuni tafsiri kama vifaa vinavyosimamiwa,  
mtaalamu wa kazi za tafsiri huendana asili na  
mbinu za kisasa za usimamizi wa utegemezi wa programu na vifaa.

→ [Jinsi hali ya tafsiri inavyosimamiwa](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)


## Kuanza Haraka

```bash
# Unda na kuwaamsha mazingira ya mtandao (inapendekezwa)
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
# Sakinisha kifurushi
pip install co-op-translator
# Tafsiri
translate -l "ko ja fr" -md
```

Docker:

```bash
# Vuta picha ya umma kutoka GHCR
docker pull ghcr.io/azure/co-op-translator:latest
# Endesha ukiwa umewekea folda ya sasa na .env (Bash/Zsh)
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## Mpangilio wa chini kabisa

1. Hakikisha una toleo la Python linalosindikwa (sasa 3.10-3.12). Kwa kutumia poetry (pyproject.toml) hili hufanywa moja kwa moja.
2. Tengeneza faili `.env` ukitumia kiolezo: [.env.template](../../.env.template)
3. Sanidi muhudumu mmoja wa LLM (Azure OpenAI au OpenAI)
4. (Hiari) Kwa tafsiri ya picha (`-img`), sanidi Azure AI Vision
5. (Hiari) Unaweza kusanidi seti kadhaa za vyeti kwa kunakili vigezo na viambishi kama `_1`, `_2`, nk. Vigezo vyote katika seti lazima vishe herufi ile ile ya kiambishi.
6. (Inapendekezwa) Safisha tafsiri zilizopita ili kuepuka migongano (mfano, `translations/`)
7. (Inapendekezwa) Ongeza sehemu ya tafsiri kwenye README yako kwa kutumia [kiolezo cha lugha za README](./getting_started/README_languages_template.md)
8. Angalia: [Sanidi Azure AI](./getting_started/set-up-azure-ai.md)

## Matumizi

Tafsiri aina zote zinazounga mkono:

```bash
translate -l "ko ja"
```

Markdown tu:

```bash
translate -l "de" -md
```

Markdown + picha:

```bash
translate -l "pt" -md -img
```

Daftari tu:

```bash
translate -l "zh" -nb
```

Bendera zaidi: [Marejeo ya amri](./getting_started/command-reference.md)

- Huhifadhi muundo na usanifu wa Markdown

## AI Inayewajibika

Microsoft imejizatiti kusaidia wateja wetu kutumia bidhaa zetu za AI kwa uwajibikaji, kushiriki maarifa yetu, na kujenga ushirikiano unaotegemea uaminifu kupitia zana kama Vidokezo vya Uwazi na Tathmini za Athari. Rasilimali nyingi kati ya hizi zinaweza kupatikana kwenye [https://aka.ms/RAI](https://aka.ms/RAI).
Njia ya Microsoft kuelekea AI inayowajibika inatokana na kanuni zetu za AI za haki, uaminifu na usalama, faragha na usalama, ujumuishaji, uwazi, na uwajibikaji.

Miundo mikubwa ya lugha ya asili, picha, na hotuba - kama ile inayotumika katika sampuli hii - inaweza kudhihirika kwa njia ambazo si za haki, zisizotegemeka, au za kukera, na kusababisha madhara. Tafadhali angalia [Kumbukumbu ya Uwasi ya Huduma ya Azure OpenAI](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ili upate taarifa kuhusu hatari na vizingiti.

Njia inayopendekezwa ya kupunguza hatari hizi ni kujumuisha mfumo wa usalama katika usanifu wako ambao unaweza kugundua na kuzuia tabia hatarishi. [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) hutoa kinga huru inayoweza kugundua maudhui hatarishi yanayotengenezwa na watumiaji na AI katika programu na huduma. Azure AI Content Safety inajumuisha API za maandishi na picha ambazo zinakuwezesha kugundua nyenzo hatarishi. Pia tunayo Studio ya Usalama wa Maudhui yenye mwingiliano inayokuwezesha kuona, kuchunguza na kujaribu mifano ya msimbo wa kugundua maudhui hatarishi kwa aina mbalimbali. Nyaraka za [kuanza haraka](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) zinaongoza unapoomba huduma hii.

Jambo lingine la kuzingatia ni utendaji wa jumla wa programu. Kwa programu zinazotumia njia nyingi na mifano mingi, tunachukulia utendaji kama mfumo unaotenda kama wewe na watumiaji wako mnavyotarajia, ikiwa ni pamoja na kutotengeneza matokeo hatarishi. Ni muhimu kutathmini utendaji wa programu yako kwa ujumla ukitumia [vigezo vya ubora wa uzalishaji na hatari na usalama](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in).

Unaweza kutathmini programu yako ya AI katika mazingira yako ya maendeleo ukitumia [SDK ya prompt flow](https://microsoft.github.io/promptflow/index.html). Ikiwa unayo seti ya data ya mtihani au lengo, uzalishaji wa programu yako ya AI unapimwa kitabia kwa kutumia wahakiki waliyojengwa ndani au wahakiki wa kawaida wa chaguo lako. Ili kuanza kutumia SDK ya prompt flow kutathmini mfumo wako, unaweza kufuata [mwongozo wa kuanza haraka](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk). Mara unapoendesha mtihani wa tathmini, unaweza [kuona matokeo katika Azure AI Studio](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results).
