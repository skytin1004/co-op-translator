## Security

Microsoft ਸਾਡੇ ਸੋਫਟਵੇਅਰ ਉਤਪਾਦਾਂ ਅਤੇ ਸੇਵਾਵਾਂ ਦੀ ਸੁਰੱਖਿਆ ਨੂੰ ਗੰਭੀਰਤਾ ਨਾਲ ਲੈਂਦਾ ਹੈ, ਜਿਸ ਵਿੱਚ ਸਾਡੇ GitHub ਸੰਗਠਨਾਂ ਰਾਹੀਂ ਪ੍ਰਬੰਧਿਤ ਸਾਰੇ ਸਰੋਤ ਕੋਡ ਰਿਪੋਜ਼ਿਟਰੀਜ਼ ਸ਼ਾਮਲ ਹਨ, ਜਿਨ੍ਹਾਂ ਵਿੱਚ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ਅਤੇ [Xamarin](https://github.com/xamarin) ਸ਼ਾਮਲ ਹਨ।

ਜੇ ਤੁਸੀਂ ਮੰਨਦੇ ਹੋ ਕਿ ਤੁਸੀਂ ਕਿਸੇ Microsoft-ਮਲਕੀਅਤ ਵਾਲੀ ਰਿਪੋਜ਼ਿਟਰੀ ਵਿੱਚ ਸੁਰੱਖਿਆ ਦੀ ਕਮਜ਼ੋਰੀ ਲੱਭੀ ਹੈ ਜੋ [Microsoft ਦੀ ਸੁਰੱਖਿਆ ਦੀ ਕਮਜ਼ੋਰੀ ਦੀ ਪਰਿਭਾਸ਼ਾ](https://aka.ms/security.md/definition) ਨੂੰ ਪੂਰਾ ਕਰਦੀ ਹੈ, ਤਾਂ ਕਿਰਪਾ ਕਰਕੇ ਹੇਠਾਂ ਦਿੱਤੇ अनुसार ਸਾਨੂੰ ਇਹ ਰਿਪੋਰਟ ਕਰੋ।

## Reporting Security Issues

**ਕਿਰਪਾ ਕਰਕੇ ਸੁਰੱਖਿਆ ਕਮਜ਼ੋਰੀਆਂ ਨੂੰ ਜਨਤਕ GitHub ਮਸਲੇ ਰਾਹੀਂ ਰਿਪੋਰਟ ਨਾ ਕਰੋ।**

ਇਸ ਦੀ ਬਜਾਏ, ਕਿਰਪਾ ਕਰਕੇ Microsoft Security Response Center (MSRC) ਨੂੰ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) 'ਤੇ ਰਿਪੋਰਟ ਕਰੋ।

ਜੇ ਤੁਸੀਂ ਲੌਗਇਨ ਕੀਤੇ ਬਿਨਾਂ ਸਬਮਿਟ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ ਇਮੇਲ ਭੇਜੋ [secure@microsoft.com](mailto:secure@microsoft.com)। ਜੇ ਸੰਭਵ ਹੋਵੇ, ਤਾਂ ਸਾਡੇ PGP ਕੁੰਜੀ ਨਾਲ ਆਪਣਾ ਸੁਨੇਹਾ ਕ੍ਰਿਪਟ ਕਰੋ; ਕਿਰਪਾ ਕਰਕੇ ਇਸਨੂੰ [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ਤੋਂ ਡਾਊਨਲੋਡ ਕਰੋ।

ਤੁਹਾਨੂੰ 24 ਘੰਟਿਆਂ ਦੇ ਅੰਦਰ ਜਵਾਬ ਮਿਲ ਜਾਣਾ ਚਾਹੀਦਾ ਹੈ। ਜੇ ਕਿਸੇ ਕਾਰਨ ਤੁਹਾਨੂੰ ਜਵਾਬ ਨਾ ਮਿਲੇ, ਤਾਂ ਇਹ ਪੁਸ਼ਟੀ ਕਰਨ ਲਈ ਕਿ ਅਸੀਂ ਤੁਹਾਡਾ ਮੂਲ ਸੁਨੇਹਾ ਪ੍ਰਾਪਤ ਕੀਤਾ ਹੈ, ਕਿਰਪਾ ਕਰਕੇ ਇਮੇਲ ਰਾਹੀਂ ਫਾਲੋਅਪ ਕਰੋ। ਵਾਧੂ ਜਾਣਕਾਰੀ ਲਈ [microsoft.com/msrc](https://www.microsoft.com/msrc) 'ਤੇ ਜਾਓ।

ਕਿਰਪਾ ਕਰਕੇ ਹੇਠ ਲਿਖੀ ਮੰਗੀ ਗਈ ਜਾਣਕਾਰੀ(ਜਿੰਨੀ ਤੁਸੀਂ ਦੇ ਸਕਦੇ ਹੋ) ਸ਼ਾਮਲ ਕਰੋ ਤਾਂ ਜੋ ਅਸੀਂ ਸੰਭਾਵਿਤ ਮੁੱਦੇ ਦੀ ਕਿਸਮ ਅਤੇ ਸੀਮਾ ਨੂੰ ਬਿਹਤਰ ਸਮਝ ਸਕੀਏ:

  * ਮੁੱਦੇ ਦੀ ਕਿਸਮ (ਜਿਵੇਂ ਕਿ ਬਫਰ ਓਵਰਫਲੋ, SQL ਇੰਜੈਕਸ਼ਨ, ਕ੍ਰਾਸ-ਸਾਈਟ ਸਕ੍ਰਿਪਟਿੰਗ, ਆਦਿ)
  * ਸਮੱਸਿਆ ਦੇ ਪ੍ਰਗਟਾਵੇ ਨਾਲ ਸਬੰਧਤ ਸਰੋਤ ਫਾਈਲ(ਆਂ) ਦੇ ਪੂਰੇ ਨੇੜੇ ਰਹਿਣ ਵਾਲੇ ਰਸਤੇ
  * ਪ੍ਰਭਾਵਿਤ ਸਰੋਤ ਕੋਡ ਦਾ ਸਥਾਨ (ਟੈਗ/ਬਰਾਂਚ/ਕਮੀਟ ਜਾਂ ਸਿੱਧਾ URL)
  * ਮੁੱਦੇ ਨੂੰ ਦੁਬਾਰਾ ਬਣਾਉਣ ਲਈ ਕੋਈ ਖਾਸ ਸੰਰਚਨਾ
  * ਮੁੱਦੇ ਨੂੰ ਦੁਬਾਰਾ ਬਣਾਉਣ ਲਈ ਕਦਮ ਦਰ ਕਦਮ ਹਦਾਇਤਾਂ
  * ਪ੍ਰੂਫ-ਆਫ-ਕਾਂਸੈਪਟ ਜਾਂ ਐਕਸਪਲੋਇਟ ਕੋਡ (ਜੇ ਸੰਭਵ ਹੋਵੇ)
  * ਮੁੱਦੇ ਦਾ ਪ੍ਰਭਾਵ, ਜਿਸ ਵਿੱਚ ਇਹ ਵੀ ਸ਼ਾਮਲ ਹੈ ਕਿ ਕਿਸ ਤਰ੍ਹਾਂ ਕੋਈ ਹਮਲਾਵਰ ਇਸ ਮੁੱਦੇ ਦਾ ਲਾਭ ਉਠਾ ਸਕਦਾ ਹੈ

ਇਹ ਜਾਣਕਾਰੀ ਸਾਡੇ ਲਈ ਤੁਹਾਡੀ ਰਿਪੋਰਟ ਨੂੰ ਤੇਜ਼ੀ ਨਾਲ ਜਾਨਚਣ ਵਿੱਚ ਮਦਦ ਕਰੇਗੀ।

ਜੇ ਤੁਸੀਂ ਬੱਗ ਬਾਊਂਟੀ ਲਈ ਰਿਪੋਰਟ ਕਰ ਰਹੇ ਹੋ, ਤਾਂ ਵਧੀਕ ਪੂਰੀਆਂ ਰਿਪੋਰਟਾਂ ਉੱਚ ਬਾਊਂਟੀ ਇਨਾਮ ਦੇ ਯੋਗ ਹੋ ਸਕਦੀਆਂ ਹਨ। ਕਿਰਪਾ ਕਰਕੇ ਸਾਡੇ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ਸਫ਼ੇ 'ਤੇ ਜਾਓ ਜਿੱਥੇ ਸਾਡੇ ਸਰਗਰਮ ਪ੍ਰੋਗਰਾਮਾਂ ਬਾਰੇ ਵਧੇਰੇ ਜਾਣਕਾਰੀ ਹੈ।
