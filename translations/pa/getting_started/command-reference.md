# Command reference

The **Co-op Translator** CLI ਕਈ ਵਿਕਲਪਾਂ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ ਜੋ ਅਨੁਵਾਦ ਪ੍ਰਭਿਰਣੀਕਰਨ ਨੂੰ ਅਨੁਕੂਲਿਤ ਕਰਨ ਲਈ ਹਨ:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | ਤੁਹਾਡੇ ਪ੍ਰੋਜੈਕਟ ਨੂੰ ਨਿਰਧਾਰਤ ਭਾਸ਼ਾਵਾਂ ਵਿੱਚ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ। ਉਦਾਹਰਨ: translate -l "es fr de" ਸਪੇਨੀ, ਫ਼ਰਾਂਸੀਸੀ ਅਤੇ ਜਰਮਨ ਵਿੱਚ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ। ਸਾਰੇ ਸਮਰਥਿਤ ਭਾਸ਼ਾਵਾਂ ਵਿੱਚ ਅਨੁਵਾਦ ਲਈ translate -l "all" ਵਰਤੋ।
translate -l "language_codes" -u              | ਮੌਜੂਦਾ ਅਨੁਵਾਦਾਂ ਨੂੰ ਮਿਟਾ ਕੇ ਅਨੁਵਾਦ ਅਪਡੇਟ ਕਰਦਾ ਹੈ ਅਤੇ ਫਿਰ ਵਾਪਸ ਬਣਾਉਂਦਾ ਹੈ। ਚੇਤਾਵਨੀ: ਇਹ ਨਿਰਧਾਰਤ ਭਾਸ਼ਾਵਾਂ ਲਈ ਸਾਰੇ ਮੌਜੂਦਾ ਅਨੁਵਾਦ ਮਿਟਾ ਦੇਵੇਗਾ।
translate -l "language_codes" -img            | ਸਿਰਫ਼ ਚਿੱਤਰ ਫਾਈਲਾਂ ਦਾ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ।
translate -l "language_codes" -md             | ਸਿਰਫ਼ Markdown ਫਾਈਲਾਂ ਦਾ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ।
translate -l "language_codes" -nb             | ਸਿਰਫ਼ Jupyter ਨੋਟਬੁੱਕ ਫਾਈਲਾਂ (.ipynb) ਦਾ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ।
translate -l "language_codes" --fix           | ਪਿਛਲੇ ਮੁਲਾਂਕਣ ਨਤੀਜਿਆਂ ਦੇ ਆਧਾਰ 'ਤੇ ਘੱਟ ਭਰੋਸੇਮੰਦ ਅਨੁਵਾਦਾਂ ਨੂੰ ਦੁਬਾਰਾ ਅਨੁਵਾਦ ਕਰਦਾ ਹੈ।
translate -l "language_codes" -d              | ਵਿਸਥਾਰਪੂਰਵਕ ਲੌਗਿੰਗ ਲਈ ਡੀਬੱਗ ਮੋਡ ਨੂੰ ਯੋਗ ਕਰਦਾ ਹੈ।
translate -l "language_codes" --save-logs, -s | DEBUG-ਸਤਹ ਦੇ ਲੌਗ ਫਾਈਲਾਂ ਨੂੰ <root_dir>/logs/ ਹੇਠਾਂ ਸੁਰੱਖਿਅਤ ਕਰਦਾ ਹੈ (ਕੰਸੋਲ -d ਨਾਲ ਨਿਯੰਤਰਿਤ ਰਹਿੰਦਾ ਹੈ)
translate -l "language_codes" -r "root_dir"   | ਪ੍ਰੋਜੈਕਟ ਦੀ ਮੁੱਖ ਡਾਇਰੈਕਟਰੀ ਦੱਸਦਾ ਹੈ
translate -l "language_codes" -f              | ਚਿੱਤਰ ਅਨੁਵਾਦ ਲਈ ਤੇਜ਼ ਮੋਡ ਵਰਤਦਾ ਹੈ (ਥੋੜ੍ਹੀ ਕੁਆਲਿਟੀ ਅਤੇ ਅਨੁਕੂਲਤਾ ਦੀ ਕਮੀ 'ਤੇ 3 ਗੁਣਾ ਤੇਜ਼ ਪਲਾਡਿੰਗ).
translate -l "language_codes" -y              | ਸਭ ਪ੍ਰਾਂਪਟਾਂ ਨੂੰ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ ਪੁਸ਼ਟੀ ਕਰਦਾ ਹੈ (CI/CD ਪਾਈਪਲਾਈਨਾਂ ਲਈ ਲਾਭਦਾਇਕ)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | ਅਨੁਵਾਦ ਕੀਤੇ ਮਾਰਕਡਾਊਨ ਅਤੇ ਨੋਟਬੁੱਕ ਵਿੱਚ ਮਸ਼ੀਨ ਅਨੁਵਾਦ ਅਸਤੀਤਵ ਖੰਡ ਜੋੜਨ ਜਾਂ ਅਣਜੋੜਨ ਲਈ ਸਕਰੀਆ ਜਾਂ ਨਿਸ਼ਕ੍ਰਿਆ ਕਰਦਾ ਹੈ (ਡਿਫੌਲਟ: ਸਹੀ).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | README ਭਾਸ਼ਾਵਾਂ ਭਾਗ ਦੀ ਸਲਾਹਕਾਰਤਾ ਨੂੰ ਤੁਹਾਡੇ ਰਿਪੋਜ਼ਟਰੀ URL ਨਾਲ ਨਿੱਜੀ ਬਣਾਉਂਦਾ ਹੈ।
translate -l "language_codes" --help          | CLI ਵਿੱਚ ਉਪਲਬਧ ਕਮਾਂਡਾਂ ਦਾ ਸਹਾਇਤਾ ਵੇਰਵਾ
evaluate -l "language_code"                  | ਕਿਸੇ ਵਿਸ਼ੇਸ਼ ਭਾਸ਼ਾ ਲਈ ਅਨੁਵਾਦ ਗੁਣਵੱਤਾ ਦਾ ਮੁਲਾਂਕਣ ਕਰਦਾ ਹੈ ਅਤੇ ਭਰੋਸੇ ਸੂਰਾਂਖ ਦਿੱਦਾ ਹੈ
evaluate -l "language_code" -c 0.8           | ਆਪਣੀ ਭਰੋਸਾ ਹਦ ਬਰਾਬਰ ਕਰਕੇ ਅਨੁਵਾਦ ਮੁਲਾਂਕਣ ਕਰਦਾ ਹੈ
evaluate -l "language_code" -f               | ਤੇਜ਼ ਮੁਲਾਂਕਣ ਮੋਡ (ਸਿਰਫ਼ ਨਿਯਮ-ਆਧਾਰਿਤ, ਕੋਈ LLM ਨਹੀਂ)
evaluate -l "language_code" -D               | ਡੀਪ ਮੁਲਾਂਕਣ ਮੋਡ (ਸਿਰਫ਼ LLM ਆਧਾਰਿਤ, ਵਧੇਰੇ ਤਫਸੀਲੀ ਪਰ धीਮੀ)
evaluate -l "language_code" --save-logs, -s  | DEBUG-ਸਤਹ ਦੇ ਲੌਗ ਫਾਈਲਾਂ ਨੂੰ <root_dir>/logs/ ਹੇਠਾਂ ਸੁਰੱਖਿਅਤ ਕਰਦਾ ਹੈ
migrate-links -l "language_codes"             | ਅਨੁਵਾਦ ਕੀਤੀਆਂ Markdown ਫਾਈਲਾਂ ਨੂੰ ਦੁਬਾਰਾ ਕਾਰਜ ਕਰਦਾ ਹੈ ਤਾਂ ਕਿ ਨੋਟਬੁੱਕ (.ipynb) ਲਈ ਲਿੰਕ ਅਪਡੇਟ ਹੋਣ। ਜਦੋਂ ਮੌਜੂਦ ਹੋਵੇ ਤਾਂ ਅਨੁਵਾਦਿਤ ਨੋਟਬੁੱਕ ਪ੍ਰਾਥਮਿਕਤਾ; ਨਹੀਂ ਤਾਂ ਮੂਲ ਨੋਟਬੁੱਕਾਂ ਤੱਕ ਵਾਪਸੀ ਹੋ ਸਕਦੀ ਹੈ।
migrate-links -l "language_codes" -r          | ਪ੍ਰੋਜੈਕਟ ਦੀ ਮੁੱਖ ਡਾਇਰੈਕਟਰੀ ਦੱਸੋ (ਡਿਫੌਲਟ: ਮੌਜੂਦਾ ਡਾਇਰੈਕਟਰੀ)।
migrate-links -l "language_codes" --dry-run   | ਦਿਖਾਓ ਕਿ ਕਿਹੜੀਆਂ ਫਾਈਲਾਂ ਬਦਲਦੀਆਂ, ਬਿਨਾਂ ਲਿਖਤ ਬਦਲਾਅ ਤੋਂ।
migrate-links -l "language_codes" --no-fallback-to-original | ਜਦੋਂ ਅਨੁਵਾਦਿਤ ਨੋਟਬੁੱਕ ਉਪਲਬਧ ਨਾ ਹੋਣ, ਤਦ ਮੂਲ ਵਾਲੇ ਲਿੰਕ ਦੁਬਾਰਾ ਨਹੀਂ ਲਿਖਦੇ (ਸਿਰਫ਼ ਅਨੁਵਾਦਿਤ ਮੌਜੂਦ ਹੋਣ 'ਤੇ ਅਪਡੇਟ ਕਰਦੇ ਹਨ)।
migrate-links -l "language_codes" -d          | ਵਿਸਥਾਰਪੂਰਵਕ ਲੌਗਿੰਗ ਲਈ ਡੀਬੱਗ ਮੋਡ ਯੋਗ ਕਰੋ।
migrate-links -l "language_codes" --save-logs, -s | DEBUG-ਸਤਹ ਲੌਗ ਨੂੰ <root_dir>/logs/ ਹੇਠਾਂ ਸੁਰੱਖਿਅਤ ਕਰੋ।
migrate-links -l "all" -y                      | ਸਾਰੀਆਂ ਭਾਸ਼ਾਵਾਂ ਦਾ ਪ੍ਰਕਿਰਯਾਕਰਨ ਕਰੋ ਅਤੇ ਚੇਤਾਵਨੀ ਪ੍ਰਾਂਪਟ ਆਟੋ ਪੁਸ਼ਟੀ ਕਰੋ।

## ਵਰਤੋਂ ਦੇ ਉਦਾਹਰਨ

  1. ਡਿਫੌਲਟ ਵਰਤੋਂ (ਮੌਜੂਦਾ ਅਨੁਵਾਦਾਂ ਨੂੰ ਮਿਟਾਏ ਬਿਨਾਂ ਨਵੇਂ ਅਨੁਵਾਦ ਜੋੜੋ):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. ਸਿਰਫ਼ ਨਵੇਂ ਕੋਰੀਆਈ ਚਿੱਤਰ ਅਨੁਵਾਦ ਜੋੜੋ (ਮੌਜੂਦਾ ਅਨੁਵਾਦ ਨਹੀਂ ਮਿਟਾਏ ਜਾਣਗੇ):    translate -l "ko" -img

  3. ਸਾਰੇ ਕੋਰੀਆਈ ਅਨੁਵਾਦ ਅਪਡੇਟ ਕਰੋ (ਚੇਤਾਵਨੀ: ਇਹ ਸਾਰੇ ਮੌਜੂਦਾ ਕੋਰੀਆਈ ਅਨੁਵਾਦ ਦੂਰ ਕਰਕੇ ਫਿਰ ਅਨੁਵਾਦ ਕਰੇਗਾ):    translate -l "ko" -u

  4. ਸਿਰਫ਼ ਕੋਰੀਆਈ ਚਿੱਤਰ ਅਪਡੇਟ ਕਰੋ (ਚੇਤਾਵਨੀ: ਇਹ ਸਾਰੇ ਮੌਜੂਦਾ ਕੋਰੀਆਈ ਚਿੱਤਰ ਮਿਟਾ ਕੇ ਫਿਰ ਅਨੁਵਾਦ ਕਰੇਗਾ):    translate -l "ko" -img -u

  5. ਦੂਸਰੇ ਅਨੁਵਾਦਾਂ 'ਤੇ ਪ੍ਰਭਾਵ ਪਾਏ ਬਿਨਾਂ ਕੋਰੀਆਈ ਲਈ ਨਵੇਂ ਮਾਰਕਡਾਊਨ ਅਨੁਵਾਦ ਜੋੜੋ:    translate -l "ko" -md

  6. ਪਿਛਲੇ ਮੁਲਾਂਕਣ ਨਤੀਜਿਆਂ ਦੇ ਆਧਾਰ 'ਤੇ ਘੱਟ ਭਰੋਸੇਮੰਦ ਅਨੁਵਾਦਾਂ ਨੂੰ ਠੀਕ ਕਰੋ: translate -l "ko" --fix

  7. ਖਾਸ ਫਾਈਲਾਂ ਲਈ ਘੱਟ ਭਰੋਸੇਮੰਦ ਅਨੁਵਾਦ (ਮਾਰਕਡਾਊਨ) ਠੀਕ ਕਰੋ: translate -l "ko" --fix -md

  8. ਖਾਸ ਫਾਈਲਾਂ ਲਈ ਘੱਟ ਭਰੋਸੇਮੰਦ ਅਨੁਵਾਦ (ਚਿੱਤਰ) ਠੀਕ ਕਰੋ: translate -l "ko" --fix -img

  9. ਚਿੱਤਰ ਅਨੁਵਾਦ ਲਈ ਤੇਜ਼ ਮੋਡ ਵਰਤੋਂ:    translate -l "ko" -img -f

  10. ਘੱਟ ਭਰੋਸੇਮੰਦ ਅਨੁਵਾਦ ਲਈ ਸਵੈ-ਨਿਰਧਾਰਤ ਹੱਦ ਵਰਤੋਂ: translate -l "ko" --fix -c 0.8

  11. ਡੀਬੱਗ ਮੋਡ ਉਦਾਹਰਨ: - translate -l "ko" -d: ਡੀਬੱਗ ਲੌਗਿੰਗ ਯੋਗ ਕਰੋ।
  12. ਲੌਗ ਫਾਈਲਾਂ ਵਿੱਚ ਸੇਵ ਕਰੋ: translate -l "ko" -s
  13. ਕੰਸੋਲ DEBUG ਅਤੇ ਫਾਈਲ DEBUG: translate -l "ko" -d -s
  14. ਮਸ਼ੀਨ ਅਨੁਵਾਦ ਅਸਤੀਤਵ ਖੰਡ ਜੋੜ੍ਹਣ ਤੋਂ ਬਿਨਾ ਅਨੁਵਾਦ ਕਰੋ: translate -l "ko" --no-disclaimer

  15. ਕੋਰੀਆਈ ਅਨੁਵਾਦਾਂ ਲਈ ਨੋਟਬੁੱਕ ਲਿੰਕ ਮਾਈਗਰੇਟ ਕਰੋ (ਜਦੋਂ ਮੌਜੂਦ ਹੋਵੇ ਤਾਂ ਅਨੁਵਾਦਿਤ ਨੋਟਬੁੱਕ ਲਿੰਕ ਅਪਡੇਟ ਕਰੋ):    migrate-links -l "ko"

  15. ਡ੍ਰਾਈ-ਰਨ ਨਾਲ ਲਿੰਕ ਮਾਈਗਰੇਟ ਕਰੋ (ਕੋਈ ਫਾਈਲ ਲਿਖਤ ਨਹੀਂ):    migrate-links -l "ko" --dry-run

  16. ਸਿਰਫ਼ ਅਨੁਵਾਦਿਤ ਨੋਟਬੁੱਕ ਮੌਜੂਦ ਹੋਣ 'ਤੇ ਲਿੰਕ ਅਪਡੇਟ ਕਰੋ (ਮੂਲਾਂ ਵੱਲ ਵਾਪਸੀ ਨਾ ਕਰੋ):    migrate-links -l "ko" --no-fallback-to-original

  17. ਸਾਰੀਆਂ ਭਾਸ਼ਾਵਾਂ ਦਾ ਪ੍ਰਕਿਰਯਾਕਰਨ ਪੁਸ਼ਟੀ ਪ੍ਰਾਂਪਟ ਨਾਲ ਕਰੋ:    migrate-links -l "all"

  18. ਸਾਰੀਆਂ ਭਾਸ਼ਾਵਾਂ ਪ੍ਰਕਿਰਯਾਕਰਨ ਅਤੇ ਆਟੋ-ਪੁਸ਼ਟੀ:    migrate-links -l "all" -y
  19. migrate-links ਲਈ ਲੌਗ ਫਾਈਲਾਂ ਸੇਵ ਕਰੋ:    migrate-links -l "ko ja" -s

  20. README ਭਾਸ਼ਾ ਸਲਾਹਕਾਰ ਨੂੰ ਤੁਹਾਡੇ ਰਿਪੋ URL ਨਾਲ ਨਿੱਜੀ ਬਣਾਓ:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### ਮੁਲਾਂਕਣ ਉਦਾਹਰਨ

> [!WARNING]  
> **Beta Feature**: ਇਹ ਮੁਲਾਂਕਣ ਫੰਕਸ਼ਨਾਲਟੀ ਇਸ ਵੇਲੇ ਬੀਟਾ ਵਿੱਚ ਹੈ। ਇਹ ਫੀਚਰ ਅਨੁਵਾਦ ਕੀਤੇ ਦਸਤਾਵੇਜ਼ਾਂ ਦੇ ਮੁਲਾਂਕਣ ਲਈ ਜਾਰੀ ਕੀਤਾ ਗਿਆ ਸੀ, ਅਤੇ ਮੁਲਾਂਕਣ ਦੇ ਤਰੀਕੇ ਅਤੇ ਵਿਸਥਾਰਤ ਲਾਗੂ ਕਰਨ ਦੀ ਪ੍ਰਕਿਰਿਆ ਅਜੇ ਵੀ ਵਿਕਾਸ ਦੌਰ ਵਿੱਚ ਹੈ ਅਤੇ ਬਦਲਾਅ ਦੇ ਅਧੀਨ ਹੈ।

  1. ਕੋਰੀਆਈ ਅਨੁਵਾਦਾਂ ਦਾ ਮੁਲਾਂਕਣ: evaluate -l "ko"

  2. ਆਪਣੀ ਭਰੋਸੇਮੰਦ ਹੱਦ ਨਾਲ ਮੁਲਾਂਕਣ: evaluate -l "ko" -c 0.8

  3. ਤੇਜ਼ ਮੁਲਾਂਕਣ (ਸਿਰਫ਼ ਨਿਯਮ-ਆਧਾਰਿਤ): evaluate -l "ko" -f

  4. ਡੀਪ ਮੁਲਾਂਕਣ (LLM ਆਧਾਰਿਤ): evaluate -l "ko" -D
