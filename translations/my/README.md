## အကျဉ်းချုပ်

**Co-op Translator** သည် သင်၏ ပညာရေး GitHub အကြောင်းအရာများကို ဘာသာစကားအမျိုးမျိုးသို့ လွယ်ကူစွာ တည်ဆောက်နိုင်ရန် ကူညီပေးပါသည်။  
Markdown ဖိုင်များ၊ ပုံများ သို့မဟုတ် notebook များကို अद्यतन တင်သွင်းသောအခါ ဘာသာပြန်များကို အလိုအလျောက် synchronization လုပ်ပြီး လေ့လာသူများအတွက် အချက်အလက်ထိပ်တန်းစွာတိကျပြီး နောက်ဆုံးပေါ်ဖြစ်စေပါသည်။

ဘာသာပြန်ထားသောအကြောင်းအရာများ စီမံခန့်ခွဲမှုကို ဥပမာပြပါ။

## ဘာသာပြန် အခြေအနေကို မည်သို့ စီမံခန့်ခွဲသနည်း

Co-op Translator သည် ဘာသာပြန်ထားသောအကြောင်းအရာများကို **ဗားရှင်းပြုပြင်ထားသော software artifact** များအဖြစ် စီမံပါသည်၊  
သင်္ချာဖိုင် အနေနှင့် မဟုတ်ပါ။

ယောဂလ်ကိရိယာသည် ဘာသာပြန်ထားသော Markdown များ၊ ပုံများနှင့် notebook များ၏ အခြေအနေကို  
**ဘာသာစကားအလိုက် metadata** ကို အသုံးပြုကာ ထိန်းသိမ်းသည်။

ဒီဆွဲချက်က Co-op Translator ကို ခွင့်ပြုသည်။

- ဟောကြားမှု အဟောင်းများကို ယုံကြည်စွာ ရှာဖွေရန်
- Markdown, ပုံများနှင့် notebook များကို တူညီသော နည်းဖြင့် ကိုင်တွယ်ရန်
- အကြီးစား၊ အမြန်လွှတ်သော၊ ဘာသာစကားစုံ repository များအား အန္တရာယ်ကင်းစေရန်

ဘာသာပြန်ထားသည့်အရာများကို စီမံထားသော အချက်အလက်များအနေနှင့်  
ဘာသာပြန်လုပ်ငန်းစဉ်များကို ခေတ်မီ  
software dependency နှင့် artifact စီမံခန့်ခွဲမှုနည်းလမ်းများနှင့် သဘာဝကျစေပါသည်။  

→ [ဘာသာပြန်အခြေအနေ စီမံပုံ](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)


## အလျင်အမြန် စတင်ခြင်း

```bash
# သင်္ဘောလှည့်ပတ် ပတ်ဝန်းကျင်တစ်ခု ဖန်တီးပြီး တက်နေစေပါ (အကြံပြုချက်)
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS/Linux
source .venv/bin/activate
# ပက်ကေ့ချ်ကို တပ်ဆင်ပါ
pip install co-op-translator
# ဘာသာပြန်ဆိုချက်တင်ပါ
translate -l "ko ja fr" -md
```

Docker:

```bash
# GHCR မှ ပေါ်ပြူထားသော ပုံကို ဆွဲယူပါ
docker pull ghcr.io/azure/co-op-translator:latest
# လက်ရှိဖိုလ်ဒါကို စီမံပြီး .env ဖိုင်ပါဝင်စေပြီး (Bash/Zsh) အတည်ပြုလိုက်ပါ
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## အနည်းဆုံး စနစ်တက်ခြင်း

1. သင်မှာ ထောက်ခံထားသော Python ဗားရှင်းရှိခြင်းအား သက်ဆိုင်စစ်ဆေးပါ (လက်ရှိမှာ 3.10-3.12)။ Poetry (pyproject.toml) မှာ အလိုအလျောက်ထိန်းသိမ်းထားသည်။
2. `.env` ဖိုင်ကို နမူနာအတိုင်းဖန်တီးပါ: [.env.template](../../.env.template)
3. LLM provider တစ်ခု (Azure OpenAI သို့မဟုတ် OpenAI) ကို ညှိနှိုင်းပါ
4. (Optional) ပုံဘာသာပြန်ရေးအတွက် (`-img`) Azure AI Vision ကို ညှိနှိုင်းပါ
5. (Optional) အကောင့်အရေအတွက် တစ်ခုထက်ပိုရှိပါက `_1`, `_2` စသည့် suffix များဖြင့် ညှိနှိုင်းနိုင်သည်။ တစ်ခုချင်း credential set တွင် suffix တူညီရပါမည်။
6. (အကြံပြုချက်) ယခင်ဘာသာပြန်များကို conflicts မဖြစ်စေရန် သန့်ရှင်းပါ (ဥပမာ - `translations/`)
7. (အကြံပြုချက်) README မှာ ဘာသာပြန်အစိတ်အပိုင်း ထည့်သွင်းရန် [README languages template](./getting_started/README_languages_template.md) ဖိုင်ကို ကြည့်ပါ
8. Azure AI ကို ဆက်တင်ရန်: [Set up Azure AI](./getting_started/set-up-azure-ai.md)

## အသုံးပြုနည်း

ထောက်ခံထားသော အမျိုးအစားအားလုံး ဘာသာပြန်ပါ -

```bash
translate -l "ko ja"
```

Markdown အပ်ဒိတ်သာ -

```bash
translate -l "de" -md
```

Markdown နှင့် ပုံများ -

```bash
translate -l "pt" -md -img
```

Notebook များသာ -

```bash
translate -l "zh" -nb
```

နောက်ထပ် flag များ: [Command reference](./getting_started/command-reference.md)

## အင်္ဂါရပ်များ

- Markdown, notebook နှင့် ပုံများအတွက် အလိုအလျောက် ဘာသာပြန်မှု
- မူရင်း အချက်အလက် အပေါ် မူတည်သည့် ဘာသာပြန်ချက်များကို တစ်ပြိုင်နက် sync လုပ်
- ဒေသတွင်း (CLI) သို့မဟုတ် CI (GitHub Actions) တွင် လုပ်ဆောင်နိုင်
- Azure OpenAI သို့မဟုတ် OpenAI ကိုအသုံးပြုသည်။ ပုံများအတွက် optional Azure AI Vision ကို သုံး
- Markdown format နှင့် ဖွဲ့စည်းမှုကို ထိန်းသိမ်းထား

## တာဝန်ခံ AI

Microsoft သည် ကျွန်တော်တို့၏ ဖောက်သည်များကို AI ထုတ်ကုန်များကို တာဝန်ရှိစွာ အသုံးပြုစေခြင်း၊ သင်ယူမှုများကို မျှဝေခြင်းနှင့် Transparency Notes ရဲ့ စာတမ်းများနှင့် Impact Assessments ကဲ့သို့သော ကိရိယာများဖြင့် ယုံကြည်မှု အခြေပြု မိတ်ဖက်ဆက်ဆံရေးများကို တည်ဆောက်ပေးရန် အတော်ကြိုးပမ်းနေပါသည်။ ၎င်းအဖွဲ့အစည်းများအများစုကို [https://aka.ms/RAI](https://aka.ms/RAI) တွင် တွေ့နိုင်ပါသည်။
Microsoft ၏ တာဝန်ခံ AI ကို ဝါဒများဖြစ်သည့် တန်းတူညီမျှမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် လုံခြုံမှု၊ ကိုယ်ရေးကိုယ်တာနှင့် လုံခြုံရေး၊ ထည့်သွင်းဖွဲ့စည်းမှု၊ ထင်ရှားသာမြင်မှုနှင့် တာဝန်ယူမှုတို့အပေါ် အခြေပြုထားပါသည်။

အကြီးစား သဘာဝဘာသာစကား၊ ပုံပေါ်တွင်၊ နှင့် စကားပြောပုံစံ မော်ဒယ်များ — ဤနမူနာတွင် သုံးသောမော်ဒယ်များကဲ့သို့ — ဖြစ်စဉ်များသည် မတန်းတူညီမျှသော၊ ယုံကြည်စိတ်ချရမှု မရှိသော သို့မဟုတ် ရိုက်ခတ်ထိုးနှက်သော အပြုအမူများ ပြုလုပ်နိုင်ပြီး၊ ထိုကဲ့သို့ဖြစ်ခဲ့လျှင် ထိခိုက်မှုများဖြစ်ပေါ်စေတတ်ပါသည်။ အန္တရာယ်များနှင့် ကန့်သတ်ချက်များကို သိရှိနိုင်ရန် [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ကို ကြည့်ရှုရန် အကြံပြုပါသည်။

ဤအန္တရာယ်များကို လျှော့ချရန် ညွှော်ကြားချက်အနေဖြင့် သင့်အခြေခံ ဖွဲ့စည်းပုံတွင် နှိမ်နင်းမှုရှိသော အပြုအမူများကို ရှာဖွေနိုင်ကာ တားဆီးနိုင်သော လုံခြုံမှုစနစ် တစ်ခု ထည့်သွင်းရန် အကြံပြုပါသည်။ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) သည် လွတ်လပ်သော ကာကွယ်မှုအလွှာတစ်ခုဖြစ်၍ အသုံးပြုသူက ဖန်တီးသော မာတိကာများနှင့် AI ဖန်တီးသော မာတိကာများကို သင့်တပ်ဆင်ထားသော အက်ပလီကေးရှင်းများနှင့် ဝန်ဆောင်မှုများတွင် တုံ့ပြန် စစ်ဆေးနိုင်သည်။ Azure AI Content Safety တွင် စာသားနှင့်ပုံ API များ ပါဝင်ပြီး ရိုက်ခတ်မှုရှိသော မာတိကာများကို စစ်ဆေးနိုင်သည်။ ထို့အပြင် Content Safety Studio ထည့်သွင်းထားပြီး ဖြစ်နိုင်သော အကြောင်းအရာများကို ကြည့်ရှု၊ ရှာဖွေ၊ နမူနာကုဒ်များကို စမ်းသပ်နိုင်ပါသည်။ အောက်ပါ [quickstart ညွှန်ကြားချက်](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) သည် ဝန်ဆောင်မှုကို မေးပွဲတင်ရန် လမ်းညွှန်ပေးပါသည်။

တစ်ခြားအချက်တစ်ခုမှာ စုစုပေါင်း အက်ပလီကေးရှင်း လည်ပတ်မှု ဖြစ်ပါသည်။ မျိုးစုံ နှင့် မော်ဒယ်စုံ ပါဝင်သည့် အက်ပလီကေးရှင်းများတွင် စနစ်သည် သင်နှင့် သင်၏အသုံးပြုသူများ မျှော်လင့်သကဲ့သို့ လည်ပတ်မှုရှိသည်၊ ထိုအပြင် ရိုက်ခတ်မှုရှိသော အဖြေများ မထုတ်ပေးရန် ဖြစ်စေခြင်းကို မှတ်ချက်ပြုပါသည်။ သင့်အက်ပလီကေးရှင်း၏ လုပ်ဆောင်မှုကို [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ဖြင့် သုံးသပ်ရန် အရေးကြီးပါသည်။

သင်၏ AI အက်ပလီကေးရှင်းကို တိုးတက်မှုပတ်ဝန်းကျင်တွင် [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) အသုံးပြု၍ အကဲဖြတ်နိုင်ပါသည်။ စမ်းသပ် ဒေတာသိုလ်တစ်ခု သို့မဟုတ် ပစ်မှတ်တစ်ခု ပေးသည့်အခါ သင့်တီထွင်မှု AI အက်ပလီကေးရှင်း၏ ထုတ်လုပ်မှုအား built-in evaluator များ သို့မဟုတ် သင့်ရွေးချယ်သော မူလတည် evaluator များဖြင့် ကန့်သတ်ချက်ပြု၍ အရေအတွက်ဖြင့် တိုင်းတာသည်။ prompt flow sdk ကို အသုံးပြုပြီး စနစ် အကဲဖြတ်မှု စတင်ရန် [quickstart လမ်းညွှန်ချက်](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ကို လိုက်နာနိုင်ပါသည်။ အကဲဖြတ်မှု တစ်ခု ဖျော်ဖြေပြီးနောက် [Azure AI Studio တွင် ရလဒ်များကို မြင်ကွင်းဖော်](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results) နိုင်ပါသည်။
