## လုံခြုံရေး

Microsoft သည် ၎င်း၏ ဆော့ဖ်ဝဲထုတ်ကုန်များနှင့် ဝန်ဆောင်မှုများ၏ လုံခြုံရေးကို အထူးဂရုပြုပါသည်၊ ၎င်းတွင် ကျွန်ုပ်တို့၏ GitHub အဖွဲ့အစည်းများမှ ကိုင်တွယ်ထားသည့် မူလကုဒ် repositories များအားလုံး ပါဝင်ပြီး၊ ၎င်းတို့မှာ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) နှင့် [Xamarin](https://github.com/xamarin) တို့ ဖြစ်ပါသည်။

Microsoft ပိုင်ဆိုင်သည့် repository များတွင် [Microsoft ၏ လုံခြုံရေးအားနည်းချက်၏ သတ်မှတ်ချက်](https://aka.ms/security.md/definition)နှင့် ကိုက်ညီသည့် လုံခြုံရေးအားနည်းချက်တစ်ခု ရှာဖွေတွေ့ရှိခဲ့သည်ဟု သင် ယူဆပါက၊ အောက်တွင် ဖော်ပြထားသည့်နည်းလမ်းဖြင့် ကျွန်ုပ်တို့ထံ သတင်းပို့ပေးပါ။

## လုံခြုံရေးပြဿနာများ မှတ်တမ်းတင်ခြင်း

**ကျေးဇူးပြု၍ လာအပ်သည့် လုံခြုံရေးအားနည်းချက်များကို လူထု GitHub issue များဖြင့် မဖြန့်ဝေပါနဲ့။**

တန်းတူအားဖြင့် Microsoft Security Response Center (MSRC) ကို [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) တွင် သတင်းပို့ပါ။

Login မလုပ်ဘဲ တင်သွင်းလိုပါက [secure@microsoft.com](mailto:secure@microsoft.com) သို့ အီးမေးလ်ပို့ပေးပါ။ ဖြစ်နိုင်ပါက ကျွန်ုပ်တို့၏ PGP key ဖြင့် သင်၏စာကို ကာကွယ်ပေးပါရန်; ၎င်းကို [Microsoft Security Response Center PGP Key စာမျက်နှာ](https://aka.ms/security.md/msrc/pgp) မှ ဒေါင်းလုတ်ပြုလုပ်နိုင်ပါသည်။

၂၄ နာရီအတွင်း တုံ့ပြန်ချက် ရရှိသင့်ပါသည်။ မရနိုင်ပါက ကျွန်ုပ်တို့၏ ကြိုတင်စာကို လက်ခံရရှိကြောင်း အီးမေးလ်ဖြင့် စစ်ဆေးပါရန်။

နောက်ထပ်သတင်းအချက်အလက်များကို [microsoft.com/msrc](https://www.microsoft.com/msrc) တွင် ရနိုင်ပါသည်။

အောက်ပါ လိုအပ်ချက်များ (ရနိုင်သမျှ အများဆုံး) ကို ပေးပို့ပါရန် မေတ္တာရပ်ခံပါသည်၊ ဤအချက်များက ကျွန်ုပ်တို့အား ပြဿနာ၏ သဘာဝနှင့် အကျယ်အဝန်းကို ပိုမိုနားလည်ရန် ကူညီပါလိမ့်မည်-

  * ပြဿနာအမျိုးအစား (ဥပမာ buffer overflow, SQL injection, cross-site scripting စသည်)
  * ပြဿနာဖြစ်ပေါ်ရာ နှင့်သက်ဆိုင်သော မူလဖိုင်လမ်းကြောင်းများ အပြည့်အစုံ
  * ထိခိုက်သော မူလကုဒ်တည်နေရာ (tag/branch/commit သို့မဟုတ် တိုက်ရိုက် URL)
  * ပြဿနာကို ထပ်တူထပ်မျှ ပြန်ပေါ်လာစေရန် လိုအပ်သော အထူးသတ်မှတ်မှုပုံစံ များ
  * ပြဿနာကို ပြန်လည် ရှာဖွေရေးအဆင့်ဆင့် လမ်းညွှန်ချက်များ
  * သက်သေခံသည့် အကြံအဖြေ သို့မဟုတ် exploit ကုဒ် (ဖြစ်နိုင်ပါက)
  * ပြဿနာ၏ သက်ရောက်မှုနှင့် အတူ တိုက်ခိုက်သူတစ်ဦး အဘယ်လိုအသုံးချနိုင်သည်မှာ

ဤအချက်အလက်များက ကျွန်ုပ်တို့အား သင်၏အစီရင်ခံစာကို အမြန်ဆုံး သတ်မှတ်အဆင့်သို့ ရှာဖွေနိုင်စေပါလိမ့်မည်။

bug bounty အတွက် သတင်းပို့ပါက၊ ပိုပြီး ပြည့်စုံသော အစီရင်ခံစာများသည် ပိုမိုမြင့်မားသော ဆုကြေးငွေ ရရှိစေပါသည်။ ကျေးဇူးပြု၍ ကျွန်ုပ်တို့၏ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) စာမျက်နှာတွင် လက်ရှိ အစီအစဉ်များနှင့် ပိုမိုသိရှိပါ။

## ဦးစားပေးဘာသာစကားများ

ကျွန်ုပ်တို့သည် ဆက်သွယ်မှုအားလုံးကို အင်္ဂလိပ်ဘာသာဖြင့် စိတ်တိုင်းကျ မြှင့်တင်လိုပါသည်။
