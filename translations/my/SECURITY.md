## Security

Microsoft သည် ကျွန်ုပ်တို့၏ ဆော့ဖ်ဝဲထုတ်ကုန်များနှင့် ဝန်ဆောင်မှုများ၏ လုံခြုံမှုကို တောင့်တင်းစွာယူဆောင်သည်၊ ၎င်းတွင် ကျွန်ုပ်တို့ GitHub အဖွဲ့အစည်းများမှ စီမံခန့်ခွဲသော မူရင်းကုဒ်သိုလှောင်တိုက်များအားလုံး ပါဝင်သည်၊ ၎င်းတွင် [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) နှင့် [Xamarin](https://github.com/xamarin) ပါဝင်သည်။

Microsoft ပိုင်ဆိုင်သော မည်သည့် သိုလှောင်တိုက်တွင်မဆို [Microsoft ၏ လုံခြုံရေးထိခိုက်မှု အဓိပ္ပါယ်](https://aka.ms/security.md/definition) နှင့် ကိုက်ညီသည့် လုံခြုံရေးထိခိုက်မှု တစ်ခု ရှာဖွေတွေ့ရှိထားသည်ဟု နိုင်ငံတကာထောက်ခံစိတ်ရှိပါက၊ အောက်ပါအတိုင်း ကျွန်ုပ်တို့ထံ ၎င်းကို ရှင်းလင်းကြရအောင်။

## လုံခြုံရေးပြဿနာများ ကို အစီရင်ခံခြင်း

**လုံခြုံရေးထိခိုက်မှုများကို ပြည်သူ GitHub ပြဿနာများမှတဆင့် မအစီရင်ခံရန် ကျေးဇူးပြု၍ မလုပ်ပါနှင့်။**

၎င်းအစား Microsoft Security Response Center (MSRC) သို့ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) မှတဆင့်အစီရင်ခံပါ။

အကယ်၍ လော့ဂ်အင်မလုပ်ဘဲ တင်သွင်းလိုပါက [secure@microsoft.com](mailto:secure@microsoft.com) သို့ အီးမေလ် ပို့ပါ။ ဖြစ်နိုင်လျှင် ကျွန်ုပ်တို့၏ PGP key ဖြင့် သင်၏ မက်ဆေ့ကို စာရ Encryption ပြုလုပ်ပါရန် အညွန်းကောင်းဖြစ်သည်။ ကျေးဇူးပြု၍ [Microsoft Security Response Center PGP Key စာမျက်နှာ](https://aka.ms/security.md/msrc/pgp) မှ ဒေါင်းလုပ်လုပ်ပါ။

၂၄ နာရီအတွင်း တုံ့ပြန်ချက် ရရှိသင့်ပါသည်။ မရရှိပါက အီးမေလ်မှတဆင့် ထပ်မံဆက်သွယ်၍ ကျွန်ုပ်တို့ သင့်မက်ဆေ့ကို လက်ခံရရှိကြောင်း သေချာစေပါ။

ထပ်တိုးသိရှိလိုပါက [microsoft.com/msrc](https://www.microsoft.com/msrc) တွင် ရှိသည်။

သင်၏အစီရင်ခံချက်ကို ပိုမိုနားလည်ရန် အောက်ပါ အချက်အလက်များကို (နိုင်သမျှဖြည့်စွက်ပေးပါ) ပါပါစေ-

  * ပြဿနာအမျိုးအစား (ဥပမာ buffer overflow, SQL injection, cross-site scripting စသဖြင့်)
  * ပြသမှု ဖြစ်ပေါ်သည့် မူရင်းဖိုင်များ၏အပြည့်အစုံလမ်းကြောင်းများ
  * ထိခိုက်မှု ရှိသည့် မူရင်းကုဒ်တည်နေရာ (tag/branch/commit သို့မဟုတ်တိုက်ရိုက် URL)
  * ပြဿနာကို ပြန်လည်ဖော်ထုတ်ရန် လိုအပ်သည့် အထူး ဖော်ပြချက်များ
  * ပြဿနာကို ပြန်ဖော်ထုတ်ရန် အဆင့်ဆင့်ညွှန်ကြားချက်များ
  * သက်သေချက်ဖေါ်ပြခြင်း သို့မဟုတ် exploit code (ဖြစ်နိုင်လျှင်)
  * ပြဿနာ၏ သက်ရောက်မှုနှင့် ထိခိုက်မှုကို အပြင် ခံနိုင်သူက အားလုံးကို ရယူနိုင်မည့် နည်းလမ်းများ

ဤအချက်အလက်များသည် ကျွန်ုပ်တို့အား သင့်အစီရင်ခံတင်ပြချက်ကို လျင်မြန်စွာ စိစစ်နိုင်ရန် အထောက်အကူဖြစ်သည်။

Bug bounty အတွက်အစီရင်ခံပါက၊ များပြားစွာဖြည့်စွက်သောအစီရင်ခံချက်များသည် စျေးလတ် ဖေါ်မယ့် ထောက်ခံငွေ ထပ်မံရရှိနိုင်စေပါသည်။ ကျေးဇူးပြု၍ ကျွန်ုပ်တို့၏ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) စာမျက်နှာတွင် လက်ရှိ ရှိပါသောအစီအစဉ်များအကြောင်း အသေးစိတ်ကို ကြည့်ရှုပါ။
