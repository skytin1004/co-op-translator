## Security

Microsoft 非常重視我們軟件產品及服務的安全性，這包括通過我們的 GitHub 組織管理的所有源代碼庫，當中包括 [Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet) 及 [Xamarin](https://github.com/xamarin)。

如果你認為你已發現任何由 Microsoft 擁有的倉庫中的安全漏洞，並且符合 [Microsoft 對安全漏洞的定義](https://aka.ms/security.md/definition)，請按以下方法向我們報告。

## Reporting Security Issues

**請不要通過公共 GitHub issue 報告安全漏洞。**

請改為通過 Microsoft 安全響應中心（MSRC）在 [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) 報告。

如果你希望不登入而提交，請發郵件至 [secure@microsoft.com](mailto:secure@microsoft.com)。如有可能，請使用我們的 PGP 金鑰加密你的訊息；可從 [Microsoft 安全響應中心 PGP 金鑰頁面](https://aka.ms/security.md/msrc/pgp) 下載。

你應會在 24 小時內收到回覆。如因某些原因未收到，請通過電郵跟進以確保我們已收到你的原始訊息。更多資訊請參閱 [microsoft.com/msrc](https://www.microsoft.com/msrc)。

請包括以下所需資料（盡可能提供）以幫助我們更好地理解可能的問題性質及範圍：

  * 問題類型（例如緩衝區溢出、SQL 注入、跨站腳本等）
  * 與問題表現相關的源文件完整路徑
  * 受影響的源代碼位置（標籤／分支／提交或直接 URL）
  * 重現問題所需的特殊配置
  * 重現問題的逐步指引
  * 概念驗證或利用代碼（如可能）
  * 問題影響，包括攻擊者可能如何利用該問題

這些資訊有助於我們更快地分類你的報告。

如果你是報告錯誤懸賞漏洞，更完整的報告有助提高賞金金額。請參閱我們的 [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) 頁面，了解我們的活躍項目詳情。
