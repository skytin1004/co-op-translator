# مرجع الأوامر

يقدم **Co-op Translator** CLI عدة خيارات لتخصيص عملية الترجمة:

الأمر                                       | الوصف
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | يترجم مشروعك إلى اللغات المحددة. مثال: translate -l "es fr de" يترجم إلى الإسبانية والفرنسية والألمانية. استخدم translate -l "all" للترجمة إلى جميع اللغات المدعومة.
translate -l "language_codes" -u              | يُحدّث الترجمات بحذف الترجمات الموجودة وإعادة إنشائها. تحذير: هذا سيحذف جميع الترجمات الحالية للغات المحددة.
translate -l "language_codes" -img            | يترجم ملفات الصور فقط.
translate -l "language_codes" -md             | يترجم ملفات Markdown فقط.
translate -l "language_codes" -nb             | يترجم ملفات Jupyter notebook فقط (.ipynb).
translate -l "language_codes" --fix           | يُعيد ترجمة الملفات ذات درجات الثقة المنخفضة بناءً على نتائج التقييم السابقة.
translate -l "language_codes" -d              | يُفعّل وضع التصحيح لتسجيل تفصيلي.
translate -l "language_codes" --save-logs, -s | يحفظ سجلات DEBUG في ملفات تحت <root_dir>/logs/ (الكونسول يبقى محكوماً بـ -d)
translate -l "language_codes" -r "root_dir"   | يحدد الدليل الجذري للمشروع
translate -l "language_codes" -f              | يستخدم الوضع السريع لترجمة الصور (حتى 3 أضعاف سرعة الرسم مع تأثير طفيف على الجودة والمحاذاة).
translate -l "language_codes" -y              | يؤكد تلقائياً جميع المطالبات (مفيد لسلاسل CI/CD)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | تمكين أو تعطيل إضافة قسم إخلاء مسؤولية الترجمة الآلية لملفات markdown والدفاتر المترجمة (افتراضي: مفعّل).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | تخصيص القسم الاستشاري للغات في README (الفحص المتقطع) باستخدام رابط المستودع الخاص بك.
translate -l "language_codes" --help          | تفاصيل المساعدة داخل CLI تُظهر الأوامر المتاحة
evaluate -l "language_code"                  | تقييم جودة الترجمة للغة محددة وتوفير درجات الثقة
evaluate -l "language_code" -c 0.8           | تقييم الترجمات باستخدام عتبة ثقة مخصصة
evaluate -l "language_code" -f               | وضع التقييم السريع (قاعدي فقط، بدون LLM)
evaluate -l "language_code" -D               | وضع التقييم العميق (معتمد على LLM فقط، أكثر دقة لكنه أبطأ)
evaluate -l "language_code" --save-logs, -s  | يحفظ سجلات DEBUG في ملفات تحت <root_dir>/logs/
migrate-links -l "language_codes"             | إعادة معالجة ملفات Markdown المترجمة لتحديث الروابط إلى دفاتر (.ipynb). يفضل الدفاتر المترجمة عند توفرها؛ وإلا يمكن الرجوع إلى الدفاتر الأصلية.
migrate-links -l "language_codes" -r          | تحديد الدليل الجذري للمشروع (الافتراضي: الدليل الحالي).
migrate-links -l "language_codes" --dry-run   | عرض الملفات التي ستتغير دون كتابة تغييرات.
migrate-links -l "language_codes" --no-fallback-to-original | عدم إعادة كتابة الروابط إلى الدفاتر الأصلية عند غياب النسخ المترجمة (التحديث فقط عند وجود النسخة المترجمة).
migrate-links -l "language_codes" -d          | تمكين وضع التصحيح لتسجيل تفصيلي.
migrate-links -l "language_codes" --save-logs, -s | يحفظ سجلات DEBUG في ملفات تحت <root_dir>/logs/
migrate-links -l "all" -y                      | معالجة جميع اللغات وتأكيد تلقائي لمطالبة التحذير.

## أمثلة الاستخدام

  1. السلوك الافتراضي (إضافة ترجمات جديدة دون حذف الترجمات الموجودة):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. إضافة ترجمات صور كورية جديدة فقط (لا تُحذف الترجمات الحالية):    translate -l "ko" -img

  3. تحديث جميع الترجمات الكورية (تحذير: هذا يحذف كل الترجمات الكورية الحالية قبل إعادة الترجمة):    translate -l "ko" -u

  4. تحديث صور كورية فقط (تحذير: هذا يحذف كل صور كورية حالية قبل إعادة الترجمة):    translate -l "ko" -img -u

  5. إضافة ترجمات Markdown جديدة للكورية دون التأثير على الترجمات الأخرى:    translate -l "ko" -md

  6. إصلاح الترجمات ذات الثقة المنخفضة بناءً على نتائج التقييم السابقة: translate -l "ko" --fix

  7. إصلاح الترجمات ذات الثقة المنخفضة لملفات معينة فقط (markdown): translate -l "ko" --fix -md

  8. إصلاح الترجمات ذات الثقة المنخفضة لملفات معينة فقط (صور): translate -l "ko" --fix -img

  9. استخدام الوضع السريع لترجمة الصور:    translate -l "ko" -img -f

  10. إصلاح الترجمات ذات الثقة المنخفضة مع عتبة مخصصة: translate -l "ko" --fix -c 0.8

  11. مثال وضع التصحيح: - translate -l "ko" -d: تمكين تسجيل التصحيح.
  12. حفظ السجلات إلى ملفات: translate -l "ko" -s
  13. تصحيح الكونسول وتصحيح الملفات: translate -l "ko" -d -s
  14. الترجمة بدون إضافة إخلاءات مسؤولية الترجمة الآلية إلى النتائج: translate -l "ko" --no-disclaimer

  15. ترحيل روابط الدفاتر لترجمات كورية (تحديث الروابط إلى الدفاتر المترجمة عند توفرها):    migrate-links -l "ko"

  15. ترحيل الروابط مع التجربة الجافة (بدون كتابة ملفات):    migrate-links -l "ko" --dry-run

  16. تحديث الروابط فقط عند وجود دفاتر مترجمة (عدم الرجوع للأصلية):    migrate-links -l "ko" --no-fallback-to-original

  17. معالجة جميع اللغات مع مطالبة التأكيد:    migrate-links -l "all"

  18. معالجة جميع اللغات والتأكيد التلقائي:    migrate-links -l "all" -y
  19. حفظ السجلات في ملفات لترحيل الروابط:    migrate-links -l "ko ja" -s

  20. تخصيص النص التوصيلي لقسم اللغات في README باستخدام رابط الريبو الخاص بك:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### أمثلة التقييم

> [!WARNING]  
> **ميزة بيتا**: وظيفة التقييم حالياً في مرحلة بيتا. تم إصدار هذه الميزة لتقييم الوثائق المترجمة، وطُرق التقييم والتنفيذ التفصيلي لا تزال قيد التطوير وقابلة للتغيير.

  1. تقييم الترجمات الكورية: evaluate -l "ko"

  2. تقييم بعتبة ثقة مخصصة: evaluate -l "ko" -c 0.8

  3. تقييم سريع (قاعدي فقط): evaluate -l "ko" -f

  4. تقييم عميق (معتمد على LLM فقط): evaluate -l "ko" -D
