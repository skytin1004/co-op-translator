## 安全性

Microsoft 非常重視我們軟件產品和服務的安全性，這包括所有通過我們的 GitHub 組織管理的原始碼庫，這些組織包括 [Microsoft](https://github.com/Microsoft)、[Azure](https://github.com/Azure)、[DotNet](https://github.com/dotnet)、[AspNet](https://github.com/aspnet) 和 [Xamarin](https://github.com/xamarin)。

如果您認為在任何 Microsoft 擁有的原始碼庫中發現了符合 [Microsoft 安全漏洞定義](https://aka.ms/security.md/definition) 的安全漏洞，請按照下文說明向我們報告。

## 報告安全問題

**請勿通過公共 GitHub 問題報告安全漏洞。**

請改為通過 Microsoft 安全回應中心 (MSRC) 於 [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) 報告。

如果您希望不登錄即可提交，請發送電子郵件至 [secure@microsoft.com](mailto:secure@microsoft.com)。如果可能，請使用我們的 PGP 金鑰加密您的訊息；請從 [Microsoft 安全回應中心 PGP 金鑰頁面](https://aka.ms/security.md/msrc/pgp) 下載。

您應該會在 24 小時內收到回覆。如果因某種原因未收到，請透過電子郵件進行後續聯絡，以確保我們已收到您的原始訊息。更多資訊可於 [microsoft.com/msrc](https://www.microsoft.com/msrc) 查詢。

請附上以下所要求的資訊（盡可能多提供），以幫助我們更好地了解問題的性質和範圍：

  * 問題類型（例如緩衝區溢出、SQL 注入、跨站腳本等）
  * 與該問題有關的原始碼檔案完整路徑
  * 受影響原始碼的位置（標籤／分支／提交或直接 URL）
  * 復現該問題所需的特殊配置
  * 復現問題的逐步指示
  * 概念驗證或利用程式碼（若可能）
  * 問題的影響，包括攻擊者可能如何利用此問題

這些資訊將幫助我們更快地審核您的報告。

如果您是為漏洞賞金計劃報告，完整的報告能有助於獲得較高的獎金。請參閱我們的 [Microsoft 漏洞賞金計劃](https://aka.ms/security.md/msrc/bounty) 頁面，了解我們現有計劃的更多詳情。
