# Command reference

The **Co-op Translator** CLI कई विकल्प प्रदान करता है ताकि अनुवाद प्रक्रिया को अनुकूलित किया जा सके:

Command                                       | Description
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | आपके प्रोजेक्ट को निर्दिष्ट भाषाओं में अनुवादित करता है। उदाहरण: translate -l "es fr de" स्पेनिश, फ्रेंच, और जर्मन में अनुवाद करता है। सभी समर्थित भाषाओं में अनुवाद के लिए translate -l "all" का उपयोग करें।
translate -l "language_codes" -u              | अनुवादों को अपडेट करता है मौजूदा अनुवादों को हटाकर उन्हें फिर से बनाता है। चेतावनी: यह निर्दिष्ट भाषाओं के लिए सभी वर्तमान अनुवादों को हटा देगा।
translate -l "language_codes" -img            | केवल छवि फ़ाइलों का अनुवाद करता है।
translate -l "language_codes" -md             | केवल Markdown फ़ाइलों का अनुवाद करता है।
translate -l "language_codes" -nb             | केवल Jupyter नोटबुक फ़ाइलों (.ipynb) का अनुवाद करता है।
translate -l "language_codes" --fix           | पिछली मूल्यांकन परिणामों के आधार पर कम आत्मविश्वास स्कोर वाली फाइलों को पुनः अनुवादित करता है।
translate -l "language_codes" -d              | विस्तृत लॉगिंग के लिए डिबग मोड सक्षम करता है।
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ के अंतर्गत DEBUG-स्तर की लॉग फाइलों को सहेजता है (कंसोल -d द्वारा नियंत्रित रहता है)
translate -l "language_codes" -r "root_dir"   | प्रोजेक्ट के रूट डायरेक्टरी को निर्दिष्ट करता है
translate -l "language_codes" -f              | छवि अनुवाद के लिए फास्ट मोड का उपयोग करता है (गुणवत्ता और संरेखण में थोड़े से अंतर के साथ लगभग 3 गुना तेजी से प्लॉटिंग)।
translate -l "language_codes" -y              | सभी प्रॉम्प्ट्स को स्वचालित रूप से पुष्टि करता है (CI/CD पाइपलाइनों के लिए उपयोगी)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | मशीन अनुवाद अस्वीकरण अनुभाग को अनुवादित markdown और नोटबुकों में जोड़ने या न जोड़ने के लिए सक्षम या अक्षम करता है (डिफ़ॉल्ट: सक्षम)।
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | README भाषाओं अनुभाग सलाहकार (स्पर्ष चेकआउट) को आपके रिपोजिटरी URL के साथ व्यक्तिगत बनाता है।
translate -l "language_codes" --help          | CLI में उपलब्ध कमांड्स के साथ मदद विवरण
evaluate -l "language_code"                  | किसी विशिष्ट भाषा के लिए अनुवाद गुणवत्ता का मूल्यांकन करता है और आत्मविश्वास स्कोर प्रदान करता है
evaluate -l "language_code" -c 0.8           | कस्टम आत्मविश्वास सीमा के साथ अनुवादों का मूल्यांकन करता है
evaluate -l "language_code" -f               | तेज़ मूल्यांकन मोड (केवल नियम-आधारित, कोई LLM नहीं)
evaluate -l "language_code" -D               | गहरा मूल्यांकन मोड (केवल LLM-आधारित, अधिक व्यापक लेकिन धीमा)
evaluate -l "language_code" --save-logs, -s  | <root_dir>/logs/ के अंतर्गत DEBUG-स्तर की लॉग फाइलों को सहेजता है
migrate-links -l "language_codes"             | अनुवादित Markdown फ़ाइलों को पुनः संसाधित करता है ताकि नोटबुक (.ipynb) के लिंक अपडेट किए जा सकें। उपलब्ध होने पर अनुवादित नोटबुक पसंद करता है; अन्यथा मूल नोटबुक पर fallback कर सकता है।
migrate-links -l "language_codes" -r          | प्रोजेक्ट रूट डायरेक्टरी निर्दिष्ट करता है (डिफ़ॉल्ट: वर्तमान डायरेक्टरी)।
migrate-links -l "language_codes" --dry-run   | बिना बदलाव लिखे दिखाता है कि कौन सी फाइलें बदलेंगी।
migrate-links -l "language_codes" --no-fallback-to-original | जब अनुवादित नोटबुक नहीं हों तो मूल नोटबुक के लिंक पुनः न लिखें (केवल जब अनुवादित मौजूद हो तो अपडेट करें)।
migrate-links -l "language_codes" -d          | विस्तृत लॉगिंग के लिए डिबग मोड सक्षम करता है।
migrate-links -l "language_codes" --save-logs, -s | <root_dir>/logs/ के अंतर्गत DEBUG-स्तर की लॉग फाइलों को सहेजता है
migrate-links -l "all" -y                      | सभी भाषाओं को प्रोसेस करता है और चेतावनी प्रॉम्प्ट को स्वचालित रूप से पुष्टि करता है।

## Usage examples

  1. डिफ़ॉल्ट व्यवहार (मौजूदा अनुवादों को हटाए बिना नए अनुवाद जोड़ें):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. केवल नए कोरियाई छवि अनुवाद जोड़ें (कोई मौजूदा अनुवाद नहीं हटाए जाएंगे):    translate -l "ko" -img

  3. सभी कोरियाई अनुवाद अपडेट करें (चेतावनी: यह पुनः अनुवाद करने से पहले सभी मौजूदा कोरियाई अनुवाद हटा देगा):    translate -l "ko" -u

  4. केवल कोरियाई छवियों को अपडेट करें (चेतावनी: यह पुनः अनुवाद करने से पहले सभी मौजूदा कोरियाई छवियां हटा देगा):    translate -l "ko" -img -u

  5. अन्य अनुवादों को प्रभावित किए बिना कोरियाई के लिए नए मार्कडाउन अनुवाद जोड़ें:    translate -l "ko" -md

  6. पिछले मूल्यांकन परिणामों के आधार पर कम आत्मविश्वास वाले अनुवाद सुधारें: translate -l "ko" --fix

  7. केवल विशिष्ट फाइलों के लिए कम आत्मविश्वास वाले अनुवाद सुधारें (मार्कडाउन): translate -l "ko" --fix -md

  8. केवल विशिष्ट फाइलों के लिए कम आत्मविश्वास वाले अनुवाद सुधारें (छवियाँ): translate -l "ko" --fix -img

  9. छवि अनुवाद के लिए फास्ट मोड का उपयोग करें:    translate -l "ko" -img -f

  10. कस्टम सीमा के साथ कम आत्मविश्वास वाले अनुवाद सुधारें: translate -l "ko" --fix -c 0.8

  11. डिबग मोड का उदाहरण: - translate -l "ko" -d: डिबग लॉगिंग सक्षम करें।
  12. लॉग फाइलों को सहेजें: translate -l "ko" -s
  13. कंसोल DEBUG और फाइल DEBUG: translate -l "ko" -d -s
  14. आउटपुट में मशीन अनुवाद अस्वीकरण न जोड़ें: translate -l "ko" --no-disclaimer

  15. कोरियाई अनुवादों के लिए नोटबुक लिंक माइग्रेट करें (जब उपलब्ध हो तो अनुवादित नोटबुक के लिंक अपडेट करें):    migrate-links -l "ko"

  15. ड्राई-रन के साथ लिंक माइग्रेट करें (कोई फाइल लेखन नहीं):    migrate-links -l "ko" --dry-run

  16. केवल तब लिंक अपडेट करें जब अनुवादित नोटबुक मौजूद हों (मूल लिंक पर fallback न करें):    migrate-links -l "ko" --no-fallback-to-original

  17. पुष्टि प्रॉम्प्ट के साथ सभी भाषाओं को प्रोसेस करें:    migrate-links -l "all"

  18. सभी भाषाओं को प्रोसेस करें और स्वतः पुष्टि करें:    migrate-links -l "all" -y
  19. migrate-links के लिए लॉग फाइलों को सहेजें:    migrate-links -l "ko ja" -s

  20. अपने रिपोजिटरी URL के साथ README भाषाओं सलाहकार को व्यक्तिगत बनाएं:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Evaluation Examples

> [!WARNING]  
> **Beta Feature**: मूल्यांकन कार्यक्षमता वर्तमान में बीटा में है। यह फीचर अनुवादित दस्तावेज़ों का मूल्यांकन करने के लिए जारी किया गया था, और मूल्यांकन विधियां और विस्तार से कार्यान्वयन अभी विकासाधीन हैं और परिवर्तन के अधीन हैं।

  1. कोरियाई अनुवादों का मूल्यांकन करें: evaluate -l "ko"

  2. कस्टम आत्मविश्वास सीमा के साथ मूल्यांकन करें: evaluate -l "ko" -c 0.8

  3. तेज़ मूल्यांकन (केवल नियम-आधारित): evaluate -l "ko" -f

  4. गहरा मूल्यांकन (केवल LLM-आधारित): evaluate -l "ko" -D
