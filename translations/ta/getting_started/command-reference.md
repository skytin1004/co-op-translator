# கட்டளை குறிப்புகள்

**Co-op Translator** CLI மொழிபெயர்ப்பு செயல்முறையை தனிப்பயனாக்க பல விருப்பங்களை வழங்குகிறது:

கட்டளை                                       | விளக்கம்
----------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"                 | உங்கள் திட்டத்தை குறிப்பிட்ட மொழிகளில் மொழிபெயர்க்கின்றது. உதாரணம்: translate -l "es fr de" என்பதனால் ஸ்பானிஷ், பிரெஞ்சு மற்றும் ஜெர்மன் மொழிகளில் மொழிபெயர்க்கப்படும். translate -l "all" என்பதைக் கொண்டு அனைத்து ஆதரவு பெற்ற மொழிகளிலும் மொழிபெயர்க்கலாம்.
translate -l "language_codes" -u              | மொழிபெயர்ப்புகளை இயல்புநிலையில் நீக்கி மீண்டும் உருவாக்கி புதுப்பிக்கின்றது. எச்சரிக்கை: இதில் குறிப்பிட்ட மொழிகளுக்கான அனைத்து உள்ளூர்மொழிபெயர்ப்புகளும் நீக்கப்படும்.
translate -l "language_codes" -img            | படங்கள் கோப்புகளுக்கு மட்டுமே மொழிபெயர்க்கின்றது.
translate -l "language_codes" -md             | Markdown கோப்புகளுக்கு மட்டுமே மொழிபெயர்க்கின்றது.
translate -l "language_codes" -nb             | Jupyter நோட்புக் கோப்புகள் (.ipynb) க்கு மட்டுமே மொழிபெயர்க்கின்றது.
translate -l "language_codes" --fix           | முந்தைய மதிப்பீட்டின் அடிப்படையில் குறைந்த நம்பகத் தன்மை மதிப்பெண்கள் உள்ள கோப்புகளை மறுமொழிபெயர்க்கின்றது.
translate -l "language_codes" -d              | விரிவான பதிவு இட அனுமதிக்கும் டிபக் முறையை இயக்கு.
translate -l "language_codes" --save-logs, -s | <root_dir>/logs/ என்ற கோப்புறையில் DEBUG நிலை பதிவுகளை சேமிக்கவும் (கான்சோல் இன்னும் -d மூலம் கட்டுப்படுகிறது)
translate -l "language_codes" -r "root_dir"   | திட்டத்தின் ரூட் கோப்புறையை குறிப்பது
translate -l "language_codes" -f              | பட மொழிபெயர்ப்பிற்கு விரைவான முறையை பயன்படுத்துவது (த гэтு தரம் மற்றும் அமைவிற்கு சிறிய பாதிப்புடன் 3 மடங்கு வேகமாக வரைபடம் வரைப்பு).
translate -l "language_codes" -y              | அனைத்து கேள்விகளையும் தானாக உறுதிப்படுத்து (CI/CD குழாய்களுக்கு உதவும்)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | மொழிபெயர்க்கப்பட்ட markdown மற்றும் நோட்புக் பாகங்களில் இயந்திர மொழிபெயர்ப்பு ஒப்புகை பிரிவு சேர்க்க/அனுமதிக்க (இயல்பு: இயக்கு).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | உங்கள் GitHub App URL மூலம் README மொழிகள் பகுதிக்கு தனிப்பயன் ஆலோசனை வழங்கும் (சில தெரியும் பாகம்).
translate -l "language_codes" --help          | CLI உள்ள தனித்துவமான கட்டளை உதவி விவரங்கள்
evaluate -l "language_code"                  | ஒரு குறிப்பிட்ட மொழிக்கான மொழிபெயர்ப்பு தரத்தை மதிப்பிடுகிறது மற்றும் நம்பகத் தன்மை மதிப்பெண்களை வழங்குகிறது
evaluate -l "language_code" -c 0.8           | தனிப்பயன் நம்பகத் தன்மை உடைய மொழிபெயர்ப்பை மதிப்பிடுகின்றது
evaluate -l "language_code" -f               | விரைவு மதிப்பீட்டு முறை (விதி அடிப்படையிலானது மட்டும், LLM இல்லை)
evaluate -l "language_code" -D               | ஆழமான மதிப்பீட்டு முறை (LLM அடிப்படையிலானது மட்டும், தீவிரமானது ஆனால் மெதுவானது)
evaluate -l "language_code" --save-logs, -s  | DEBUG நிலை பதிவுகளை <root_dir>/logs/ உட்பட கோப்புகளில் சேமிக்கவும்
migrate-links -l "language_codes"             | மொழிபெயர்க்கப்பட்ட Markdown கோப்புகளை மீண்டும் செயலாக்கி நோட்புக் (.ipynb) நிலையிலான இணைப்புகளை புதுப்பிக்கிறது. மொழிபெயர்க்கப்பட்ட நோட்புக் இருந்தால் அவற்றை முன்னுரிமை அளிக்கின்றது, இல்லையெனில் மூல நோட்புக் இணைப்புகளுக்கு மறு fallback செய்யலாம்.
migrate-links -l "language_codes" -r          | திட்ட ரூட் கோப்புறையை குறிப்பது (இயல்பு: தற்போதைய கோப்புறை).
migrate-links -l "language_codes" --dry-run   | எந்த கோப்புகள் மாற்றப்படுமோ அவற்றை காண்பி கோப்பு எழுத்துக்கள் செய்யாமல்.
migrate-links -l "language_codes" --no-fallback-to-original | மொழிபெயர்க்கப்பட்ட நோட்புக் கிடைக்காமல் இருந்தால் மூல நோட்புக் இணைப்புகளை மாற்ற வேண்டாம் (மொழிபெயர்க்கப்பட்டவை இருந்தால் மட்டும் புதுப்பிக்க).
migrate-links -l "language_codes" -d          | விரிவான பதிவு இட அனுமதிக்கும் டிபக் முறையை இயக்கு.
migrate-links -l "language_codes" --save-logs, -s | DEBUG நிலை பதிவுகளை <root_dir>/logs/ இடத்தில் சேமிக்கவும்
migrate-links -l "all" -y                      | அனைத்து மொழிகளையும் செயலாக்கி எச்சரிக்கை கேள்வியை தானாக உறுதிப்படுத்து.

## பயன்பாட்டு உதாரணங்கள்

  1. இயல்புநிலை நடத்தைகள் (உள்ளூர்மொழிபெயர்ப்புகளை நீக்காமல் புதிய மொழிபெயர்ப்புகளைச் சேர்க்க):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. புதிய கொரிய பட மொழிபெயர்ப்புகளை மட்டும் சேர்க்க (அதிகால உள்ளூர் மொழிபெயர்ப்புகளை நீக்காதது):    translate -l "ko" -img

  3. அனைத்து கொரிய மொழிபெயர்ப்புகளை புதுப்பிக்க (எச்சரிக்கை: இப்போது உள்ள அனைத்து கொரிய மொழிபெயர்ப்புகளும் நீக்கப்படுகின்றன):    translate -l "ko" -u

  4. கொரிய படங்களை மட்டும் புதுப்பிக்க (எச்சரிக்கை: இப்போது உள்ள அனைத்து கொரிய படங்களும் நீக்கப்படுகின்றன):    translate -l "ko" -img -u

  5. மற்ற மொழிபெயர்ப்புகளை பாதிக்காமல் கொரிய மார்க்டவுன் மொழிபெயர்ப்புகளைச் சேர்க்க:    translate -l "ko" -md

  6. முந்தைய மதிப்பீடும் அடிப்படையில் குறைந்த நம்பகத் தன்மையுள்ள மொழிபெயர்ப்புகளை சீரமைக்க: translate -l "ko" --fix

  7. குறிப்பிட்ட கோப்புகளுக்கான குறைந்த நம்பகத் தன்மை மொழிபெயர்ப்புகளை மட்டும் சீரமைக்க (markdown): translate -l "ko" --fix -md

  8. குறிப்பிட்ட கோப்புகளுக்கான குறைந்த நம்பகத் தன்மை மொழிபெயர்ப்புகளை மட்டும் சீரமைக்க (படங்கள்): translate -l "ko" --fix -img

  9. பட மொழிபெயர்க்க விரைவான முறையை பயன்படுத்த:    translate -l "ko" -img -f

  10. தனிப்பயன் எல்லையை வைத்து குறைந்த நம்பகத் தன்மை மொழிபெயர்ப்புகளை சீரமைக்க: translate -l "ko" --fix -c 0.8

  11. டிபக் முறையின் உதாரணம்: - translate -l "ko" -d: டிபக் பதிவு இயக்கு.
  12. பதிவுகளை கோப்புகளில் சேமிக்க: translate -l "ko" -s
  13. கான்சோல் DEBUG மற்றும் கோப்பு DEBUG: translate -l "ko" -d -s
  14. இயந்திர மொழிபெயர்ப்பு ஒப்புகை சேர்க்காமல் மொழிபெயர்க்க: translate -l "ko" --no-disclaimer

  15. கொரிய மொழிபெயர்ப்புகளுக்கான நோட்புக் இணைப்புகளை மாற்றுக (மொழிபெயர்க்கப்பட்ட நோட்புக் கிடைத்தால் இணைப்புகளை புதுப்பிக்க):    migrate-links -l "ko"

  15. எழுதாமல் மாற்றங்களை காட்டுக:    migrate-links -l "ko" --dry-run

  16. மொழிபெயர்க்கப்பட்ட நோட்புக் இருந்தால் மட்டும் இணைப்புகளை புதுப்பிக்க (மூலங்கள் fallback செய்ய வேண்டாம்):    migrate-links -l "ko" --no-fallback-to-original

  17. அனைத்து மொழிகளையும் உறுதிப்படுத்தி செயலாக்கு:    migrate-links -l "all"

  18. அனைத்து மொழிகளையும் செயலாக்கி தானாக உறுதிப்படுத்து:    migrate-links -l "all" -y
  19. migrate-links க்கான பதிவுகளை கோப்புகளில் சேமிக்க:    migrate-links -l "ko ja" -s

  20. README மொழி அறிவுறுத்தலுக்கு உங்கள் repo URL ஐ தனிப்பயனாக்குக:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### மதிப்பீட்டு உதாரணங்கள்

> [!WARNING]  
> **பீட்டா அம்சம்**: மதிப்பீட்டு செயல்பாடு தற்போது பீட்டாவில் உள்ளது. இந்த அம்சம் மொழிபெயர்க்கப்பட்ட ஆவணங்களை மதிப்பீடு செய்ய வெளியிடப்பட்டது, மற்றும் மதிப்பீட்டு முறைகள் மற்றும் விரிவான செயல்படுத்தல் இன்னும் உருவாக்கத்தின் கீழ் உள்ளது மற்றும் மாற்றம் ஏற்படும்.

  1. கொரிய மொழிபெயர்ப்புகளை மதிப்பீடு செய்க: evaluate -l "ko"

  2. தனிப்பயன் நம்பகத்தன்மை எல்லையை வைத்து மதிப்பீடு செய்ய: evaluate -l "ko" -c 0.8

  3. விரைவு மதிப்பீடு (விதி அடிப்படையானது மட்டும்): evaluate -l "ko" -f

  4. ஆழமான மதிப்பீடு (LLM அடிப்படையானது மட்டும்): evaluate -l "ko" -D
