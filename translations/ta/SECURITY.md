## பாதுகாப்பு

Microsoft எங்கள் மென்பொருள் தயாரிப்புகள் மற்றும் சேவைகளின் பாதுகாப்பை முக்கியமாகக் கருதுகிறது, இதில் எங்கள் GitHub அமைப்புகள் மூலம் நிர்வகிக்கப்படும் அனைத்து மூலக் கோப்பு தொகுப்புகளும் அடங்கும், இதில் [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) மற்றும் [Xamarin](https://github.com/xamarin) உள்ளன.

நீங்கள் எந்த Microsoft-உரிமையுள்ள தொகுப்பில் [Microsoft-இன் பாதுகாப்பு பலவீனத் தேர்வுக்கு](https://aka.ms/security.md/definition) பொருந்தும் பாதுகாப்பு பலவீனத்தை கண்டுபிடித்ததாக நினைத்தால், கீழே விளக்கப்பட்டுள்ளபடி அதை எங்களுக்கு அறிக்கை செய்யவும்.

## பாதுகாப்பு பிரச்சினைகளை அறிக்கை செய்தல்

**பொதுவான GitHub பிரச்சினைகள் மூலம் பாதுகாப்பு பலவீனங்களை அறிக்கை செய்ய வேண்டாம்.**

மாறாக, Microsoft பாதுகாப்பு பதிலளிப்பு மையத்துக்கு (MSRC) [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) இல் அதை அறிக்கை செய்யவும்.

உள்நுழைவதால் விலகி சமர்ப்பிக்க விரும்பினால், [secure@microsoft.com](mailto:secure@microsoft.com) என்ற மின்னஞ்சலுக்கு அனுப்பவும். சாத்தியமானால், எங்கள் PGP சாவியுடன் உங்கள் செய்தியை குறியாக்கம் செய்யவும்; அதை [Microsoft பாதுகாப்பு பதிலளிப்பு மையம் PGP சாவி பக்கம்](https://aka.ms/security.md/msrc/pgp) இருந்து பதிவிறக்கம் செய்யவும்.

நீங்கள் 24 மணி நேரத்துக்குள் பதிலைப் பெற வேண்டும். ஏதோ காரணத்தால் பெறவில்லை என்றால், உங்கள் தொடக்க செய்தி பெற்றுள்ளதா என்பதை உறுதிப்படுத்த மின்னஞ்சலில் தொடர்ந்து தொடர்பு கொள்ளவும். கூடுதல் தகவல்கள் [microsoft.com/msrc](https://www.microsoft.com/msrc) இல் கிடைக்கின்றன.

பயன்பாட்ட அகலம் மற்றும் இயல்பு பற்றி நமக்கு بہتر புரிதல் அளிக்க கீழ்க்கண்ட கேட்கப்பட்ட தகவல்கள் (நீங்கள் வழங்கக்கூடிய जितவ்வளவு) உள்ளடக்கவும்:

  * பிரச்சினையின் வகை (எ.கா. பப்ஃபர் ஓவர்கள் ஃப்லோ, SQL ஊடுருவல், குறுக்குவலை குறியீடு, மற்றும் பிற)
  * பிரச்சினையின் வெளிப்பாட்டுக்கு தொடர்புடைய மூலக் கோப்பு(கள்) முழு பாதைகள்
  * பாதிக்கப்பட்ட மூலக் கோப்பின் இடம் (tag/branch/commit அல்லது நேரடி URL)
  * பிரச்சினையை மீண்டும் உருவாக்க தேவையான எந்தவொரு சிறப்பு கட்டமைப்புகளும்
  * பிரச்சினையை மீண்டும் உருவாக்க படி நடவடிக்கை வழிமுறைகள்
  * சான்று-கருத்து அல்லது குறியீடு (சாத்தியமானால்)
  * பிரச்சினையின் தாக்கம், அதனை எப்படி தாக்குபவர் பயன்படுத்தக்கூடும் என்பதுசহ

இந்தத் தகவல் உங்கள் அறிக்கையை வேகமாக விசாரணை செய்ய உதவும்.

நீங்கள் பக் பவுண்டி (bug bounty)க்காக அறிக்கை அளிப்பதாக இருந்தால், முழுமையான அறிக்கைகள் அதிக பவுண்டி விருதிற்கு உதவும். எங்கள் [Microsoft பக் பவுண்டி திட்டம்](https://aka.ms/security.md/msrc/bounty) பக்கத்தை விவரங்களுக்கு பார்வையிடவும்.

## விரும்பிய மொழிகள்

எங்கள் அனைத்து தொடர்புகளும் ஆங்கிலத்தில் இருக்க அவசியம்.

## கொள்கை

Microsoft [ஒத்துழைப்பு பாதுகாப்பு வெளியீடு](https://aka.ms/security.md/cvd) கொள்கையை பின்பற்றுகிறது.
