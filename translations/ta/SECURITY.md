## பாதுகாப்பு

Microsoft எங்கள் மென்பொருள் தயாரிப்புகள் மற்றும் சேவைகளின் பாதுகாப்பை மிகுந்த முக்கியத்துவத்துடன் எடுத்துக்கொள்கிறது, இதில் எங்கள் GitHub நிறுவனங்கள் வழியாக நிர்வகிக்கப்படும் அனைத்து மூலக் குறியீடு சேமிப்பிடங்களும் அடங்கும், அவைகளில் [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) மற்றும் [Xamarin](https://github.com/xamarin) உள்ளன.

நீங்கள் எந்த Microsoft-ஆக்கப்படுத்தப்பட்ட சேமிப்பிடத்தில் [Microsoft-இன் பாதுகாப்பு குறைபாடு வரையறைக்கு](https://aka.ms/security.md/definition) உட்பட்ட பாதுகாப்பு குறைபாட்டை கண்டுபிடித்திருப்பதாக நினைக்கினால், கீழே தரப்பட்டுள்ளவாறு எங்களிடம் அதனை அறிக்கை செய்யவும்.

## பாதுகாப்பு பிரச்சினைகளை அறிக்கை செய்வது

**தயவுசெய்து பொதுஉலக GitHub பிரச்சினைகள் மூலம் பாதுகாப்பு குறைபாடுகளை அறிக்கை செய்ய வேண்டாம்.**

பதிலாக, Microsoft பாதுகாப்பு பதிலளிக்கும் மையத்தில் (MSRC) [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) இதனை அறிக்கை செய்யவும்.

உள்நுழையாமலேயே சமர்ப்பிக்க விரும்பினால், [secure@microsoft.com](mailto:secure@microsoft.com) என்ற முகவரிக்கு மின்னஞ்சல் அனுப்பவும். முடியுமானால், எங்கள் PGP சாவி மூலம் உங்கள் செய்தியை குறியாக்கம் செய்யவும்; [Microsoft பாதுகாப்பு பதிலளிக்கும் மையத்தின் PGP சாவி பக்கம்](https://aka.ms/security.md/msrc/pgp)யில் இருந்து அதை பதிவிறக்கம் செய்யலாம்.

நீங்கள் 24 மணி நேரத்துக்குள் பதிலைப் பெற வேண்டும். ஏதேனும் காரணத்தால் பெறவில்லையெனில், உங்கள் முதற்படி செய்தி எங்களுக்கு வந்ததா என்பதை உறுதி செய்வதற்காக மின்னஞ்சலில் தொடர்ந்து தொடர்பு கொள்ளவும். கூடுதல் தகவல் [microsoft.com/msrc](https://www.microsoft.com/msrc) இல் காணப்படும்.

ஒரு சாத்தியமான பிரச்சினையின் இயல் மற்றும் பரப்பினை நன்கு புரிந்து கொள்ள, கீழ்க்காணும் கோரப்பட்ட தகவலை (மிகவும் அளவு அளித்து) வழங்கவும்:

  * பிரச்சினையின் வகை (எ.கா., பப்பர் ஓவர்ஃபிளோ, SQL ஊழல், குறுக்கு இடற்பயன்பாடு, போன்றவை)
  * பிரச்சினை போன்ற மூலக் கோப்புகளின் முழு பாதைகள்
  * பாதிக்கப்பட்ட மூலக் குறியீட்டின் இடம் (டேக்/கிளை/கமிட்டோ அல்லது நேரடி URL)
  * பிரச்சினையை மீண்டும் உருவாக்க எந்தவொரு சிறப்பு கட்டமைப்பும் தேவைப்பட்டால்
  * பிரச்சினையை மீண்டும் உருவாக்கும் படிகள் படி வழிகாட்டுதல்கள்
  * சான்று-கூறு அல்லது குறியீடு (இயலுமானால்)
  * பிரச்சினையின் தாக்கம், எப்படி ஒரு தாக்குதலை செல்வாக்கு செய்யலாம் என்பதைக் குறிக்கும்

இந்த தகவல் எங்கள் அறிக்கையை விரைவாகத் துவக்க உதவும்.

நீங்கள் பக் பவுண்டிக்கு அறிக்கை செய்கிறீர்கள் என்றால், முழுமையான அறிக்கைகள் அதிக பவுண்டி விருப்பத்துக்கு உதவலாம். எங்கள் [Microsoft பக் பவுண்டி நிகழ்ச்சி](https://aka.ms/security.md/msrc/bounty) பக்கத்தை மூலமாக ஆதரவளிக்கப்படும் நிகழ்ச்சிகள் பற்றிய கூடுதல் விவரங்களுக்கு பார்வையிடவும்.
