## முன்னோட்டம்

**Co-op Translator** உங்கள் கல்வி GitHub உள்ளடக்கத்தை பல மொழிகளில் எளிதாக உள்ளூரிட உதவுகிறது.  
நீங்கள் உங்கள் Markdown கோப்புகள், படங்கள் அல்லது நோட்புக் புதுப்பித்தால், மொழிபெயர்ப்புகள் தானாக ஒத்திசைவாகி, உங்கள் உள்ளடக்கம் உலகமெங்கும் கற்றலாளர்களுக்கு துல்லியமாகவும் புதுப்பிக்கப்பட்டதாகவும் இருக்கும்.

மொழிபெயர்க்கப்பட்ட உள்ளடக்கம் எவ்வாறு ஒழுங்குபடுத்தப்பட்டுள்ளது என்பதன் உதாரணம்:

![உதாரணம்](../../translated_images/ta/translation-ex.0c8aa6a7ee0aad2b.webp)

## மொழிபெயர்ப்பு நிலை எவ்வாறு நிர்வகிக்கப்படுகிறது

Co-op Translator மொழிபெயர்க்கப்பட்ட உள்ளடக்கத்தை **பரிணாமமான மென்பொருள் வினியோகங்களாக**  
நிறுத்துவதை அல்லாமல் நிர்வகிக்கிறது.

மேலும் இந்த கருவி மொழிபெயர்க்கப்பட்ட Markdown, படங்கள், மற்றும் நோட்புக் நிலையை  
**மொழியின்படி பகுத்து metadata** மூலம் கண்காணிக்கிறது.

இந்த வடிவமைப்பு Co-op Translator-க்கு அனுமதிக்கிறது:

- பழைய மொழிபெயர்ப்புகளை நம்பகமாக கண்டறிதல்
- Markdown, படங்கள் மற்றும் நோட்பூக்களை நெறியுடன் நடத்தல்
- பெரிய, வேகமாக நகரும், பலமொழி களஞ்சியங்களில் பாதுகாப்பாக விரிவாக்கம்

மொழிபெயர்ப்புகளை நிர்வகிக்கப்படும் வினியோகங்களாக மாதிரியாக்குவதன் மூலம்,  
மொழிபெயர்ப்பு பணிகள் இயல்பாக நவீன மென்பொருள் சார்பு மற்றும் வினியோக மேலாண்மை நடைமுறைகளுடன் இணைகின்றன.

→ [மொழிபெயர்ப்பு நிலை கையாள்வது எப்படி](https://techcommunity.microsoft.com/blog/azuredevcommunityblog/rethinking-documentation-translation-treating-translations-as-versioned-software/4491755)


## விரைவு தொடக்கம்

```bash
# ஒரு மெய்நிகர் சூழலை உருவாக்கி செயல்படுத்தவும் (பரிந்துரைக்கப்படுகிறது)
python -m venv .venv
# விண்டோசுகள்
.venv\Scripts\activate
# மேகோஎஸ்/லினக்ஸ்
source .venv/bin/activate
# தொகுப்பை நிறுவவும்
pip install co-op-translator
# மொழிபெயர்
translate -l "ko ja fr" -md
```

Docker:

```bash
# GHCR இருந்து பொதுப் படத்தை ஏற்று
docker pull ghcr.io/azure/co-op-translator:latest
# தற்போதைய கோப்புறை பொருத்தப்பட்டு .env வழங்கப்பட்டு இயக்கவும் (Bash/Zsh)
docker run --rm -it --env-file .env -v "${PWD}:/work" ghcr.io/azure/co-op-translator:latest -l "ko ja fr" -md
```

## குறைந்தபட்ச அமைப்பு

1. ஆதரிக்கப்பட்ட Python பதிப்பைப் (தற்போது 3.10-3.12) நீங்கள் வைத்திருக்கிறீர்கள் என்று உறுதிப்படுத்தவும். poetry (pyproject.toml) இல் இது தானாக நடைபெறும்.
2. ஒரு `.env` கோப்பை வடிவமைப்பைப் பயன்படுத்தி உருவாக்கவும்: [.env.template](../../.env.template)
3. ஒரு LLM வழங்குநரை கட்டமைக்கவும் (Azure OpenAI அல்லது OpenAI)
4. (விருப்பம்) பட மொழிபெயர்ப்பிற்கு (`-img`), Azure AI Vision ஐ அமைக்கவும்
5. (விருப்பம்) `_1`, `_2` போன்ற suffix களைப் பயன்படுத்தி பல நம்பிக்கை பட்டியல்களை உருவாக்கலாம். ஒரே பட்டியலில் அனைத்து மாறிலிகள் suffix இல் ஒரே மாதிரியாக இருக்க வேண்டும்.
6. (பரிந்துரை) முன்னாள் மொழிபெயர்ப்புகளை அனைத்து மோதல்களையும் தவிர்க்க அழிக்கவும் (எ.கா., `translations/`)
7. (பரிந்துரை) உங்கள் READMEவில் மொழிபெயர்ப்பு பகுதியை சேர்க்கவும் [README மொழிகள் ஃடெம்ப்ளேட்](./getting_started/README_languages_template.md) மூலம்
8. பார்க்கவும்: [Azure AI-யை அமைக்கவும்](./getting_started/set-up-azure-ai.md)

## பயன்பாடு

அனைத்து ஆதரிக்கப்பட்ட வகைகளையும் மொழிபெயர்க்கவும்:

```bash
translate -l "ko ja"
```

Markdown மட்டும்:

```bash
translate -l "de" -md
```

Markdown + படங்கள்:

```bash
translate -l "pt" -md -img
```

நோட்புக் மட்டும்:

```bash
translate -l "zh" -nb
```

மேலும் விருப்பங்கள்: [கட்டளைக் குறிப்பு](./getting_started/command-reference.md)

## பொறுப்புள்ள செயற்கை அறிவு

Microsoft, எங்கள் AI தயாரிப்புகளை பொறுப்புடன் பயன்படுத்துவதற்கு வாடிக்கையாளர்களை உதவ பாடுபடுகிறது, எங்கள் கற்றல்களை பகிர்ந்து, Transparency Notes மற்றும் Impact Assessments போன்ற கருவிகள் மூலம் நம்பிக்கை அடிப்படையிலான கூட்டுறவை உருவாக்குகிறது. இந்த வளங்கள் பலுள்ளன [https://aka.ms/RAI](https://aka.ms/RAI) இல் காணலாம்.  
Microsoft உடனான பொறுப்புள்ள செயற்கை அறிவு அணுகுமுறை நியாயம், நம்பகத்தன்மை மற்றும் பாதுகாப்பு, தனியுரிமை மற்றும் பாதுகாப்பு, உட்புகுந்தல், வெளிப்படைத்தன்மை மற்றும் பொறுப்புடையும் AI விதிகளில் அடிப்படையாக அமைந்துள்ளது.

பெரும் அளவிலான இயற்கை மொழி, படம் மற்றும் பேச்சு மாதிரிகள் - இந்த மாதிரி பயன்பாட்டில் பயன்படுத்தப்படும் மாதிரிகளால் - சில நேரங்களில் நீதி இல்லாமல், நம்பமுடியாத அல்லது அவமானகரமான வழிகளில் செயல்பட முடியும், இதனால் சேதங்கள் ஏற்பட வாய்ப்பு உள்ளது. அபாயங்கள் மற்றும் வரையறைகள் பற்றி தகவல் பெற [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ஐ பரிசீலிக்கவும்.

இந்த ஆபத்துக்களை குறைக்க பரிந்துரைக்கப்படுவது, உங்கள் கட்டமைப்பில் சேதகரமான நடத்தை கண்டறிந்து தடுக்கக்கூடிய பாதுகாப்பு அமைப்பை சேர்ப்பது ஆகும். [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) ஒரு சுதந்திரமான பாதுகாப்பு தளம் வழங்குகிறது, இது பயன்பாடுகளில் மற்றும் சேவைகளில் பயனர் உருவாக்கிய மற்றும் AI உருவாக்கிய சேதகரமான உள்ளடக்கத்தை கண்டறிய முடியும். Azure AI Content Safety உரை மற்றும் படம் API-களை கொண்டுள்ளது, இது சேதகரமான உள்ளடக்கத்தைக் கண்டறிய உதவுகிறது. மேலும், நாம் உள்ளடக்க பாதுகாப்பு ஸ்டுடியாகும், இது பலவிதமான பகுதிகளிலான சேதகரமான உள்ளடக்கத்தை கண்டறிய சில எடுத்துக்காட்டு குறியீடுகளைப் பார்ப்பதற்கும், ஆராய்வதற்கும், முயற்சிப்பதற்கும் அனுமதிக்கிறது. பின்வரும் [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) இந்த சேவைக்கு கோரிக்கைகள் செய்வதற்கான வழிகாட்டலை வழங்குகிறது.

மற்றொரு முக்கிய அம்சம் அது உங்கள் முழுமையான பயன்பாட்டின் செயல்திறனை கருத்தில் கொள்ள வேண்டும். பலவித விதிவகையுடன் மற்றும் பன்மாதிரி பயன்பாடுகளோடு, செயல்திறன் என்பது உங்கள் மற்றும் உங்கள் பயனர்கள் எதிர்பார்க்கும் விதமாக அமைப்பு செயல்படும் என்பதைச் சார்ந்தது, அதில் சேதகரமான முடிவுகளை உருவாக்குதல் தவிர்க்கப்பட வேண்டும். [generation quality and risk and safety metrics](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) ஆகியவை உங்கள் பயன்பாட்டின் செயல்திறனை மதிப்பிடம்செய்ய உபயோகப்படுத்தப்படுகின்றன.

நீங்கள் உங்கள் AI பயன்பாட்டை உங்கள் அபிவிருத்தி சூழலில் [prompt flow SDK](https://microsoft.github.io/promptflow/index.html) மூலம் மதிப்பிடலாம். ஒரு சோதனை தரவுத்தொகுப்போ அல்லது இலக்கோ வழங்கப்படும் போது, உங்கள் உருவாக்க AI விண்ணப்ப வடிவங்கள் உள்ளமைக்கப்பட்ட மதிப்பீட்டாளர்களால் அல்லது உங்களுக்கேற்ற தனிப்பயன் மதிப்பீட்டாளர்களால் அளவிடப்படுகின்றன. prompt flow sdk ஐ கொண்டு உங்கள் அமைப்பை மதிப்பிட தொடங்க [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ஐ பின்பற்றலாம். மதிப்பீடு இயக்கம் இயங்கியதும், நீங்கள் [Azure AI Studio இல் முடிவுகளை பார்வையிட](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results)லாம்.

## வர்த்தக குறியீடுகள்

இந்த திட்டத்தில் திட்டங்கள், தயாரிப்புகள் அல்லது சேவைகளுக்கான வர்த்தகக் குறியீடுகள் அல்லது லோகோக்கள் இருக்கலாம். Microsoft வர்த்தகக் குறியீடுகளை அல்லது லோகோக்களை பயன்படுத்த கீழ்க்காணும் விதிமுறைகளுக்கு உடனும் பின்பற்றப்படவேண்டும்  
[Microsoft's Trademark & Brand Guidelines](https://www.microsoft.com/en-us/legal/intellectualproperty/trademarks/usage/general).  
இந்த திட்டத்தின் மாற்றிய பதிப்புகளில் Microsoft வர்த்தகக் குறியீடுகள் அல்லது லோகோக்களை பயன்படுத்துவது குழப்பத்தை உருவாக்கக்கூடாது அல்லது Microsoft ஆதரவைக் குறிக்க கூடாது.  
மூன்றாம் தரப்பு வர்த்தகக் குறியீடுகள் அல்லது லோகோக்கள் பயன்படுத்தப்படும்போது அந்த மூன்றாம் தரப்பின் கொள்கைகளுக்கு உட்பட்டதே ஆகும்.
