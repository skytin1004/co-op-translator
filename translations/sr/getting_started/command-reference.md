# Команда референт

**Co-op Translator** CLI нуди неколико опција за прилагођавање процеса превођења:

Команда                                     | Опис
--------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
translate -l "language_codes"               | Преводи ваш пројекат на одређене језике. Пример: translate -l "es fr de" преводи на шпански, француски и немачки. Користите translate -l "all" за превођење на све подржане језике.
translate -l "language_codes" -u            | Ажурира преводе брисањем постојећих и поновним креирањем. Упозорење: Ово ће обрисати све тренутне преводе за наведене језике.
translate -l "language_codes" -img          | Преводи само сликовне фајлове.
translate -l "language_codes" -md           | Преводи само Markdown фајлове.
translate -l "language_codes" -nb           | Преводи само Jupyter notebook фајлове (.ipynb).
translate -l "language_codes" --fix         | Поново преводи фајлове са ниским нивоом поузданости на основу претходних резултата процене.
translate -l "language_codes" -d            | Омогућава debug режим за детаљно логовање.
translate -l "language_codes" --save-logs, -s | Снима DEBUG ниво логова у фајлове под <root_dir>/logs/ (конзола остаје под контролом -d)
translate -l "language_codes" -р "root_dir" | Означава коренски директоријум пројекта
translate -l "language_codes" -f            | Користи брзи режим за превођење слика (до 3 пута брже цртање уз мали губитак квалитета и поређења).
translate -l "language_codes" -y            | Аутоматски потврђује све упите (корисно за CI/CD токове)
translate -l "language_codes" --add-disclaimer/--no-disclaimer | Омогућава или онемогућава додавање секције одрицања одговорности машинског превода у преведени markdown и notebook-ове (подразумевано: омогућено).
translate -l "language_codes" --repo-url "https://github.com/org/repo.git" | Персонализује саветодајни одељак за језике у README (sparse checkout) са URL-ом вашег репозиторијума.
translate -l "language_codes" --help        | приказ детаља помоћи у CLI-ју који показује доступне команде
evaluate -l "language_code"                  | Процењује квалитет превода за одређени језик и пружа поузданости резултате
evaluate -l "language_code" -c 0.8           | Процењује преводе са прилагођеним прагом поузданости
evaluate -l "language_code" -f               | Брзи режим процене (само по правилима, без LLM-а)
evaluate -l "language_code" -D               | Дубински режим процене (само заснован на LLM-у, темељнији али спорији)
evaluate -l "language_code" --save-logs, -s  | Снима DEBUG ниво логова у фајлове под <root_dir>/logs/
migrate-links -l "language_codes"             | Репроцесуира преведене Markdown фајлове ради ажурирања линкова ка notebook-овима (.ipynb). Преферира преведене notebook-ове ако су доступни; у супротном може да користи оригиналне notebook-ове.
migrate-links -l "language_codes" -r          | Назначава корен директоријум пројекта (подразумевано: тренутни директоријум).
migrate-links -l "language_codes" --dry-run   | Приказује које би се датотеке промениле без писања измена.
migrate-links -l "language_codes" --no-fallback-to-original | Не преписује линкове ка оригиналним notebook-овима кад недостају преведени (ажурира само ако постоји превод).
migrate-links -l "language_codes" -d          | Омогућава debug режим за детаљно логовање.
migrate-links -l "language_codes" --save-logs, -s | Снима DEBUG ниво логова у фајлове под <root_dir>/logs/
migrate-links -l "all" -y                      | Обрађује све језике и аутоматски потврђује упозорења.

## Примери коришћења

  1. Подразумевано понашање (додаје нове преводе без брисања постојећих):   translate -l "ko"    translate -l "es fr de" -r "./my_project"

  2. Додаје само нове корејске преводе слика (постојећи преводи се не бришу):    translate -l "ko" -img

  3. Ажурира све корејске преводе (Упозорење: Ово брише све постојеће корејске преводе пре поновног превођења):    translate -l "ko" -u

  4. Ажурира само корејске слике (Упозорење: Ово брише све постојеће корејске слике пре поновног превођења):    translate -l "ko" -img -u

  5. Додаје нове markdown преводе за корејски без утицаја на друге преводе:    translate -l "ko" -md

  6. Поправља преводе са ниским нивоом поузданости на основу претходних резултата процене: translate -l "ko" --fix

  7. Поправља преводе са ниским нивоом поузданости само за одређене фајлове (markdown): translate -l "ko" --fix -md

  8. Поправља преводе са ниским нивоом поузданости само за одређене фајлове (слике): translate -l "ko" --fix -img

  9. Користи брзи режим за превођење слика:    translate -l "ko" -img -f

  10. Поправља преводе са ниским нивоом поузданости са прилагођеним праговима: translate -l "ko" --fix -c 0.8

  11. Пример debug режима: - translate -l "ko" -d: Омогућава debug логовање.
  12. Снимање логова у фајлове: translate -l "ko" -s
  13. DEBUG на конзоли и у фајловима: translate -l "ko" -d -s
  14. Превод без додавања одрицања машинског превода у излазима: translate -l "ko" --no-disclaimer

  15. Мигрирање линкова notebook-ова за корејске преводе (ажурира линкове на преведене notebook-ове када су доступни):    migrate-links -l "ko"

  15. Мигрирање линкова уз dry-run (без писања у фајлове):    migrate-links -l "ko" --dry-run

  16. Ажурирава само линкове када преведени notebook-ови постоје (не враћа се на оригинале):    migrate-links -l "ko" --no-fallback-to-original

  17. Обрађује све језике са потврђивањем упита:    migrate-links -l "all"

  18. Обрађује све језике и аутоматски потврђује:    migrate-links -l "all" -y
  19. Снимање логова у фајлове за migrate-links:    migrate-links -l "ko ja" -s

  20. Персонализује саветодајни одељак у README са URL-ом вашег репозиторијума:
      translate -l "ko" --repo-url "https://github.com/org/repo.git"

### Примери процене

> [!WARNING]  
> **Бета Функција**: Функционалност процене је тренутно у бета стању. Ова функција је објављена ради процене преведених докумената, а методе процене и детаљна имплементација су још у развоју и подложне су променама.

  1. Процени корејске преводе: evaluate -l "ko"

  2. Процени са прилагођеним прагом поузданости: evaluate -l "ko" -c 0.8

  3. Брза процена (само правила): evaluate -l "ko" -f

  4. Дубинска процена (само LLM): evaluate -l "ko" -D
