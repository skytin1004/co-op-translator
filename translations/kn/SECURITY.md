## Security

Microsoft ನಮ್ಮ ಸಾಫ್ಟ್‌ವೇರ್ ಉತ್ಪನ್ನಗಳು ಮತ್ತು ಸೇವೆಗಳ ಸುರಕ್ಷತೆ ಬಗ್ಗೆ ಗಂಭೀರ ಕ್ರಮಗಳನ್ನ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ, ಇದರಲ್ಲಿ ನಮ್ಮ GitHub ಸಂಸ್ಥೆಗಳ ಮೂಲಕ ನಿರ್ವಹಿಸಲ್ಪಡುವ ಎಲ್ಲಾ ಮೂಲ ಕೋಡ್ ಸಂಗ್ರಹಾಲಯಗಳು ಸೇರಿವೆ, ಅವುಗಳಲ್ಲಿ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) ಮತ್ತು [Xamarin](https://github.com/xamarin) ಇದ್ದಾರೆ.

ನೀವು ಯಾವುದಾದರೂ Microsoft ಮಾಲೀಕತ್ವದಲ್ಲಿರುವ ಸಂಗ್ರಹಾಲಯದಲ್ಲಿ [Microsoft ನ ಸುರಕ್ಷತಾ ದೋಷದ ವ್ಯಾಖ್ಯಾನ](https://aka.ms/security.md/definition) ಗೆ ಹೊಂದಿಕೆಯಾಗುವ ಸುರಕ್ಷತಾ ದುರ್ಬಲತೆಯನ್ನು ಕಂಡುಹಿಡಿದಿದ್ದೀರಿ ಎಂದು ನಂಬುವಿದ್ದರೆ, ದಯವಿಟ್ಟು ಕೆಳಗಿನಂತೆ ನಮಗೆ ವರದಿ ಮಾಡಿ.

## Reporting Security Issues

**ದಯವಿಟ್ಟು ಸಾರ್ವಜನಿಕ GitHub ಸಮಸ್ಯೆಗಳ ಮೂಲಕ ಸುರಕ್ಷತಾ ದುರ್ಬಲತೆಗಳನ್ನು ವರದಿ ಮಾಡಬೇಡಿ.**

ಬದಲಿಗೆ, ದಯವಿಟ್ಟು Microsoft Security Response Center (MSRC) ಗೆ ಈ ವಿಳಾಸದಲ್ಲಿ ವರದಿ ಮಾಡಿ: [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report).

ಲಾಗಿನ್ ಮಾಡದೆ ಸಲ್ಲಿಸಲು ಇಚ್ಛಿಸುವವರಾದರೆ, ದಯವಿಟ್ಟು ಈ ವಿಳಾಸಕ್ಕೆ ಇಮೇಲ್ ಮಾಡಿ: [secure@microsoft.com](mailto:secure@microsoft.com). ಸಾಧ್ಯವಾದರೆ, ನಮ್ಮ PGP ಕೀಲಿಯನ್ನು ಬಳಸಿ ನಿಮ್ಮ ಸಂದೇಶವನ್ನು ಎನ್ಕ್ರಿಪ್ಟ್ ಮಾಡಿ; ದಯವಿಟ್ಟು ಇದನ್ನು [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) ನಿಂದ ಡೌನ್ಲೋಡ್ ಮಾಡಿ.

ನೀವು 24 ಗಂಟೆಗಳ ಒಳಗೆ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಸ್ವೀಕರಿಸುವಿರಿ. ಯಾವುದೇ ಕಾರಣಕ್ಕಾಗಿ ಪ್ರತಿಕ್ರಿಯೆ ಸಿಗದೇ ಇದ್ದರೆ, ದಯವಿಟ್ಟು ಇಮೇಲ್ ಮೂಲಕ ಓ ಅವತರಣಿಕೆಯನ್ನು ಫಾಲೋಅಪ್ ಮಾಡಿ ನಮ್ಮು ಮೂಲ ಸಂದೇಶವನ್ನು ನಮಗೆ ಪಡೆದಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಹೆಚ್ಚುವರಿ ಮಾಹಿತಿ [microsoft.com/msrc](https://www.microsoft.com/msrc) ನಲ್ಲಿ ಲಭ್ಯವಿದೆ.

ದಯವಿಟ್ಟು ಕೆಳಗಿನ ಕೋರುವ ಮಾಹಿತಿಯನ್ನು (ನೀವು ಅಷ್ಟು ככל ಸಾಧ್ಯ ವಂತೆ) ಸೇರಿಸಿ, ಇದರಿಂದ ನಾವು ಸಮಸ್ಯೆಯ ಸ್ವಭಾವ ಮತ್ತು ವ್ಯಾಪ್ತಿಯನ್ನು ಉತ್ತಮವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯವಾಗುತ್ತದೆ:

  * ಸಮಸ್ಯೆಯ ಪ್ರಕಾರ (ಉದಾಹರಣೆ: ಬ್ಯುಫರ್ ಓವರ್ಫ್ಲೋ, SQL ಇಂಜೆಕ್ಷನ್, ಕ್ರಾಸ್-ಸೈಟ್ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್, ಇತ್ಯಾದಿ)
  * ಸಮಸ್ಯೆ ಉಂಟಾದ ಮೂಲ ಫೈಲ್(ಗಳ) ಸಂಪೂರ್ಣ ಮಾರ್ಗಗಳು
  * ಪ್ರಭಾವಿತ ಮೂಲ ಕೋಡ್ ಸ್ಥಳ (ಟ್ಯಾಗ್/ಬ್ರಾಂಚ್/ಕಮಿಟ್ ಅಥವಾ ನೇರ URL)
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರುತ್ಪಾದನೆ ಮಾಡಲು ಅಗತ್ಯವಿರುವ ಯಾವುದೇ ವಿಶೇಷ ಸಂರಚನೆ
  * ಸಮಸ್ಯೆಯನ್ನು ಪುನರುತ್ಪಾದನೆ ಮಾಡಲು.step-by-step ಸೂಚನೆಗಳು
  * ಪರಿಚಯ ಪರಿಕಲ್ಪನೆ ಅಥವಾ ದುರ್ಬಳಕೆ ಕೋಡ್ (ಸಾಧ್ಯವಾದರೆ)
  * ಸಮಸ್ಯೆಯ ಪರಿಣಾಮ, ಹೇಗೆ ಶತ್ರು ಇದನ್ನು ದುರ್ಬಳಕೆ ಮಾಡಬಹುದು ಎಂಬುದರ ವಿವರ

ಈ ಮಾಹಿತಿ ನಮಗೆ ನಿಮ್ಮ ವರದಿಯನ್ನು ವೇಗವಾಗಿ ಪರಿಶೀಲಿಸುವಲ್ಲಿ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

ನೀವು ಬಗ್ ಬೌಂಟಿಗಾಗಿ ವರದಿ ಮಾಡುತ್ತಿದ್ದರೆ, ಹೆಚ್ಚು ಪೂರ್ಣ ವರದಿಗಳು ಹೆಚ್ಚಿನ ಬೌಂಟಿ ಬಹುಮಾನಕ್ಕೆ ಕಾರಣವಾಗಬಹುದು. ದಯವಿಟ್ಟು ನಮ್ಮ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ಪುಟವನ್ನು ಭೇಟಿನೀಡಿ ನಮ್ಮ ಸಕ್ರಿಯ ಕಾರ್ಯಕ್ರಮಗಳ ವಿವರಗಳಿಗೆ.
